Go 语言函数

发表于 2024-01-14 更新于 2024-07-23 分类于 Go ， 3.核心概念 阅读次数： 本文字数： 8k 阅读时长 ≈ 27 分钟

函数定义

函数（Function）是能完成程序预定义功能的代码逻辑单位，一个程序至少由一到多个函数组成。函数接受输入并返回输出，数据流经函数时，将被变换。编写函数的目的是为了将冗长的代码划分成较小的功能模块，也方便反复调用程序某些功能，提高代码复用性。

虽然函数编写顺序没规定，但一般约定把 main 函数写在文件前面，其他函数按照一定逻辑顺序（调用顺序和用途等）编写。

Go 语言函数不支持嵌套（nested，函数内部定义命名函数）、重载（overload，一个函数名用于不同的函数实现）、命名实参（named arguments，调用函数时指定参数名字）和默认参数（default parameter，定义函数时为其参数提供默认值）功能。但支持可变参数、多返回值和具名返回值。

函数声明

一个完整的函数声明结构格式如下：

func 函数名(参数1 参数类型1, 参数2 参数类型2...) (返回值1 返回值类型1, 返回值2 返回值类型2...) {
    // 函数体
    return 返回值1, 返回值2... // 返回多值
}

和其他引用类型不同，函数值（functions value，函数类型的值）之间可以互相比较，例如它们引用的是相同的函数或者返回值都是 nil，则可以认为它们是相同的函数。

函数签名

函数第一行叫函数签名（signature），指出函数接受的输入和提供的输出。下面是各部分说明：

  • func：定义函数的关键字。
  • 函数名：函数名命名规则等同于标识符命名规则。
  • 输入参数：函数从外部接受的输入数据。有参数必须声明参数类型和名称。
  • 返回值：返回值是函数执行后返回的内容。有返回值必须声明返回值类型。
  • {}：函数体，能被重复调用的代码片段。左大括号必须和 func 在一行，右大括号必须单独占一行。
  • return：返回语句。如果函数声明中包含返回值，那么函数体内必须包含终止语句，即 return 或导致函数终止的 panic 语句。函数块里终止语句之后的语句都不会执行。

函数调用

函数都有从属的包，根据包的类型可以分为几种调用情况：

  • 内置函数：共有 15 个内置函数，函数名均为小写。这些函数可直接调用，无需导入任何外部包。
  • 标准函数：需要导入所属包后使用，函数名首字母大写。
  • 自定义函数：和主调函数在一个包中时，无需要导入。
  • **自定义包中定义函数：**和标准库一样用法。

不需导入包使用的函数调用格式为：函数名(参数1, 参数2)，需要导入包使用的函数调用格式为：包名.函数名(参数)，参数顺序必须与函数签名完全一致（_ 参数也不能省略）。如果函数声明定义没参数或可变参数，则调用时可以不必传参。

函数种类

如果把方法和匿名函数视为普通函数，那么 Go 语言中的函数按照功能性分为三类。

主函数

在 Go 语言中，main 函数固定作为程序的入口点，整个程序只能有一个 main 函数入口。main 函数必须位于 main 包中，没有参数也没有返回值，程序会在 main 函数执行完毕后结束。其标准格式为：

func main() {
    // 函数体
}

main 函数本身不返回值，但可以通过调用 os.Exit 来结束程序并给操作系统返回一个自定义状态码：

package main

import (
	"fmt"
	"os"
)

func main() {
	fmt.Println("Hello, World!")

	// 自定义退出码
	if 1 < 2 {
		fmt.Println("自定义退出码为 1")
		os.Exit(1) // 非零状态码表示错误
	}

	// 正常结束时的退出
	fmt.Println("默认退出码为 0")
	os.Exit(0) // 通常用 0 表示成功
}

main 函数本身在主协程中运行，不会自动并行执行。如果需要并行处理，需要显式启动其他协程。

初始化函数

init 函数是一个特殊函数，在包被导入时自动调用，会在任何其他代码之前运行，并且不能手动调用。一般用于初始化常量的值（不是定义常量），或对数据进行检验与修复，保证程序正常运行：

package main

import (
	"fmt"
	"os"
)

var GOROOT string

func init() {
	GOROOT = os.Getenv("GOROOT") // 先于 main 函数获取系统变量 GOROOT 的值
}

func init() {
	fmt.Println("init 2")
}

func init() {
	fmt.Println("init 3")
}

func main() {
	fmt.Println(GOROOT) // 打印出全局变量 GOROOT 的值：D:\Software\Program\go\go1.10
}

如果一个源文件中有多个 init 函数，会按照它们在文件中的声明顺序执行。

如果主函数中导入多个包，所有包初始化函数都会在主函数执行前执行，且只执行一次。

普通函数

最常见的函数类型，用于执行特定任务或计算。普通函数可以有参数和返回值：

package main

import "fmt"

func add(a int, b int) int {
    return a + b
}

func main() {
    result := add(3, 4)
    fmt.Println(result)
}

函数返回值

Go 语言中函数返回值可以有零到多个返回值。

无返回值

无返回值的函数指不返回任何值的函数，在函数声明签名中不指定返回类型，函数体中不用 return 关键字来返回一个具体的值。无返回值函数主要用于执行一些操作，比如日志记录、修改配置参数、写入数据库等，不需要向调用者传递结果。

在无返回值的函数中，也可以使用无参数的 return 语句来提前退出函数：

package main

import "fmt"

func strTest(s string) {
	if s == "" {
		fmt.Println("empty string")
		return // 满足条件则提前退出
	}
    
    // 正常流程代码
	fmt.Println("normal string")
}

func main() {
	strTest("test") // 不能获取函数返回值
}

单返回值

最简单的情况是函数返回单一值。可以不对返回值命名，返回类型紧随参数列表之后，非具名返回值不需要冗余括号：

package main

import "fmt"

func add(a int, b int) int { return a + b }

func main() {
	fmt.Println(add(3, 4))
}

多返回值

Go 语言函数支持多返回值，在调用多返回值的函数时，可直接将返回值赋给变量：

package main

import "fmt"

func calculate(a, b int) (int, int) {
	return a + b, a - b
}

func main() {
	sum, diff := calculate(10, 5)
	fmt.Println("两数之和:", sum, "两数之差:", diff) // 输出：两数之和: 15 两数之差: 5
}

除了可以将不需要的返回值赋值给空标识符，Go 语言还允许隐式地忽略函数的所有返回值，如直接调用 calculate(10, 5) 但不给结果赋值。

具名返回值

具名返回值让函数能够在返回前将值赋给具名变量，然后自动返回这些具名变量。要使用具名返回值，可在函数签名的返回值部分指定变量名，函数在开始运行时会自动创建这些具名变量，并初始化为相应类型的零值，可以在函数内部直接使用：

package main

import "fmt"

func returnName() (firstN, lastN string) {
	firstN = "Alex"
	lastN = "Bishop"
	return 
}

func main() {
	fmt.Println(returnName())
}

由于函数签名定义了返回值变量名，因此 return 语句中可以不指明变量名，也被称为「裸 return」语句。

此外，也可以无视具名返回值，返回自定义值。例如：return "A", "B"，函数会自动将 "A" 赋值给第一个返回变量 firstN，将 "B" 赋值给第二个具名返回变量 lastN，然后再返回。

函数参数

Go 语言中函数可以有零到多个参数，每个参数名后面带有类型，并以逗号「,」分隔参数。如果参数列表中多个相邻参数类型相同，可采用复合的形式定义参数：参数1, 参数2 1和2相同参数类型, 参数3 参数类型3。

参数传递

有时能看到实参（Actual Parameter）和形参（Formal Parameter）的说法，实参就是实际参数，指传入函数的变量，而形参则是在函数签名中的形式参数，实参通过值传递的方式将值赋给形参。形参在进行函数调用时自动初始化，不用重复声明，调用结束后释放，所以作用域仅限于函数体内。

在 Go 语言中对值类型参数传递，会直接复制实参的数据副本给行参。对引用类型参数传递，复制的是指针值副本到行参，实参和行参指向的数据内存地址一样，所以可以在函数内部修改外部实参的值。

如果函数参数太多，可以将参数整合到一个结构体中，以便于管理参数：

package main

import "fmt"

// User 定义了用户的结构
type User struct {
	FirstName string
	LastName  string
	Age       int
	Email     string
	Phone     string
}

// registerUser 接收一个 User 类型的参数
func registerUser(user User) {
	fmt.Printf("Registering user: %s %s, Age: %d, Email: %s, Phone: %s\n",
		user.FirstName, user.LastName, user.Age, user.Email, user.Phone)
}

func main() {
	alice := User{
		FirstName: "Alice",
		LastName:  "Smith",
		Age:       30,
		Email:     "alice@example.com",
		Phone:     "123-456-7890",
	}

	// 这样看起来干净多了
    registerUser(alice)
}

可变参数

Go 语言中函数支持可变数量的参数，例如常见的 fmt.Printf 和 exec.Command。要指定变参，在参数列表中使用三个点加类型来定义：变参名 ...参数类型。传入的实参可以看作类型为 变参名 []参数类型 的切片，但函数获取的是值的副本：

package main

import "fmt"

func main() {
	result := sumNumbers(1, 2, 3, 4)       // 如果不传参数，则得到默认返回值 0
	fmt.Printf("The result is %v", result) // 返回运算结果：The result is 10
}

// 可接受任意多个参数的函数
func sumNumbers(numbers ...int) int {
	total := 0
	for _, number := range numbers {
		total += number
	}
	return total
}

上面函数 sumNumbers 可以接受多个整数，并返回它们的和。变量 numbers 是一个包含所有参数的切片。

如果函数要有其他固定参数，需要把变参放在参数列表末尾，作为函数的最后一个参数。例如：func sumNumbers(init int, numbers ...int) int。

任意类型

如果想要传递任意类型的参数，可以指参数定类型为空接口 interface{}：

package main

import "fmt"

func typeCheck(values ...interface{}) {
	for _, value := range values {
		switch v := value.(type) {
		case int:
			fmt.Println("Integer:", v)
		case float64:
			fmt.Println("Float:", v)
		case string:
			fmt.Println("String:", v)
		case bool:
			fmt.Println("Boolean:", v)
		default:
			fmt.Printf("Unrecognized type: %T\n", v)
		}
	}
}

func main() {
	typeCheck(5, 3.14, "Hello", true)
}

一等函数

Go 语言中的函数是「头等公民（first-class citizens）」，所以函数可以像其他数据类型（如整型、字符串等）一样被使用和操作。

函数变量

将函数赋值给变量，则变量的值是函数本身，而不是调用函数获得的结果：

package main

import "fmt"

func main() {
	x := addNum          // 把函数赋值给变量，等价于 var x func(int, int) int = addNum
	fmt.Println(x(1, 2)) // 等于调用 addNum(1, 2)，输出：3
	x = subNum           // 函数签名相同，所以可以赋值
	fmt.Println(x(1, 2)) // 输出：-1
	//x = len // 报错：类型不匹配
}

func addNum(a, b int) int { return a + b }
func subNum(a, b int) int { return a - b }

函数变量本身是一个可调用的函数，上面的 x 隐式地成为了 addNum 函数的签名，即 func(int, int) int 。不能将一个具有不同签名（不同的参数类型或不同的返回值类型）的函数赋值给 x。

函数组合

在函数式编程中，可以将一个函数作为参数传递给其他函数，Go 语言也支持这一特性。只要被调用函数的返回值个数、返回值类型和返回值顺序与调用函数所需输入参数一致，就可以把这个被调用的函数当作其他函数的参数：

package main

import "fmt"

func main() {
	fmt.Println(f2(f1(1, 2))) // 输出计算结果：-3
	fmt.Println(f3(1, 2, f1)) // 输出计算结果：-3
}

func f1(a, b int) (int, int) {
	return a + b, a - b // 函数 f1 返回两个整数值
}

func f2(x, y int) int { // 函数 f2 接受两个整数参数
	return x * y // 返回一个整数
}

func f3(a, b int, f func(int, int) (int, int)) int { // 接受一个函数作为参数
	x, y := f(a, b) // 调用函数 f，获取两个返回值
	return f2(x, y) // 将两个返回值传递给 f2，并返回结果
}

工厂函数

一个返回值为另一个函数的函数，也叫工厂函数，可以用来创建一系列相似的函数：

package main

import (
	"fmt"
	"strings"
)

// FileSuffixAdder 是一个工厂函数，返回一个新的函数，用于给文件名添加特定的后缀名
func FileSuffixAdder(suffix string) func(string) string {
	return func(filename string) string {
		// 检查文件名是否已经有该后缀，如果没有则添加
		if !strings.HasSuffix(filename, suffix) {
			return filename + suffix
		}
		return filename
	}
}

func main() {
	// 创建一个添加".txt"后缀的函数
	addTxtSuffix := FileSuffixAdder(".txt")

	// 创建一个添加".jpg"后缀的函数
	addJpgSuffix := FileSuffixAdder(".jpg")

	// 使用这些函数
	fmt.Println(addTxtSuffix("report"))  // 输出: report.txt
	fmt.Println(addTxtSuffix("log.txt")) // 输出: log.txt (不会重复添加后缀)

	fmt.Println(addJpgSuffix("image"))   // 输出: image.jpg
	fmt.Println(addJpgSuffix("log.txt")) // 输出: log.txt.jpg
}

函数类型

Go 语言中可以定义函数类型：type 类型名 func(参数类型) (返回值类型)。这个类型随后能被用来声明变量，并被赋予任何具有相同签名的函数：

package main

import "fmt"

// 声明一个函数类型
type binOp func(int, int) int

// 实现一个符合该类型签名的函数
func add(a, b int) int {
	return a + b
}

// 另一个符合该类型签名的函数
func multiply(a, b int) int {
	return a * b
}

func main() {
	// 使用函数类型声明变量，并赋予具体的函数
	var operation binOp // 为 nil
	operation = add
	fmt.Println("Addition:", operation(5, 6))  // 输出 Addition: 11

	// 更改函数变量为另一个函数
	operation = multiply
	fmt.Println("Multiplication:", operation(5, 6))  // 输出 Multiplication: 30
}

函数类型是引用类型，默认零值为 nil。函数类型相同是指：拥有相同的参数列表和返回值列表，参数名和返回值名不重要。

回调函数

回调函数是一种常见的编程模式，通过函数类型可以方便地实现回调机制：

package main

import "fmt"

type Callback func(int) int

// process 函数接受参数为函数类型变量
func process(numbers []int, callback Callback) {
	for _, number := range numbers {
		fmt.Println(callback(number))
	}
}

func main() {
	numbers := []int{1, 2, 3, 4, 5}
	// 定义一个匿名函数，符合函数类型 Callback
	double := func(n int) int {
		return n * 2
	}
	// 如果函数类型不一致，会报错：无法将 'double' (类型 func(n int) int) 用作类型 Callback
	// 可以定义多个类似的匿名函数，只要函数类型符合 Callback，都可以作为参数传给 process 函数
    process(numbers, double)
}

高阶函数

高阶函数是指能够接受其他函数作为参数或者返回值是函数的函数。实现高阶函数也是以函数类型作为基础：

package main

import "fmt"

type Adder func(int, int) int

func createAdder() Adder {
	return func(a, b int) int {
		return a + b
	}
}

func main() {
	adder := createAdder()
	fmt.Println(adder(3, 4)) // 输出 7
}

匿名函数

匿名函数（Anonymous-function）是指不需要定义函数名的一种函数实现方式，由一个不带函数名的函数声明和函数体组成，可以看作是函数的字面量。匿名函数通常用于实现局部的、一次性的功能，或者需要封闭特定变量的情况，例如回调函数或闭包。命名函数则更适合需要多次调用的功能，可以在多个不同的上下文中被调用。

匿名函数可赋值于某个变量，即保存函数的地址到变量中，然后通过变量名对函数进行调用和传递：

package main

import "fmt"

func main() {
	f := func(x, y int) int { return x + y } // 可以定义在一行
	fmt.Println(f(1, 2))
}

除此之外还可以直接调用匿名函数，只需要在匿名函数定义最后，用括号传入输入参数：

package main

import "fmt"

func main() {
	fmt.Println(func(x, y int) int { return x + y }(1, 2))
}

闭包

闭包（closure）是包含了自由变量的匿名函数，也就是匿名函数封闭并包围作用域中的变量。闭包允许调用定义在其他环境下的变量，可使得某个函数捕捉到一些外部状态，例如函数被创建时的状态。一个闭包继承了函数声明时的作用域，作用域内的变量会共享到闭包的环境中，因此这些变量可以在闭包中被操作，直到被销毁。

由于 Go 的所有匿名函数都自动捕获周围作用域中的变量，实际上所有的匿名函数都是闭包。闭包常用于 goroutine 和管道操作，以及在异步操作、事件处理和延时计算中封装状态，也就是预先定义好一个或多个参数以用于包装函数：

package main

import "fmt"

func outerFunction(externalArg int) func(int) int {
	externalVar := externalArg
	return func(innerArg int) int {
		return externalVar / innerArg
	}
}

func main() {
	closure := outerFunction(15)
	fmt.Println(closure(5))
}

上面代码的执行流程如下：

  • outerFunction 是一个函数，接受一个类型的参数 externalArg 并返回一个函数。返回的函数也接收一个整数类型的参数，同样返回一个整数类型结果。
  • 在 outerFunction 内部，定义了一个变量 externalVar，并将其设置为参数 externalArg 的值。这个变量在闭包（内部函数）中被使用。
  • 然后，outerFunction 返回一个匿名函数（闭包）。这个匿名函数使用了 outerFunction 的局部变量 externalVar，并接受自己的参数 innerArg。闭包的功能是计算 externalVar 除以 innerArg 的结果。
  • 通过返回这个匿名函数，outerFunction 实际上返回了一个闭包。这个闭包保留被创建时 externalVar 的值。
  • 在 main 函数中，首先调用了 outerFunction，传递了 15 作为参数。outerFunction 执行后返回一个闭包，这个闭包存储了变量 externalVar 为 15 的环境。变量 externalVar 被分配到堆上，而堆上的变量不会随着函数返回自动消失，只有用完之后才会被垃圾回收。然后，将这个闭包赋值给变量 closure。
  • 接下来，调用 closure 函数（实际上是 outerFunction 返回的闭包），传递 5 作为参数。闭包内部执行 15 / 5，得到结果 3。

通过闭包封装状态（在这里是 externalVar 的值），在函数返回后仍然能够访问这个状态。闭包使得 externalVar 的生命周期得以延续，即使 outerFunction 执行完毕。

延迟求值

闭包会捕获其作用域内的变量引用而不是变量的副本，导致在使用循环创建闭包时，所有闭包都会捕获对同一个循环变量的引用，而不是在闭包创建时该变量的值：

package main

import "fmt"

// test 函数返回元素为函数的切片
func test() []func() {
	var s []func()

	for i := 0; i < 2; i++ {
		s = append(s, func() {
			fmt.Println(&i, i)
		})
	}

	return s
}

func main() {
	for _, f := range test() {
		f() // 打印出来一样的地址和值 2
	}
}

上面每个闭包都引用了 i，而不是 i 在闭包创建时的值。因此当闭包在循环外部执行时，都会打印相同的值，也就是 i 为 2 的状态。

从 Go 1.22 版本开始，每次迭代的循环变量都有自己的独立作用域，这样每个闭包会捕获到正确的迭代值，而不是循环结束时的最后一个值。相当于上面代码中循环内初始化变量 i 会在每次迭代中独立存在，所以最终运行结果中，i 的值为 0 和 1，指针地址也不同。 相当于之前 test 函数循环中引入一个额外变量，再将 i 赋值给新变量，并传入到函数切片中。

递归函数

递归函数（Recursive-function）是在其函数体内，含有直接或间接地调用自身语句的函数，递归调用自身的过程称为递归调用（Recursive-call）。要使用递归，可将调用自身的代码作为终止语句中的返回值。

阶乘是一个经典的递归函数示例，定义为：n! = n × (n-1) × (n-2) × ... × 1，0! = 1。下面是用递归来实现的代码：

package main

import "fmt"

// factorial 函数使用递归计算整数 n 的阶乘
func factorial(n int) int {
	if n == 0 { // 终止条件：0 的阶乘为 1
		return 1
	}
	// 递归情况：n 的阶乘是 n 乘以 n-1 的阶乘
	return n * factorial(n-1)
}

func main() {
	fmt.Println("5! =", factorial(5)) // 等同于：5*4*3*2*1 输出 5! = 120
}

递归函数在每次调用自己时，目标是更接近最终结果，因此必须有一个终止递归调用的条件控制（基本情况）。这里的的终止条件为 n 等于 0 时，函数返回 1。

如果 n 不是 0，函数计算 n * factorial(n-1)。这里 factorial(n-1) 是对自身的递归调用（递归步骤），这个过程持续到 n 降到 0。

延迟语句

defer 用于延迟调用指定函数，通常用于函数执行完毕后及时地释放资源，例如打开/关闭连接、加锁/释放锁、打开/关闭文件。

defer 语句按照逆序（后进先出）执行，也就是当代码执行到一个 defer 时，会将语句压入专门的栈中（带上相关的变量值），当场不执行，直到函数执行完毕，再从栈顶依次取出语句执行。

defer 语句

defer 能够让函数返回或遇到 panic 异常（不包括 os.Exit）前执行外部的一个函数或方法，可以支持匿名函数调用：

package main

import "fmt"

func main() {
	for i := 0; i < 3; i++ {
		fmt.Println("循环值 =", i)   // 先执行循环输出：循环值 = 0 循环值 = 1 循环值 = 2
		defer fmt.Printf("%v", i) // 然后一口气执行输出：210
	}
}

defer 语句通常用于执行清理操作或释放资源：

package main

import (
	"fmt"
	"os"
)

func main() {
	fmt.Print(ReadFile(`D:\Software\Programming\Go\my\src\file.txt`))
}

func ReadFile(filename string) bool {
	f, err := os.Open(filename)
	if err != nil {
		fmt.Println("Failed to open file:", err.Error())
		return false
	}

	defer func(f *os.File) {
		if err := f.Close(); err != nil {
			fmt.Println("Failed to close file:", err.Error())
		}
	}(f)
	return true
}

上面代码用于在函数执行完毕之前，不管是否成功，都将安全地关闭文件读写，延迟关闭逻辑且通常和文件打开代码放在一块（错误检查语句之后）。

定义位置

虽然函数发生异常时，defer 语句也能正常执行，但是必须注意延迟语句声明的位置。如果在可能引发异常的代码之后声明延迟语句，那么发生异常时，延迟语句还没有注册，自然不会运行：

package main

import (
	"fmt"
	"sync"
)

func main() {
	var lock sync.Mutex

	// 有可能因为发生异常，没有释放锁，造成死锁
	lock.Lock()
	fmt.Println("有可能引发异常的代码")
	lock.Unlock()

	// 错误的放置，发生异常时，不会执行 defer
	lock.Lock()
	fmt.Println("有可能引发异常的代码")
	defer lock.Unlock()

	// 正确用法，在申请锁之后，立即使用 defer
	lock.Lock()
	defer lock.Unlock()
	fmt.Println("有可能引发异常的代码")
}

多个 defer

函数中可以添加多个 defer 语句：

package main

import "fmt"

func main() {
	defer fmt.Println("To the End1") // 最后执行
	defer fmt.Println("To the End2") // 第二执行
	fmt.Println("Running")           // 最先执行
}

上面代码中，虽然 defer 语句定义出现在 fmt.Println 函数调用前，但它会等待函数执行完毕后，再按倒序执行每一条延迟语句。

defer 中闭包和位置

defer 语句定义的位置会影响其作用时使用的变量值。这是因为 defer 语句会在被定义的那一刻捕获相关变量的当前值，而不是在函数返回时的值。当 defer 被执行时，它使用的是这些捕获的值：

package main

import "fmt"

func main() {
	f1()
}

func f1() {
	i := 0
	defer func() {
		fmt.Println("闭包 defer", i) // 使用闭包捕获变量 i，在返回前的值为 100
	}()
	defer func(i int) {
		fmt.Println("函数 defer", i) // 这是函数不是闭包，立即求得 i 的值为 0
	}(i)
	defer fmt.Println("前个 defer", i) // 立即求得 i 的值为 0
	i++
	defer fmt.Println("后个 defer", i) // 立即求得 i 的值为 1
	i = 100
	return
}

defer fmt.Println("前个 defer", i) 这行代码在 i++ 执行之前定义，因此它捕获并使用的 i 的值是 0。所以，尽管i 在defer语句之后被递增，但 defer 语句已经捕获了 i 递增前的值。

defer fmt.Println("后个 defer", i) 捕获的是 i++ 之后的值，即 1。因此，当函数返回时，defer 语句打印的将是 1。

在第一个 defer 中，匿名函数是一个闭包，它捕获并引用了变量 i 而不是 i 的值。当 defer 语句执行时（即在 f1 函数返回之前），它会打印出 i 的当前值，此时 i 已经被更新为 100。

在第二个 defer 中，后面的是函数调用，所以捕获的是 i 的值 0。

简单总结为：

  • 在 defer 语句中直接使用函数调用时，函数的所有参数都会在 defer 语句被执行的那一刻立即求值。
  • 当使用 defer 关键字后跟一个闭包时，闭包可以捕获到其周围作用域中的变量引用，而不是变量的值。因此，闭包内部使用的变量的最终值，将是在包围函数（如 main 函数）即将返回之前变量的当前值。

另外一个著名的例子：

package main

import "fmt"

func main() {
	for i := 0; i < 3; i++ {
		defer func() {
			fmt.Println(i) // 输出三次 3
		}()
	}
}

由于捕获的是变量 i，在循环结束后 i 的值变为 3，所以最后 defer 语句输出三次 3，没有实用价值。

要想倒序打印 2、1、0，可以升级到 Go 1.22 版本：对于 for 循环子语句中的变量，不再是固定不变，而是每次循环都分配一个新的内存地址，因此最终输出的是每次循环中求得的值。或者使用下面的方式修改函数：

package main

import "fmt"

func main() {
	for i := 0; i < 3; i++ {
		// 接受并传入变量 i，虽然 i 地址不变，
		// 但是每次循环传参复制 i 的值进函数，
		// 函数现在使用的是 i 的副本，有独立的地址
		defer func(i int) {
			fmt.Println(i)
		}(i)
	}
}

defer 和具名返回值

函数的返回值是否命名，可能会对 defer 行为造成影响，具名返回值可以被函数中的 defer 语句访问和修改：

package main

import "fmt"

func main() {
	fmt.Println("f1 return", f1()) // 输出：0
	fmt.Println("f2 return", f2()) // 输出：1
}

func f1() int {
	var i int
	defer func() { i++; fmt.Println("i in f1", i) }() // 输出：i in f1 1
	return i
}

func f2() (i int) {
	defer func() { i++; fmt.Println("i in f2", i) }() // 输出：i in f2 1
	return
}

按正常理解，函数 f1 和具名返回值函数 f2 应该返回相同的值 1，然而事实并非如此。在执行 return i 时，会发生以下几步：

  1. 返回值决定： f1 中标记变量 i 的当前值 0 为返回值，而 f2 标记变量 i（此时为 0）作为返回值。
  2. defer 执行：执行 defer 语句，i 被递增为 1。
  3. 函数返回：f1 函数返回已经确定的返回值 0，f2 函数返回变量 i（此时值为 1）。

由于 f2 使用了命名返回值，任何对 i 的修改，包括在 defer 函数中的修改，都会反映到最终的返回值中。

为了避免这个陷阱，可以完全不使用具名返回值，或者禁止在 defer 语句中对返回值进行修改。

错误处理

在编程中错误和异常经常被混淆，理解两者差异对于编写健壮的程序很重要：

  • 错误：错误用来表示在程序执行过程中可能遇到的预期问题，例如文件未找到、网络连接失败等，错误应由程序逻辑处理。Go 语言通过内置 error 接口实现错误处理。
  • 异常：异常用于表示程序中预料外的不可恢复的错误情况，通常是严重的错误，如数组越界、空指针引用等。异常发生时，需要停止正常的程序流程。Go 语言不能抛出异常，而是使用 panic 和 recover 函数作为代替。

error

一个错误值是 Error 方法返回的字符串，通过检测函数返回的 error 值是否为 nil 来判断是否有错误：

package main

import (
	"errors"
	"fmt"
	"log"
)

func main() {
	err := errors.New("错误！")           // 标准错误定义方法，只支持字符串
	err = fmt.Errorf("错误次数 %d", 4)     // 支持字符串格式化方法
	varErr := err.Error()              //  Error 方法返回的字符串
	fmt.Printf("%T %T\n", err, varErr) // 输出类型：*errors.errorString string
	fmt.Println(err)                   // 对于 fmt 包可以直接传递 err，能自动获取错误字符串
	log.Fatal(err)                     // 对于 log 包也可以直接传递 err，log.Fatal 会导致程序立即退出
	fmt.Println(varErr) // 不会被执行
}

对于大部分函数，都可以将 err 作为多返回值中的最后一个返回值：

package main

import (
	"errors" // 引入errors包来构造error对象
	"fmt"
	"log"
	"os"
)

func main() {
	n, err := Foo(0)
	log.Println(err) // 正常打印日志信息
	if err != nil {
		fmt.Println(err)
		os.Exit(123) // 返回退出码 123
	} else {
		fmt.Println(n)
	}
}

func Foo(param int) (int, error) {
	if param == 0 {
		return 0, errors.New("division by zero") // 当除数为 0 时，返回错误
	}
	return 10 / param, nil // 如果没有错误，正常返回计算结果
}

可见判断条件 if err != nil 是常见的错误检查模式。

panic

panic 是一个内建函数，可以中断原有控制流程，主动抛出错误。当发生 panic 时，函数的执行会被中断，函数中的 defer 语句会正常执行，然后逐层向上执行函数的 defer 语句，打印调用堆栈，直到被 recover 捕获或运行到最外层函数时才会退出。

panic 可以直接调用 panic 产生，也可以由运行时错误（runtime error）产生。panic 函数参数是空接口，接收任意类型的数据：

package main

import (
	"fmt"
	"os"
)

func main() {
	var user = os.Getenv("USERNAME")
	defer func() {
		fmt.Println("用户名: ", user)

		i := 8
		arr := [8]string{}
		arr[i] = "hello world" // 下标超标引起异常: runtime error: index out of range
	}()
	if user != "root" {
		panic("请用 root 用户运行") // 主动调用 panic 引起异常: 请用 root 用户运行   
	}
}

通常在流程出现不可修复性错误时才使用 panic，否则应该用 error。

recover

如果发生的 panic 不影响主要功能，可以调用 recover 函数捕获 panic 的输出值，让进入异常的 goroutine 恢复正常执行，避免整个程序崩溃不可用。

由于 recover 函数用于终止错误处理流程，所以一般定义在 defer 语句中，以有效截取错误处理流程，也只有在 defer 的函数内直接调用才会终止错误，否则 recover总是返回 nil，没有任何作用。

在 recover 生效后，继续执行 defer 所在函数的下一条语句：

package main

import (
	"fmt"
	"log"
)

func main() {
	n, err := Foo(0) // 会引发异常的参数值
	if err != nil {
		fmt.Println("ERROR!", err)
	} else {
		fmt.Println("VALUE", n)
	}
	fmt.Println("验证恢复逻辑是否生效") // 由于加入 recover()，这条语句忽略 Foo 执行结果，一定会执行
}

func Foo(param int) (n int, err error) {
	defer func() {
		if r := recover(); r != nil {
			err = fmt.Errorf("捕获到异常！%v", r)
			log.Printf("捕获到异常！%v", r) // 用日志格式打印
		}
	}()
	return 10 / param, err
}

如果在没有发生异常的 goroutine 中直接调用 recover 函数，会导致进程打印异常信息后直接退出。
