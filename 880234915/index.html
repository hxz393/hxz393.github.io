Go 语言中文件操作

发表于 2024-06-03 更新于 2024-07-23 分类于 Go ， 4.基本应用 阅读次数： 本文字数： 3.8k 阅读时长 ≈ 13 分钟

I/O 包

在 Go 语言中，进行输入输出（I/O）操作通常用到几个内置的库，包括 io、ioutil、os、bufio 等：

  • io 包：提供了基本的接口来处理 I/O 操作，主要是 Reader 和 Writer 读写接口。

  • bufio 包：在 io 包的基础上提供了缓冲功能，适用于频繁读写操作。

  • os 包：提供与操作系统交互的接口，包括文件操作、环境变量访问、系统调用等。

  • ioutil 包：封装一些实用的 I/O 函数。在 Go 1.16 版本之后，ioutil 包被标记为废弃，函数被迁移到 os、io 和bufio 等包中。

在 io 包中，最重要的两个接口是 Reader 和 Writer，这两个接口定义了基本的数据读写操作，是许多 I/O 功能的基础。

Reader 接口

Reader 接口定义了从数据源读取数据的方法：

type Reader interface {
    Read(p []byte) (n int, err error)
}

这个接口只包含一个 Read 方法，用于从数据源读取最多 len(p) 字节的数据填充到 p 切片中，返回读取的字节数 n 和错误。如果数据源到达末尾会返回 io.EOF 错误。

Writer 接口

Writer 接口定义了一个基本的写入方法：

type Writer interface {
    Write(p []byte) (n int, err error)
}

Write 方法接受一个字节切片 p，并尝试写入数据流，返回实际写入的字节数 n 和错误。在正常情况下，n 等于 len(p)，除非遇到错误或数据流无法写入更多的数据。

解析路径

使用 filepath 包中的 Base 和 Dir 函数能将一个文件路径字符串分解为目录和文件名两部分，针对文件还能使用 Ext 函数来分解出文件扩展名。所有针对路径操作的函数都会调用 Clean 函数，用于优化路径显示：

package main

import (
	"fmt"
	"path/filepath"
)

func main() {
	path := "./internal/hello/world.go"

	// 优化路径
	fmt.Println(filepath.Clean(path)) // 输出：internal\hello\world.go
	// 分离路径
	fmt.Println(filepath.Dir(path)) // 输出：internal\hello
	// 分离文件名
	fmt.Println(filepath.Base(path)) // 输出：world.go
	// 分离扩展名
	fmt.Println(filepath.Ext(path)) // 输出：.go

	// 如果传入的是一个目录路径（结尾是 /），则不能分离出文件名和扩展名
	fmt.Println(filepath.Dir("/a/b/"))  // 输出：\a\b
	fmt.Println(filepath.Base("/a/b/")) // 输出：b
	fmt.Println(filepath.Ext("/a/b/"))  // 无输出
}

可以看到，和传入的格式不同，返回的结果中使用 \ 作为分隔符。因此在涉及路径操作时，可以使用 os.PathSeparator 来指代路径分隔符。

此外还能使用 Split 函数来切分路径，直接获得路径名和文件名：

package main

import (
	"fmt"
	"path/filepath"
)

func main() {
	path := "./internal/hello/world.go"
	// 等同于分别调用 Dir 和 Base
	dirName, fileName := filepath.Split(path)

	fmt.Println(dirName, fileName) // 输出：./internal/hello/ world.go
}

拼接路径

若要将指定的字符串凭借成一个路径，需要使用 filepath.Join 来操作：

package main

import (
	"fmt"
	"path/filepath"
)

func main() {
	dirName, fileName := filepath.Split("./internal/hello/world.go")

	// 专注把字符串拼起来，不用担心路径分隔符写反
	finalPath := filepath.Join(dirName, "/g", "\\o0", "d", fileName)
	fmt.Println(finalPath) // 输出：internal\hello\g\o0\d\world.go
}

读取文件

在 Go 语言中，读取文件可以通过多种方式实现，比如读取整个文件、逐行读取或是使用缓冲读取。

ioutil.ReadFile

这是在 1.16 版本之前最常使用的读取文件方式，适用于较小的文件，将整个文件内容加载到内存中：

package main

import (
	"fmt"
	"io/ioutil"
	"log"
)

func main() {
	// 指定读取的文件路径，输出字节切片
	content, err := ioutil.ReadFile("verified_contents.json")
	if err != nil {
		log.Fatal(err)
	}

	// 转为字符串打印出来
	fmt.Println(string(content))
}

os.Open

如果需要逐步处理文件的数据，可以使用 os.Open 配合 io.Reader 接口来读取文件：

package main

import (
	"bufio"
	"fmt"
	"log"
	"os"
)

func main() {
	file, err := os.Open("verified_contents.json")
	if err != nil {
		log.Fatal(err)
	}
	defer file.Close()

	// 逐行读取方式，每次读取返回一个布尔值
    scanner := bufio.NewScanner(file) // 为文件创建一个扫描器
    // 逐行扫描内容，直到文件末尾。如果遇到错误也会中断返回
	for scanner.Scan() { 
		fmt.Println(scanner.Text())
	}

	// 如果扫描文件时出现过错误，则报告致命错误。主要确保扫描没有因为遇到错误提前退出
    if err := scanner.Err(); err != nil {
		log.Fatal(err)
	}
}

如果需要指定读取位置，尝试使用 ReadAt 方法。

os.OpenFile

如果需要设置文件权限或者选择不同的文件打开方式（只写、追加等），可以使用 os.OpenFile 来指定文件的打开模式和权限:

package main

import (
	"log"
	"os"
)

func main() {
	// os.O_WRONLY 只写模式, os.O_CREATE 如果不存在则创建, os.O_APPEND 追加模式
	// 0644 是以 Unix 格式设置文件的读、写、执行权限
	file, err := os.OpenFile("newfile.txt", os.O_WRONLY|os.O_CREATE|os.O_APPEND, 0644)
	if err != nil {
		log.Fatal(err)
	}
	defer file.Close()

	// 追加内容到文件
	_, err = file.WriteString("Appending new line\n")
	if err != nil {
		log.Fatal(err)
	}
}

打开文件时使用的第二个标志参数决定了文件的打开模式：

  • os.O_RDONLY：只读模式。
  • os.O_WRONLY：只写模式。
  • os.O_RDWR：读写模式。
  • os.O_APPEND：追加写模式。
  • os.O_CREATE：如果指文件不存在则创建。
  • os.O_EXCL：与 os.O_CREATE 一起使用时，文件必须不存在，否则将打开文件失败。
  • os.O_SYNC：开启同步模式，没有写缓冲。
  • os.O_TRUNC：打开时清空文件。

bufio.Reader

文件很大的情况下，可以使用 bufio.Reader 来缓冲读取：

package main

import (
	"bufio"
	"fmt"
	"io"
	"log"
	"os"
)

func main() {
	file, err := os.Open("verified_contents.json")
	if err != nil {
		log.Fatal(err)
	}
	defer file.Close()

	reader := bufio.NewReader(file)
	for {
		line, err := reader.ReadString('\n')
		if err != nil {
			if err == io.EOF {
				break
			}
			log.Fatal(err)
		}
		fmt.Print(line)
	}
}

读取目录

读取目录指列出目录中的所有文件和子目录，通常使用 os 和 io/ioutil 包中的函数来实现。

ioutil.ReadDir

使用 ioutil.ReadDir 读取指定目录，返回 entry 对象列表：

package main

import (
	"fmt"
	"io/ioutil"
	"log"
)

func main() {
	entries, err := ioutil.ReadDir(".")
	if err != nil {
		log.Fatal(err)
	}

	// 遍历获取对象名字和是否是目录的标记
	for _, entry := range entries {
		fmt.Println(entry.Name(), entry.IsDir())
	}
}

os.ReadDir

使用 os.ReadDir 代替 ioutil.ReadDir 方法：

package main

import (
    "fmt"
    "os"
    "log"
)

func main() {
    entries, err := os.ReadDir(".")
    if err != nil {
        log.Fatal(err)
    }

    for _, entry := range entries {
        fmt.Println(entry.Name())
        // 使用 entry.Info() 获取更多信息，如是否是目录
        info, err := entry.Info()
        if err != nil {
            log.Fatal(err)
        }
        fmt.Println("Is Directory:", info.IsDir())
    }
}

entry.Info 返回的 FileInfo 对象中包含下面信息：

  • Name：文件或目录名；
  • Size：文件大小，单位为字节。目录大小为 0；
  • Mode：文件或目录权限；
  • ModTime：文件或目录最后修改时间；
  • IsDir：是否为目录；
  • Sys：底层数据源。

可以按照需要调用获取值。

filepath.Walk

当需要递归地读取一个目录及其所有子目录中的文件，可以使用 path/filepath 包的 Walk 函数来遍历，并对每个对象调用回调函数：

package main

import (
	"fmt"
	"log"
	"os"
	"path/filepath"
)

func visit(path string, info os.FileInfo, err error) error {
	if err != nil {
		return err
	}
	fmt.Println(path, info.IsDir())
	return nil
}

func main() {
	// 对每个文件或目录调用 visit 函数处理
	err := filepath.Walk(".", visit)
	if err != nil {
		log.Fatal(err)
	}
}

Walk 函数不会处理符号链接（快捷方式）。

获取信息

查看文件或目录属性信息可以使用 os.Stat 函数，返回一个 FileInfo 接口，供进一步查询：

package main

import (
	"fmt"
	"os"
)

func main() {
	fileInfo, err := os.Stat("internal")
	if err != nil {
        // 通过传入 err 参数判断文件是否存在
		if os.IsNotExist(err) {
			fmt.Println("文件不存在")
		} else {
			fmt.Println("获取文件信息时出错：", err)
		}
		return
	}

	fmt.Println("文件名：", fileInfo.Name())
	fmt.Println("大小：", fileInfo.Size()) // 如果是目录，大小始终为 0
	fmt.Println("权限：", fileInfo.Mode())
	fmt.Println("最后修改时间：", fileInfo.ModTime())
	fmt.Println("是目录：", fileInfo.IsDir()) // 是目录则返回 true
}

创建文件

创建文件通常使用 os 包中函数来完成。

ioutil.WriteFile

这个函数本身设计用于写入数据到文件，但也能用来创建文件：

package main

import (
	"io/ioutil"
	"log"
)

func main() {
	// 要写入的数据必须是字节切片
	b := make([]byte, 0)
	// 指定文件名、数据内容和权限。这里创建一个空文件
	err := ioutil.WriteFile("newfile.txt", b, 0644)
	if err != nil {
		log.Fatal(err)
	}
}

os.Create

直接创建新文件，如果文件已经存在，将清空文件内容，返回一个可以用于读写（权限为 0666）的文件对象：

package main

import (
	"log"
	"os"
)

func main() {
	// 指定文件名，创建一个新文件并打开，返回一个可操作的文件对象
	file, err := os.Create("newfile.txt")
	if err != nil {
		log.Fatal(err)
	}
	defer file.Close()

	// 写入内容到文件
	_, err = file.WriteString("Hello, world!\n")
	if err != nil {
		log.Fatal(err)
	}
}

创建目录

创建目录使用 os 包中的 Mkdir 或 MkdirAll 函数来实现。

os.Mkdir

os.Mkdir 函数用于创建单个目录。如果父目录不存在，则创建会失败：

package main

import (
	"log"
	"os"
)

func main() {
	// 创建一个目录，无任何权限
	err := os.Mkdir("new_directory", 0000)
	// 将权限设置为 0755，只针对 Linux 有用
	err = os.Chmod("new_directory", 0755)
	if err != nil {
		log.Fatalf("Failed to create directory: %s", err)
	} else {
		log.Println("Directory created successfully.")
	}
}

os.MkdirAll

os.MkdirAll 用来创建多级目录结构，如果中间目录不存会自动创建：

package main

import (
    "os"
    "log"
)

func main() {
    // 创建多级目录结构，权限同样设置为 0755
    err := os.MkdirAll("new_directory/sub_directory/sub_sub_directory", 0755)
    if err != nil {
        log.Fatalf("Failed to create directories: %s", err)
    } else {
        log.Println("Directories created successfully.")
    }
}

使用这个函数能保证目录结构最终结果和期望的一致。

写入文件

写入文件可以通过不同的方式进行，比如写入字符串、字节数据或按行写入。不论使用哪种方式，都应检查返回的错误，确保写入操作正确完成。

os.Write

当已经打开文件后，可以使用 os.Write 方法写入字节数组，或使用 os.WriteString 方法写入字符串:

package main

import (
	"log"
	"os"
)

func main() {
	// 先创建一个空文件
	file, err := os.Create("newfile.txt")
	if err != nil {
		log.Fatal(err)
	}
	defer file.Close()

	// 写入字节数据，成功则返回写入的字节数
	bytes, err := file.Write([]byte("Hello, world!\n"))
	if err != nil {
		log.Fatal(err)
	}
	log.Printf("Wrote %d bytes.\n", bytes)

	// 直接写入字符串，也是调用 Write 来写入，内部做了字符串转字节数组处理
	_, err = file.WriteString("Another hello, world!\n")
	if err != nil {
		log.Fatal(err)
	}
}

和读取文件类似，有个对应的 WriteAt 方法，用于指定数据写入

bufio.Writer

如果需要缓冲写入，可以使用 bufio.Writer。在多次写入少量数据时性能最好：

package main

import (
	"bufio"
	"log"
	"os"
)

func main() {
	file, err := os.Create("newfile.txt")
	if err != nil {
		log.Fatal(err)
	}
	defer file.Close()

	// 创建 Writer 结构体，然后对其写入字符串
	writer := bufio.NewWriter(file)
	_, err = writer.WriteString("Buffered string\n")
	if err != nil {
		log.Fatal(err)
	}

	// 刷新缓存，确保所有内容都被写入底层的 io.Writer
	err = writer.Flush()
	if err != nil {
		log.Fatal(err)
	}
}

使用缓冲写入方式时，如果在缓冲区被完全写入到硬盘之前发生断电或系统故障，可能会导致数据丢失。

fmt.Fprintf

fmt.Fprintf 也可用来写入文件，并且还能先对数据进行格式化字符串操作：

package main

import (
	"fmt"
	"log"
	"os"
)

func main() {
	file, err := os.Create("newfile.txt")
	if err != nil {
		log.Fatal(err)
	}
	defer file.Close()

	// 格式化字符串后写入文件
	_, err = fmt.Fprintf(file, "Value: %d\n", 43)
	if err != nil {
		log.Fatal(err)
	}
}

复制文件

复制文件涉及到从源文件读取内容并写入到目标文件。

io.Copy

io.Copy 是最直接的复制文件方法，可以在读取源文件的同时写入到目标文件，适合复制大文件：

package main

import (
	"io"
	"log"
	"os"
)

func copyFile(src, dst string) error {
	// 打开源文件
	sourceFile, err := os.Open(src)
	if err != nil {
		return err
	}
	defer sourceFile.Close()

	// 创建并打开目标文件
	destinationFile, err := os.Create(dst)
	if err != nil {
		return err
	}
	defer destinationFile.Close()

	// 复制操作，返回已复制的数据大小
	n, err := io.Copy(destinationFile, sourceFile)
	if err != nil {
		return err
	}
	log.Printf("Successfully copied %d bytes from %s to %s\n", n, src, dst)

	return destinationFile.Sync() // 确保数据已被写入到硬盘
}

func main() {
	src := "verified_contents.json"
	dst := "newfile.txt"
	if err := copyFile(src, dst); err != nil {
		log.Fatal(err)
	}
}

io.CopyBuffer

使用 io.CopyBuffer 可以指定一个自定义的缓冲区来进行复制：

package main

import (
	"io"
	"log"
	"os"
)

func copyFileWithBuffer(src, dst string, buf []byte) error {
	sourceFile, err := os.Open(src)
	if err != nil {
		return err
	}
	defer sourceFile.Close()

	destinationFile, err := os.Create(dst)
	if err != nil {
		return err
	}
	defer destinationFile.Close()

	// 比 io.Copy 多了一个 buf 参数
	_, err = io.CopyBuffer(destinationFile, sourceFile, buf)
	if err != nil {
		return err
	}

	return destinationFile.Sync() // 确保数据已被写入到硬盘
}

func main() {
	src := "verified_contents.json"
	dst := "newfile.txt"
	buf := make([]byte, 1024*1024) // 1 MB 大小缓存
	if err := copyFileWithBuffer(src, dst, buf); err != nil {
		log.Fatal(err)
	}
}

os.WriteFile

也可以直接使用 os.ReadFile 读取整个文件到内存，然后用 os.WriteFile 写入到目标。使用这两个函数不需要手动关闭文件，但要注意内存占用量：

package main

import (
	"log"
	"os"
)

func copyFileSimple(src, dst string) error {
	data, err := os.ReadFile(src)
	if err != nil {
		return err
	}
	err = os.WriteFile(dst, data, 0644)
	if err != nil {
		return err
	}
	return nil
}

func main() {
	src := "verified_contents.json"
	dst := "newfile.txt"
	if err := copyFileSimple(src, dst); err != nil {
		log.Fatal(err)
	}
}

删除文件

使用 os.Remove 函数可以删除单个文件。如果文件不存在，会返回一个错误：

package main

import (
	"log"
	"os"
)

func main() {
	// 直接删除，不会到回收站
	err := os.Remove("newfile.txt")
	if err != nil {
		log.Fatalf("Failed to delete file: %s", err)
	} else {
		log.Println("File deleted successfully.")
	}
}

删除目录

根据是否空目录，使用的函数不一样。

os.Remove

os.Remove 除了能删除文件，还适用于删除空目录。如果目录不为空，尝试删除会报错：

package main

import (
	"log"
	"os"
)

func main() {
	err := os.Remove("empty_directory")
	if err != nil {
		log.Fatalf("Failed to delete directory: %s", err)
	} else {
		log.Println("Directory deleted successfully.")
	}
}

os.RemoveAll

要删除目录及其内容，可以使用 os.RemoveAll 函数。目录不存在时不会报错：

package main

import (
    "os"
    "log"
)

func main() {
    err := os.RemoveAll("directory_to_delete")
    if err != nil {
        log.Fatalf("Failed to delete directory and its contents: %s", err)
    } else {
        log.Println("Directory and all contents deleted successfully.")
    }
}

重命名

重命名文件或目录可以通过 os 包中的 Rename 函数来实现。函数接受两个参数，当前文件或目录的路径（oldpath）和新的路径（newpath），参数调用顺序遵循操作系统习惯。如果目标：

package main

import (
	"log"
	"os"
)

func main() {
	file, _ := os.Create("old_filename.txt")
	file.Close()

	// 将文件从 old_filename.txt 重命名为 new_filename.txt
	err := os.Rename("old_filename.txt", "new_filename.txt")
	if err != nil {
		log.Fatalf("Error renaming file: %s", err)
	}
}

移动

Rename 函数也用于移动文件或目录，如果目标文件已存在则覆盖，如果目标目录已存在则报错：

package main

import (
	"log"
	"os"
)

func main() {
	_ = os.Mkdir("new", 0000)
	file, _ := os.Create("filename.txt")
	file.Close()

	// 将文件移动到指定目录，
	err := os.Rename("filename.txt", "new/mv.txt")
	if err != nil {
		log.Fatalf("Error moving file: %s", err)
	}

	// 移动目录
	err = os.Rename("new", "new_directory/new")
	if err != nil {
		log.Fatalf("Error moving dir: %s", err)
	}
}

原子操作

在进行文件写入操作时，意外的断电或系统崩溃，可能造成文件或文件系统损坏。可以实现一个原子文件复制操作，来安全地写入或复制文件，步骤是先将文件写入一个临时位置，确认文件已经完整无误地写入后，再将其重命名为目标文件名：

package main

import (
	"io"
	"log"
	"os"
)

// copyFileAtomic 采用原子文件复制操作
func copyFileAtomic(src, dst string) error {
	// 打开源文件
	sourceFile, err := os.Open(src)
	if err != nil {
		return err
	}
	defer sourceFile.Close()

	// 创建临时文件
	tmpDst := dst + ".tmp"
	destinationFile, err := os.Create(tmpDst)
	if err != nil {
		return err
	}
	defer destinationFile.Close()

	// 将数据从源文件复制到临时文件
	if _, err := io.Copy(destinationFile, sourceFile); err != nil {
		os.Remove(tmpDst) // 如果复制失败，删除临时文件
		return err
	}

	// 确保所有数据都写入硬盘
	if err := destinationFile.Sync(); err != nil {
		os.Remove(tmpDst)
		return err
	}

	// 关闭文件以确保在重命名之前不持有任何文件句柄
	if err := destinationFile.Close(); err != nil {
		os.Remove(tmpDst)
		return err
	}

	// 将临时文件重命名为目标文件为绝对的原子操作
	if err := os.Rename(tmpDst, dst); err != nil {
		os.Remove(tmpDst)
		return err
	}

	return nil
}

func main() {
	src := "verified_contents.json"
	dst := "destination.txt"
	if err := copyFileAtomic(src, dst); err != nil {
		log.Fatal(err)
	} else {
		log.Println("File was copied successfully.")
	}
}

如此处理之后，即使写入过程中出现意外，也不会影响原始目标文件。
