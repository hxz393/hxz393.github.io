Go 语言常量和变量

发表于 2024-01-18 更新于 2024-07-23 分类于 Go ， 3.核心概念 阅读次数： 本文字数： 5.9k 阅读时长 ≈ 20 分钟

定义

高级语言通过一个标识符绑定一块特定内存，后续对特定内存的操作都可以使用该标识符来替代，这类标识符分为变量（Variable）和常量（Constant）。

变量表示指向的内存可被修改，常量表示指向的内存不能被修改。不同于变量在运行期分配内存，以便存储和修改值，常量会被编译器在预处理阶段直接展开，所有引用该常量的地方都会直接替换为常量的实际值。也就是说，常量变为编译后程序中的指令数据的一部分，不会在运行时从某个内存位置读取，因此不能对常量取指针地址。这种处理方式可以减少运行时的内存访问次数，提高执行效率。

命名

对变量或常量命名的有效字符序列称为标识符。

命名规则

不管常量还是变量命名，都要遵循标识符命名规则：

  • 变量名必须以字母或下划线开始；
  • 必须使用英文字母、数字和下划线组成，不能出现空格或制表符；
  • 不能使用内置关键字和保留字；
  • 大小写敏感。

命名规范

变量和函数的命名遵循一套简洁明了的规则和最佳实践：

  • 小驼峰式命名法（lowerCamelCase）：当命名私有（未导出的）变量和函数时，第一个单词以小写字母开始，后续单词的首字母大写。例如：localVariable、myFunction。
  • 大驼峰式命名法（UpperCamelCase）：公共（即在包外可见的，导出的）变量和函数则使用大驼峰式，即所有单词的首字母都大写，也叫做 Pascal 命名法。例如：PublicVariable、ExportedFunction。
  • 短而有意义：Go 语言推荐使用简短但描述性强的命名，能用一个单词命名的，就不使用单词组合。例如，使用 i 作为循环索引，err 表示错误。变量的范围越小，使用的名称越短，只有名称的作用域很大时，才使用更长且更有意义的名称。
  • 避免使用下划线：在 Go 中，一般不使用下划线来分隔单词，这与许多其他编程语言的习惯不同。
  • 避免使用缩写：除非该全大写字母缩写是广泛认可的表示专有名词，例如用 HTTP 代替 HyperTextTransferProtocol。其他情况下不要使用缩写。

重点：小驼峰式用于私有成员，大驼峰式用于公共成员。

赋值

赋值运算符由等号「=」实现，用于将等号右边表达式的值赋予等号左边的变量，也叫赋值语句（Assignment Statement）。常见赋值方式有三种：

  • 定义变量并赋值：a := 10
  • 赋值操作：c = a
  • 赋值运算：c /= a，等同于 c = c / a

赋值运算可以使用任意二元算术运算符。

Go 函数的参数当涉及到目标和来源时，也遵循左侧是被赋值的目标，右侧是来源的顺序，反映出一种直观的「赋值」感觉。对比 Shell 命令的参数顺序，例如：cp source destination，指定从哪里（source）复制到哪里（destination），反映的是操作顺序。

初始化零值

所有变量在 Go 语言中都需要初始化，当变量被声明但没赋值时，会被赋予一个默认值，也被称为零值（zero value）。零值机制保证所有的变量被定义，不存在未初始化的变量。

基本数据类型初始化零值见下表。对于像数组或结构体这样的复合类型，零值是其所有元素或成员的零值：

类型 零值
布尔型 false
整数型 0
浮点型 0
复数型 0+0i
字符串 “”
函数、接口、通道、映射、切片、指针 nil

要检查变量是否被赋值，通常将变量值和零值比较是否相等，某些情况还需检查类型变量指针是否为 nil：

package main

import (
	"fmt"
)

func main() {
	// 声明一个函数类型变量
	var funcVar func(string) int
	fmt.Println(funcVar) // 输出：<nil> 表示它不引用任何函数实现

	// 声明一个接口类型变量
	type MyInterface interface{ DoSomething() }
	var myInterface MyInterface
	fmt.Println(myInterface) // 输出：<nil> 表示它没有包含任何值或具体类型

	// 声明一个通道类型变量
	var ch chan int
	fmt.Println(ch) // 输出：<nil> 表示它未打开，不能用于数据传递

	// 声明一个映射类型变量
	var m map[string]int
	fmt.Println(m) // 输出：map[] 表示它没有被初始化，且不能直接使用

	// 声明一个切片类型变量
	var slice []int
	fmt.Println(slice) // 输出：[] 表示它没有底层数组

	// 声明一个指向整型的指针
	var ptr *int
	fmt.Println(ptr) // 输出：<nil> 表示它不指向任何内存位置

	// 现在显式地赋值
	value := 10
	ptr = &value

	if ptr == nil {
		fmt.Println("ptr 未被赋值。")
	} else {
		fmt.Printf("ptr 被赋值，其值为 %d。\n", *ptr)
	}
}

零值为 nil 的类型作为函数参数时，可以直接传递 nil 值给函数，省掉定义步骤。

并行赋值

并行赋值也称为「多重赋值」或「元组赋值」，是一种同时对多个变量进行赋值的语法。并行赋值的执行类似于原子操作，右侧的所有表达式首先计算完成，然后再统一赋值给左侧的变量：

package main

import "fmt"

// vals 定义一个返回两个整数的函数
func vals() (int, int) {
	return 7, 13
}

func main() {
	a, b := 5, 10
	fmt.Println("原始值:", "a =", a, ", b =", b)

	// 接收两个返回值
	a, b = vals()
	fmt.Println("修改值:", "a =", a, ", b =", b)
}

变量值交换

在实际编程中，可能会用到变量值的交换，比如交换变量 a 和变量 b 的值。在 Go 语言中不需要用到额外的临时变量，可以通过多重赋值的特性来实现：

package main

import (
	"fmt"
)

func main() {
	a := 5
	b := 10

	fmt.Printf("原始值 - a: %d, b: %d\n", a, b)

	// 交换变量a和b的值
	a, b = b, a

	fmt.Printf("交换后 - a: %d, b: %d\n", a, b)
}

只有变量类型相同情况下才能进行变量值交换。

作用域

变量的作用域（Scope）是指变量可以被访问的代码区域，这由变量声明的位置决定。预定义标识符（如内置函数和类型）的作用域覆盖整个包，而非预定义标识符（如变量、常量、类型定义等）的作用域则被限制在声明的代码块内。通过导入包，可以将包级别的标识符的作用域扩展到使用这些包的文件中。

下面是各种级别作用域的示例：

package main

import "fmt"

// 全局变量
var globalVar = "全局变量"

func main() {
	// 局部变量，main 函数作用域
	var localMainVar = "main函数局部变量"

	fmt.Println(globalVar)    // 输出全局变量
	fmt.Println(localMainVar) // 输出 main 函数内的局部变量

	// 调用 anotherFunction，展示不同函数作用域
	anotherFunction()

	// for 循环中的局部变量
	for i := 0; i < 1; i++ {
		var loopVar = "for循环局部变量"
		fmt.Println(loopVar) // 在循环内部输出循环变量
	}
	// fmt.Println(loopVar) // 这里如果尝试访问 loopVar 会报错，因为 loopVar 的作用域仅限于 for 循环内部

	// 使用同名变量
	var shadowVar = "main函数内的shadowVar"
	fmt.Println(shadowVar) // 输出 main 函数内的 shadowVar
	if true {
		// 这个同名变量 shadowVar 在 if 语句的作用域内会隐藏（shadow）外部的同名变量
		var shadowVar = "if语句内的shadowVar"
		fmt.Println(shadowVar) // 输出 if 语句内的 shadowVar
	}
	// if 语句外仍然是外部的 shadowVar
	fmt.Println(shadowVar) // 再次输出 main 函数内的 shadowVar
}

func anotherFunction() {
	// anotherFunction 函数作用域内的局部变量
	var localAnotherVar = "anotherFunction函数局部变量"

	fmt.Println(globalVar)       // 可以访问全局变量
	fmt.Println(localAnotherVar) // 输出 anotherFunction 函数内的局部变量
	// fmt.Println(localMainVar) // 这里如果尝试访问 localMainVar 会报错，因为 localMainVar 的作用域仅限于 main 函数内部
}

只用记住：在内部代码块中，可以访问外部代码块中全局变量。反之，外部代码块中不能访问内部代码块中声明的变量。如果在内层代码块中定义和外部有相同名称的变量，则外部同名变量在内部代码块中会被暂时隐藏。

如果一个包中的全局变量是可导出的，那么在其他地方导入此包，也可以访问包中可导出的全局变量：

/* 
main 包中的代码演示了访问其他包的全局变量。
config.go 文件内容：
package config

// MaxConnections 是一个可导出的全局变量，表示最大连接数。
var MaxConnections int = 100
*/
package main

import (
	"fmt"
	"new/internal/config" // 导入模块路径
)

func main() {
	fmt.Println("Max connections allowed:", config.MaxConnections)

	// 修改 config 包中的 MaxConnections 变量
	config.MaxConnections = 200
	fmt.Println("Updated max connections allowed:", config.MaxConnections)
}

定义未使用

未使用的局部变量，会被编译器当作错误，以此确保所有声明的变量都被有效使用，而全局变量和常量则允许定义后不使用。这样的设计主要出于以下几个原因：

  • 代码质量：声明局部变量表示有计划使用它，否则可能是的代码未完成、垃圾代码，或拼写错误。通过报错来提醒开发者修复潜在的问题。
  • 资源管理：局部变量会占用栈空间，确保不存在无用的局部变量可以更有效地利用资源。
  • 无副作用：全局变量和常量的生命周期与程序的生命周期相同，它们在程序启动时初始化，并在程序结束时销毁，不存在副作用。
  • 跨包依赖：全局变量和常量可能会在不同的包之间传递使用。例如一个配置包定义了一些配置常量，供其他包使用，而定义包本身并不使用这些常量，这属于正常的模块化设计。

常量

常量指在整个程序生命周期内都不变的值，通常是一些字面量。常量声明后可以引用，但不能修改值。

声明格式

声明常量格式为：const 常量名 [类型] = 值或赋值表达式。其中类型声明可省略，编译器可以自动推断值类型：

package main

import (
	"fmt"
	"math"
)

func main() {
	// 显式类型声明
	const pi float64 = 3.14159265358979323846

	// 自动类型推断
	const pi2 = math.Pi * 2

	// 输出常量值
	fmt.Println("Pi:", pi)
	fmt.Println("Pi doubled:", pi2)
}

常量组

常量组是一种语法结构，允许一个连续的代码块中声明多个常量。常量组通过使用 const 关键字后跟一对括号来声明，其中可以定义多个常量。在常量组中，如果一个常量没有显式的初始化表达式，则会默认采用上一个常量的表达式和类型：

package main

import "fmt"

func main() {
	const (
		a, b, c = 100, "abc", '总' // 自动推断出常量的类型为整数、字符串、字符
		u       = "a"
		v       = a * 10
		w       // 继承上个常量 v 的值
	)
	fmt.Println(a, b, c, u, v, w)
}

上面用括号括起来多行书写的方法被称为因式分解关键字，同样适用于变量和类型的声明或定义。

初始化规则

常量赋值表达式中不能使用自定义函数或变量，只能使用内置函数和常量。常量之间的操作（包括算术、逻辑以及比较操作）会在编译时进行计算，结果也是常量：

package main

import (
	"fmt"
	"math"
)

// 自定义函数，不能用于常量的初始化表达式
func multiplyByTwo(n int) int { return n * 2 }

func main() {
	// 使用内置常量和内置函数初始化常量
	const base = 2
	const pi = math.Pi                      // 使用内置的数学常量
	const strLength = float64(len("hello")) // 使用内置函数
	const result = base * pi * strLength    // 常量之间的操作，编译时计算

	fmt.Println("结果:", result)

	//const invalidConst = multiplyByTwo(5) // 试图使用自定义函数将导致编译错误
}

无类型常量

无类型常量（untyped constant）是指在声明时没有显式指定具体类型的常量，这些常量的类型根据在表达式中的上下文动态确定。与常量不同，变量在声明时必须有明确的类型，不能是无类型的。

例如一个无类型的字面量 42，可以赋值给整型或浮点型：

package main

import "fmt"

const x = 42 // 无类型常量

func main() {
    var a int = x        // x 在这里表现为 int
    var b float64 = x    // x 在这里表现为 float64
    var c byte = x       // x 在这里表现为 byte

    fmt.Println(a, b, c) // 输出: 42 42 42
}

虽然常量声明时可以不指定类型，但 Go 语言提供了一些默认类型：

  • 整数常量默认类型是 int
  • 浮点数常量默认类型是 float64
  • 字符串常量和布尔常量分别是 string 和 bool

常量的默认类型会被自动转换为适当的类型：

package main

import "fmt"

const (
	a = 42      // 无类型整数常量
	b = 3.14    // 无类型浮点数常量
	c = "hello" // 无类型字符串常量
	d = true    // 无类型布尔常量
)

func main() {
	fmt.Printf("a: %T, b: %T, c: %T, d: %T\n", a, b, c, d) // 输出: a: int, b: float64, c: string, d: bool
	var af float64 = a                                     // 虽然被默认推断为无类型整数，但依然可以赋值给 float64 类型变量
	fmt.Println(af)
}

此外，无类型常量经常用来表示任意精度的整数或浮点数，只有将常量赋值给一个特定类型的变量时，才会检查和限制它的值是否在该类型的表示范围内。

枚举

Go 语言没有明确意义上的枚举类型（enum），而是通过使用 iota 关键字在常量组中模拟枚举类型。

定义

枚举指一系列相关的常量，如果一个常量只有几种可能的值，则可以将其定义为枚举类型（Enumeration-type）。简单点说，枚举就是将常量的值一一列举出来，常量的值只限于列举值的范围内。

最典型的枚举是一个星期七天，对于星期的枚举就包含 7 个可能的值：

package main

func main() {
	const (
		_ = iota
		Sunday
		Monday
		Tuesday
		Wednesday
		Thursday
		Friday
		Saturday
	)
	println(Sunday, Monday, Tuesday, Wednesday, Thursday, Friday, Saturday) // 输出 1 到 7
}

上面按行递增，可以省略后续的 iota 关键字。此外，iota 也可以用在表达式中：

package main

import "fmt"

func main() {
	const (
		a = iota                  // 初始值为 0
		b = iota + 2              // 因为 iota 递增到 1，所以等于 1+2
		c                         // 和上面一样，等于 iota + 2，所以值为 2+2
		d float64    = iota * 2.1 // iota 值递增到 3，等于 3*2.1
	)
	fmt.Println(a, b, c, d) // 输出：0 3 4 6.3
}

使用规则

iota 的使用规则是：常量组中使用计数器 iota 从 0 开始计数，每定义一个常量，iota 的值自动递增 1；而每遇到一个 const 关键字，iota 就会重置为 0:

package main

import "fmt"

func main() {
	const (
		a = "a"
		b
		c = iota // 尽管前面没有 iota 关键字，但 iota 只管计算常量个数，所以此时值为 2
		d        // 继续递增为 3
	)
	fmt.Println(a, b, c, d) // 输出：a a 2 3
	const (                 // 新的常量组，iota 重新计数
		e = 'e'
		f = iota // 此时为 1
		g
		h = 100
		i        // 此时 i 等于 h，没有 iota 什么事
		j = iota // 显式恢复 iota 自增，计数包括 g、h、i，所以为 5
		k
	)
	fmt.Println(e, f, g, h, i, j, k) // 输出：101 1 2 100 100 5 6
}

在常量组内，iota 在每一行中的值是相同的，而且在每新的一行中自增。即使在同一行中多次引用 iota，其值也是相同的，不会因多次引用而递增：

package main

import "fmt"

func main() {
	const (
		a, b = iota, iota * 10 // 第一行中，两个 iota 均为 0
		c, d                   // 第二行中，两个 iota 均为 1
	)
	fmt.Println(a, b, c, d) // 输出：0 0 1 10
}

使用 iota 允许定义一系列相关的常量而无需显式赋值，减少了重复和出错的机率。

位掩码

位掩码（Bit Masks）是一种使用单个整数值的各个二进制位来表示不同状态或特征的技术，通过位掩码可以有效地存储和操作多个布尔标志。在 Go 语言中，使用 iota 结合位移操作 1 << iota 可以轻松创建位掩码常量：

  • 唯一性：每次位移操作都会创建一个二进制中只有一个位是 1 的数字。这样生成的数可以作为独立的标志位，用于表示不同的状态或选项。
  • 便于组合：由于每个常量都是一个位掩码，可以通过位运算组合和比较，实现多个选项的组合。
  • 效率高：由于每个标志仅占用一个位，使用位掩码可以有效地使用内存，并且位运算速度比其他数学运算更快。

位掩码在 Go 源码中存在大量使用，就像下面代码这样：

package main

import "fmt"

type FilePermissions int

const (
	Read    FilePermissions = 1 << iota // 1 << 0 = 1 二进制 0001
	Write                               // 1 << 1 = 2 二进制 0010
	Execute                             // 1 << 2 = 4 二进制 0100
)

func main() {
	// 设置权限为读和写
	myPermissions := Read | Write
	fmt.Println("Permissions:", myPermissions) // 输出 3

	// 检查是否有写权限
	fmt.Println("Write Permission?", myPermissions&Write != 0) // 返回 true
}

下面是 iota 在储存单位计算上的应用：

package main

import "fmt"

type ByteSize float64

const (
	_           = iota
	KB ByteSize = 1 << (10 * iota)
	MB
	GB
)

func main() {
	fmt.Printf("1 KB = % .f Bytes\n", KB) // 输出：1 KB =  1024 Bytes
	fmt.Printf("1 MB = % .f Bytes\n", MB) // 输出：1 MB =  1048576 Bytes
	fmt.Printf("1 GB = % .f Bytes\n", GB) // 输出：1 GB =  1073741824 Bytes
}

常量 KB 的值被设置为 1 向左移动 10 位，得到 1024。后面依此类推。

变量

变量名是用来引用变量所储存数据的标识符，代表变量在内存中的地址。从变量中取值，就是通过变量名找到相应的内存地址，再从该内存储存单元中读取数据。

变量值除了可使用基本值类型外，还可以使用复合类型、引用类型和一些特殊类型。

声明格式

变量声明有三种格式：

  • 变量完整声明格式为：var 变量名 [变量数据类型] = 值或赋值表达式。数据类型可以省略，由编译器推断。

  • 省略值的声明格式为：var 变量名 变量数据类型。必须指定类型，值会被初始化为其数据类型的零值。

  • 简短变量声明格式为：变量名 := 值或赋值表达式。简短声明同时进行变量声明和赋值。

通常约定在函数内部使用简短声明，在函数外使用显式声明。变量被声明后，会立即分配内存空间：

package main

import "fmt"

// 类型被推断为 string，全局变量可以不被使用，不会引起编译器报错
var globalVar = "This is a global variable"

// 显式指定类型，没有初始化表达式，值初始化为零值
var explicitTypeVar int

// 语法错误：在函数外使用简短声明
//errVar := 1

// 语法错误：重复定义
//var globalVar = "This isn't a global variable"

func main() {
	// 简短变量声明格式只能用于函数内部
	shortVar := 1.1
	fmt.Println(shortVar)        // 输出：1.1
	fmt.Printf("%T", shortVar)   // 自动推断为：float64
	fmt.Println(explicitTypeVar) // 输出：0
}

还有一种情况需要在函数内部使用 var 关键字来定义局部变量，是初始化表达式的类型不精确符合变量应有的类型。比如将一个整数赋值给一个浮点型变量：var x float64 = 3，使用 var 可以在这种情况下显式定义变量的类型，以确保类型的正确性。

全局变量

全局变量（Global Variables）也称为外部变量，它是在函数外部定义的变量，可在包内所有块中访问：

package main

import "fmt"

// 声明但不赋值，值为类型初始化零值
var a string
var b int
var c bool
var d, e int

// 声明变量组
var (
	a1, b1 int
	c1     bool = true
)

// 并行赋值，通常用于 for 循环子语句
var a2, b2, c2 int = 1, 2, 3
var d2, e2 = 2, 3

func main() {
	fmt.Println(a, b, c, d, e)      // 输出： 0 false 0 0
	fmt.Println(a1, b1, c1)         // 输出：0 0 true
	fmt.Println(a2, b2, c2, d2, e2) // 输出：1 2 3 2 3
}

全局变量可以在函数外任意位置定义，但建议统一放在代码顶部，并作说明注释。

局部变量

局部变量（Local Variables）是在函数内部定义的变量，它的作用范围仅限于函数内部。在函数体内，除了使用 var 定义局部变量外，还可以使用 := 赋值运算符来简短变量声明，编译器会自动推断变量类型：

package main

import "fmt"

var globalA = 0 // 全局变量

func main() {
	fmt.Println("打印全局变量原始值 globalA：", globalA) // 输出：0
	changeA()
	fmt.Println("再次打印全局变量 globalA：", globalA) // 输出：11
	fmt.Println("全局变量 globalC 的值：", globalC)  // 输出：3
}

func changeA() {
	// 修改全局变量 globalA 的值
	globalA = 11
	fmt.Println("在函数内修改后的全局变量 globalA：", globalA) // 输出：11

	// 在函数内声明的 globalA 是局部变量
	globalA := 111
	fmt.Println("在函数内重新声明的局部变量 globalA：", globalA) // 输出：111

	// 在函数内声明另一个局部变量 b
	localB := 2
	fmt.Println("在函数内新声明的局部变量 localB：", localB) // 输出：2
}

var globalC = 3 // 另一个全局变量，定义位置不影响调用

匿名变量

由于 Go 语言规定局部变量定义了必须使用，否则编译会报错，但调用函数时经常会遇到多个返回值，其中并非每个返回值都需要用到，这时可将不需要的返回值赋予下划线「_」空标识符（Blank Identifier），也被称为匿名变量，编译时会被忽略。

匿名变量不占用命名空间，也不会分配系统内存，无法读取内容。匿名变量的使用有助于保持代码的简洁性，避免创建无用的临时变量：

package main

import "fmt"

func main() {
	indexNum, _, _ := GetData() // 忽略第 2 个和第 3 个返回值
	fmt.Print(indexNum)
}

func GetData() (int, int, int) { return 1, 0, 0 }

匿名变量 _ 可以在任何赋值操作中使用，不仅限于函数返回值。

重声明

声明变量后，在同一代码块内只能修改变量值，无法对其进行重新声明或改变其类型。但有一种特例情况：当使用简短变量声明方式进行多变量赋值时，允许在至少引入一个新变量的同时，对已在同一作用域内声明过的变量进行重声明。重声明也叫退化赋值，实际上是对这些已声明变量进行赋值操作，而不是重新声明。

这种机制常用于处理函数返回值，特别是连续处理多个函数返回错误值。通过这种方式可以重用 err 变量，而无需为每个错误检查创建新的变量名：

package main

import (
	"fmt"
	"net/http"
	"os"
)

func main() {
	file, err := os.Open("config.ini")
	// 报错内容：open config.ini: The system cannot find the file specified.
	fmt.Println(err)

	response, err := http.Get("http://localhost")
	// 报错内容：Get http://localhost: dial tcp [::1]:80: connectex: An attempt was made to access a socket in a way forbidden by its access permissions.
	fmt.Println(err)

	// 由于出现错误，所以都为空值
	fmt.Println(file, response) // 输出：<nil> <nil>
}
