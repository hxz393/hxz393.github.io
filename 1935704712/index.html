Go 语言类型：指针

发表于 2024-01-19 更新于 2024-07-23 分类于 Go ， 2.类型系统 阅读次数： 本文字数： 3.7k 阅读时长 ≈ 12 分钟

理论基础

指针的定义、Go 语言中指针类型和限制。

指针定义

计算机内存以字节（Byte）作为数据储存单位，每个字节配有唯一的内存地址（Memory Address），专门存放内存地址的数据类型叫指针（Pointer）。

在声明变量时，会在内存中给它分配一个位置，以便能够储存、修改和获取变量的值。取变量实际数据第一个字节地址值为变量的指针。所有变量在程序运行时都有指针，通过变量的指针可以在不直接使用变量名的情况下，间接读取或更新变量的值。当通过指针操作数据时，实际上是在读取或修改指针指向内存地址开始的连续内存区域。

指针类型

Go 语言中的指针 uintptr 是一个无符号整数类型，其大小根据操作系统的架构而不同：

  • 在 32 位架构操作系统上，uintptr 占用 4 个字节（32 位）。表示的地址范围是 0 到 2^32 - 1（约 4 GB），这也是 32 位系统内存不能超过 4 GB 的由来。
  • 在 64 位架构操作系统上，uintptr 占用 8 个字节（64 位）。表示的地址范围扩大到 0 到 2^64 - 1（约 16 EB，1EB = 1024 PB），近乎无上限。

指针限制

Go 语言不支持指针运算，也不支持 -> 运算符，结构体指针访问结构体字段仍然使用「.」点操作符：

  • 不支持指针运算：在一些编程语言中，指针运算允许直接操作内存地址，例如通过增加或减少指针的值来访问数组的元素（内存中连续的地址）。Go 语言中禁止这类操作，减少了因指针运算导致的错误，如数组越界访问、野指针等，从而增加了程序的安全性。

  • 没有 -> 运算符：在一些编程语言中，-> 运算符用于指针访问结构体（或联合体）的成员。Go 语言没有这个运算符，结构体指针访问结构体字段使用「.」点操作符。因此不管是直接通过结构体实例访问其字段，还是通过指向结构体的指针访问，语法是一致的，都使用点操作符。

  • 无法直接操作内存： 直接操作内存允许软件与硬件直接交互，如在嵌入式系统开发中，直接向特定的硬件寄存器写值。Go 语言不能通过指针直接操作内存地址，例如直接赋值内存地址给指针变量：var t *string = "0xc0420dc1c0" 是不被允许的。

尽管 Go 语言在标准用法中不允许直接内存操作，但通过提供的 unsafe 包，允许程序绕过类型安全执行某些不安全的操作，只推荐在绝对必要且对安全性有充分理解的情况下使用。

语法糖

数组和结构体通过指针访问时不需要显式解引用，这是 Go 语言中提供的语法糖（Syntactic sugar）技术。其他类型仍然需要显式解引用来访问指向的数据：

package main

import "fmt"

// Person 为结构体类型
type Person struct {
	Name string
	Age  int
}

func main() {
	// 数组示例
	arr := [3]int{10, 20, 30}
	pArr := &arr  // 获取数组的指针，赋值给指针变量 pArr
	pArr[0] = 100 // 修改数组元素时，无需先解引用
	fmt.Println("通过指针访问数组元素:", pArr[0])

	// 结构体示例
	bob := Person{"Bob", 25}
	pBob := &bob                          // 获取结构体的指针，赋值给指针变量 pBob
	pBob.Age++                            // 通过指针变量直接修改值，实际等同于 (*pBob).Age++
	fmt.Println("通过指针访问结构体字段:", pBob.Age) // 直接通过指针访问结构体字段，无需解引用

	// 基本数据类型示例
	num := 100
	pNum := &num                      // 获取int的指针
	fmt.Println("通过解引用指针访问值:", *pNum) // 必须解引用来访问指针指向的值，否则获取到的是指针地址

	// 修改值
	*pNum = 200                // 修改指针指向的值，必须显式解引用
	fmt.Println("修改后的值:", num) // 输出修改后的值
}

实际应用

Go 语言中指针变量的使用。

变量取址

要获取变量在内存中的指针地址，可在变量名前加上「&」字符：

package main

import "fmt"

func main() {
	sampleString := "111"
	// 使用 fmt 包中的 Printf 函数格式化输出 sampleString 的值和地址
	// %v 代表变量的值，%p 代表变量的指针地址，用十六进制表示，如 0xc0420dc1c0
	fmt.Printf("变量的值：%v，变量在内存中的地址：%p\n", sampleString, &sampleString)
}

不能获取字面量或常量的指针。

指针取值

在 Go 语言中，为表示指针变量和所指向变量之间的联系，在程序中使用「*」星号表示「指向」。要获取指针对应的值内容，可在指针变量名前加上星号来解引用：

package main

import "fmt"

func main() {
	sampleString := "111"
	// 使用 & 符号取变量的地址，将地址赋给 sampleStringPtr，类型为 *string
	sampleStringPtr := &sampleString

	fmt.Println("变量的值：", sampleString)
	// 打印 sampleStringPtr 的值，即 sampleString 的地址
	fmt.Printf("变量的内存地址：%p\n", sampleStringPtr)
	// 使用 * 符号对 sampleStringPtr 进行解引用，获取指针所指向的值，也是 sampleString 的值
	fmt.Println("通过指针获取的变量的值：", *sampleStringPtr)
}

声明指针

共有三种指针声明格式：

  • var 指针变量名 *数据类型：完整指针变量声明格式。
  • 指针变量名 :=new(数据类型) ：使用 new 函数来创建有初始化值的指针变量。
  • 指针变量名 := &变量名：短声明格式，最常使用。

指针变量名一般在变量名前添加前缀「p」、「ptr」或在变量名后添加「Ptr」来表示这是一个指针：

package main

import "fmt"

func main() {
	// 完整声明一个指向 float64 类型的空指针
	var pointerToNull *float64 // 初始化为 nil，表示当前没有指向任何地址
	fmt.Println(pointerToNull) // 打印指针默认值 <nil>

	// 使用 new() 函数创建并初始化一个指针变量
	pointerToInt := new(int)                // 使用 new 创建的指针自动被初始化为指向类型的零值
	fmt.Println(pointerToInt)               // 打印出指针的内存地址：0xc0420e8090
	fmt.Printf("指针变量地址：%p\n", pointerToInt) // 使用 %p 格式化输出指针的内存地址
	fmt.Println("指向的整数值：", *pointerToInt)   // 解引用指针，打印出指向的整数的值 0

	// 使用短变量声明和 & 操作符获取现有变量的地址
	str := "Hello"                         // 定义一个字符串变量
	pointerToStr := &str                   // 获取 str 的内存地址，pointerToStr 是一个指向 str 的指针
	fmt.Println(pointerToStr)              // 打印出字符串变量的内存地址
	fmt.Println("指向的字符串值：", *pointerToStr) // 解引用指针，打印出指向的字符串的值
}

传递指针

Go 语言的函数参数传递可以是值或引用（指针），传递指针能使多个函数操作修改同一个外部变量：

package main

import "fmt"

func main() {
	var num int = 1
	fmt.Println("在 main 函数中，num 的初始值为:", num) // 打印原值：1
	fmt.Println("在 main 函数中，num 的地址为:", &num) // 打印 num 的地址

	incrementValue(num)                                         // 通过值传递，传入变量 num 的副本
	fmt.Println("调用 incrementValue 后，main 函数中 num 的值仍然为:", num) // 函数 incrementValue 中的修改不影响 main 函数中的变量 num

	incrementPointer(&num)                                       // 通过指针传递，传入变量 num 的地址
	fmt.Println("调用 incrementPointer 后，main 函数中 num 的值变为:", num) // 函数 incrementPointer 中的修改影响了 main 函数中的变量 num
}

// incrementValue 通过值传递接收一个 int 类型的参数，并尝试修改它，但结果不会影响到原始变量
func incrementValue(num int) int {
	fmt.Println("在 incrementValue 函数中，num 的地址为:", &num) // 打印变量 num 副本的地址
	num++
	return num
}

// incrementPointer 通过指针传递接收一个指向 int 的指针，并通过这个指针修改原始变量的值
func incrementPointer(numPtr *int) int {
	fmt.Println("在 incrementPointer 函数中，num 的地址为:", numPtr) // 打印与 main 函数中变量 num 的地址相同的值
	*numPtr++                                               // 通过指针直接修改原变量的值
	return *numPtr
}

由于指针地址只占用 4 或 8 个字节，用指针传递大体积数据时，会明显节省系统开销。因此，像切片、映射、接口、通道这样的引用类型默认使用指针传递。

空指针

Go 语言中的空指针为 nil 值，也叫 nil 指针。空指针是一个特殊类型的指针，它不指向任何有效的内存地址，在使用指针之前检查它是否为 nil 可以防止运行时错误。对一个空指针的反向引用会致使程序崩溃：

package main

import "fmt"

func main() {
	var ptr *int // 默认为 nil
	//fmt.Println("ptr 的地址", *ptr) // 试图打印空指针地址时报错：内存地址不正确

	// 为 ptr 分配一个值
	//*ptr = 5 // 试图解引用一个空指针将导致运行时报错：空指针解引用
	num := 5
	ptr = &num // ptr 现在指向 num

	// 检查指针是否为 nil
	if ptr == nil {
		fmt.Println("ptr 是一个空指针")
	} else {
		// 安全地使用 ptr
		fmt.Println("ptr 的值为", *ptr)
		fmt.Println("ptr 的地址为", ptr)
	}
}

使用 nil 指针可以实现延迟初始化的模式，其中资源或对象的实际初始化延迟到真正需要时才进行。

多级指针

多级指针是一个指向指针的指针，它存储的是另一个指针变量的内存地址，可以扩展到多个层级，如二级指针、三级指针等。指针的指针通过在指针类型前再加上星号「*」来表示，多级指针可以用于间接引用或修改一个变量的值：

package main

import "fmt"

// modifyValue 函数接收一个指向 int 指针的指针 (**int)，并修改指针指向的值
func modifyValue(ptr **int) {
	// 修改 ptr 指向的指针所指向的值
	**ptr = 100
}

func main() {
	a := 20
	ptrA := &a       // ptrA 是一个指向 int 的指针 (*int)
	ptrPtrA := &ptrA // ptrPtrA 是一个指向 *int 的指针 (**int)

	// 都输出 20
	fmt.Println("修改前的 a 值 =", a)
	fmt.Println("ptrA 指向的值 =", *ptrA)
	fmt.Println("ptrPtrA 指向的 ptrA 指向的值 =", **ptrPtrA)

	// 调用函数，尝试修改 a 的值
	modifyValue(ptrPtrA)

	// 都输出修改后的值 100
	fmt.Println("\n修改后的 a 值 =", a)
	fmt.Println("ptrA 指向的值 =", *ptrA)
	fmt.Println("ptrPtrA 指向的 ptrA 指向的值 =", **ptrPtrA)
}

多级指针允许程序动态地修改和管理指针数组或复杂数据结构的内存。这对于实现动态扩展或缩减的数据结构，如可变大小的数组、堆栈、队列等，非常有用。

指针数组

一个指针数组是一个数组，其中每个元素都是指向某个类型的值的指针：

package main

import "fmt"

func main() {
	// 初始化一个整型数组
	numbers := [3]int{11, 22, 33}

	// 定义一个指针数组，用于存放整型数组 numbers 的元素地址
	var ptrs [len(numbers)]*int

	// 遍历整型数组，将元素的地址存储到指针数组中，并同时打印元素的值和地址
	for i := 0; i < len(numbers); i++ {
		ptrs[i] = &numbers[i] // 将地址赋值给指针数组
		fmt.Printf("Element %d: Address = %p, Value = %v\n", i, ptrs[i], *ptrs[i])
	}
	fmt.Printf("%v", ptrs) // 输出：[0xc0420e60a0 0xc0420e60a8 0xc0420e60b0]
}

当数组需要存储大型结构或类实例时，使用指针数组可以减少内存占用。通过存储指向大型数据项的指针，而不是数据项本身，可以减少复制的开销。

逃逸分析

逃逸分析（Escape Analysis）是编译器进行的一种分析过程，用于确定变量分配到堆或栈上。如果一个变量在函数外部无法访问，编译器选择在栈上分配该变量的内存，栈上的内存分配和回收开销小，且不需要垃圾回收器介入。相反，如果变量在函数外部仍然可以被访问，那么这个变量就会逃逸到堆上，这样垃圾回收器才能跟踪和回收它。

变量逃逸

一个典型的变量逃逸场景是函数返回局部变量的指针：

package main

import "fmt"

func main() {
	// 获取一个指向整数的指针，该整数是在函数内部定义并返回的
	escapedPointer := getIntegerPointer()          // 将返回的指针赋值给外部变量
	fmt.Printf("外部指针的地址：%p\n", escapedPointer)     // 打印出函数内部变量的地址
	fmt.Printf("通过外部指针访问的值：%d\n", *escapedPointer) // 通过指针访问并打印值
}

// getIntegerPointer 在其局部作用域中创建一个整数变量，并返回一个指向该整数的指针。
func getIntegerPointer() *int {
	localVariable := 11
	fmt.Printf("局部变量的地址：%p\n", &localVariable) // 打印局部变量的指针地址
	return &localVariable                      // 返回局部变量的指针，这里发生了变量逃逸
}

函数返回指向函数内局部变量的指针，该局部变量就会逃逸到堆上，而不是栈上。堆上的对象生命周期不是由作用域控制，而是通过垃圾回收管理。垃圾回收器定期运行，查找不再被任何指针引用的对象。一旦对象确定为不可达，垃圾回收器将回收这些对象所占用的内存。因此，任何堆上的数据，只要能被追踪到栈上的一个指针类型变量，则不会被消除。

在实际开发中，通常需要避免不必要的变量逃逸，因为会导致额外的内存分配和性能开销。

栈内存重用

逃逸分析使得栈内存重用成为可能：

package main

import "fmt"

func main() {
	ptr := returnLocalVarAddr()
	fmt.Println("Before calling another function:", *ptr)
	callAnotherFunc()
	fmt.Println("After calling another function:", *ptr)
}

func returnLocalVarAddr() *int {
	a := 10
	return &a
}

func callAnotherFunc() {
	b := 20
	fmt.Println("In another function, using stack memory:", b)
}

理论上，当 callAnotherFunc 被调用时，它的局部变量 b 可能会被分配到与 a 相同的栈内存位置。这样，通过 ptr 访问的值可能会意外地变为 20，而不是原来的 10。正是由于 Go 的逃逸分析，避免了直接的栈内存重用，返回的指针指向的是安全分配在堆上的内存。

检测逃逸分析

可以通过在 go build 命令中添加 -gcflags='-m -l' 参数来检查逃逸分析的结果：

PS D:\Software\Programming\Go\new> go build -gcflags='-m -l'
# new
./main.go:14:2: moved to heap: localVariable
./main.go:15:12: ... argument does not escape
./main.go:8:12: ... argument does not escape
./main.go:9:12: ... argument does not escape
./main.go:9:54: *escapedPointer escapes to heap

上面的结果显示，变量 localVariable 逃逸到了堆上。

额外标志参数 -m 用于输出编译器的优化细节，而 -l 用于禁用函数的内联优化，反之逃逸被编译器通过内联彻底抹除。
