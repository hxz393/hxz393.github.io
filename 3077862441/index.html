Go 语言类型：结构体

发表于 2024-03-08 更新于 2024-07-23 分类于 Go ， 2.类型系统 阅读次数： 本文字数： 2.6k 阅读时长 ≈ 9 分钟

结构体类型

结构体（Struct）是由一系列相同或不同类型的数据构成的数据集合，用来实现较复杂的数据结构。通过结构体，可以在单个变量中储存众多不同类型的数据，以方便操作。

结构体是复合类型，也是值类型。

定义结构体

组成结构体类型的数据称为字段（Field），每个字段都有命名和类型：

type 结构体名 struct {
	字段名1 字段名1类型
	字段名2 字段名2类型
	...
}

结构体的组成元素：

  • 关键字：由于结构体属于新的数据类型，因此必须使用 type 和 struct 关键字来定义。

  • 结构体名：遵循标识符命名规则。结构体和字段是否能被导出，同样由标识符首字母是否大写决定。

  • 字段名：在一个结构体中，字段名也可以使用空标识符。除了 _ ，字段名字必须唯一。

  • 字段类型：结构体字段可以是任意类型，例如基本类型、函数、接口甚至结构体。

此外，对于简单的结构体，可以使用缩写：type 名字 struct { 字段名1, 字段名2 类型 }，同类型的字段名可以写在同一行。

初始化结构体

结构体是值类型，创建结构体变量时会分配内存，返回实体，因此可通过 new 函数来创建实例并返回指针地址。在访问对象字段时，用点号「.」作为操作符，也叫做选择器（selector）：

package main

import "fmt"

type Person struct {
	Name string
	Age  int
}

func main() {
	// 使用 new() 函数初始化一个 Person 类型的指针变量
	p1 := new(Person) // 等同于 p1 := &Person{}
	// 字段赋值方式
	p1.Name = "Alice"

	// 使用字面量形式初始化一个 Person 类型的值变量
	p2 := Person{Name: "bob"} // 可以只赋值部分字段，其他字段会自动初始化零值
	p2 = Person{"Bob", 11}    // 也可以省略字段名，依次赋值。不能和键值对赋值混用，必须全部字段赋值。不推荐这样，一旦结构变化赋值会对不上

	// 使用标准定义方式初始化一个 Person 类型的值变量
	var p3 Person
	p3.Name = "Unknown" // 赋值
	p3.Name = "Charlie" // 修改值
	// 并行赋值方式
	p1.Age, p3.Age = 20, 40

	fmt.Println("p1:", p1.Name, p1.Age) // 输出：p1: Alice 20
	fmt.Println("p2:", p2.Name, p2.Age) // 输出：p2: Bob 11
	fmt.Printf("%#+v", p3)              // 输出：main.Person{Name:"Charlie", Age:40}
}

上面的变量 p1 被称为类型 Person 的一个实例（instance）或对象（object）。

匿名结构体

可以不事先声明结构体，而直接使用嵌入式结构定义结构体变量：

package main

import "fmt"

func main() {
	// 可以直接使用匿名结构体定义变量
	var regUser = struct {
		Id   int
		Name string
	}{Name: "admin"}
	// 输出：{Id:0 Name:admin}
	fmt.Printf("%+v\n", regUser)
	// new() 函数也可以使用匿名结构体
	var anonymousUser = new(struct {
		Id   int
		Name string
	})
	fmt.Printf("%+v\n", anonymousUser)
}

匿名结构体没有类型名称，无需通过 type 关键字定义就可以直接使用。当一个结构体只在很小的作用域内部使用时，比如函数内部或者实现一个很短的接口，使用匿名结构体可以使代码更加简洁，避免了全局命名空间的污染。

匿名字段

结构体可包含一个或多个匿名字段，这些字段没有命名，只指定了类型，每种数据类型只能有一个匿名字段。匿名字段常用于嵌入其他结构体或接口：

package main

import "fmt"

type Person struct {
	Name string
	Age  int
}

type Employee struct {
	Person // 匿名字段
	Salary int
}

func main() {
	e := Employee{ // 对字段的声明顺序没有要求
		Salary: 5000,
		Person: Person{
			Name: "John",
			Age:  30,
		},
	}

	// 直接访问匿名字段的属性
	fmt.Println(e.Name)   // 输出 John
	fmt.Println(e.Age)    // 输出 30
	fmt.Println(e.Salary) // 输出 5000

	// 直接修改匿名字段的属性
	e.Age = 31
	fmt.Println(e.Age) // 输出 31
}

在内嵌结构体和外层结构体中有相同字段名时，外层的字段优先使用：

package main

import "fmt"

type Person struct {
	Name string // Person 有一个 Name 字段，表示真名
}

type Employee struct {
	Person        // 内嵌 Person 结构体
	Name   string // Employee 也有一个 Name 字段，表示代号
	ID     int
}

func main() {
	e := Employee{
		Person: Person{
			Name: "Alice",
		},
		Name: "Chief",
		ID:   001,
	}

	// 访问 Employee 中的 Name 字段
	fmt.Println("Employee Name:", e.Name) // 输出：Chief

	// 访问内嵌的 Person 中的 Name 字段
	fmt.Println("Person Name:", e.Person.Name) // 输出：Alice
}

嵌套结构体

Go 语言中结构体可以嵌套结构体，以建立复杂的数据结构：

package main

import "fmt"

// BasicColor 结构体表示颜色的 RGB 值
type BasicColor struct {
	Red, Green, Blue int
}

// AdvancedColor 结构体内嵌 BasicColor，并添加透明度（Alpha）
type AdvancedColor struct {
	BasicColor // 匿名字段，直接使用类型名称
	Alpha      float32
}

func main() {
	// 实例化一个 AdvancedColor 对象
	var color AdvancedColor
	// 分别为红绿蓝三色赋值
	color.BasicColor.Red = 255 // 传统的层级访问法
	color.Green = 200 // 可以直接通过内层结构体字段名访问
	color.Blue = 100
	// 透明度设为 0.5
	color.Alpha = 0.5

	// 直接实例并初始化并一个 AdvancedColor 对象
	color2 := AdvancedColor{
		BasicColor: BasicColor{ // 匿名字段名就是类型名
			Green: 200,
			Blue:  255,
		}, // 每行赋值结尾必须带逗号
		Alpha: 0.6,
	}
	// 可以单独对内嵌结构体字段进行赋值
	color2.Red = 255

	// 可以单独对内嵌结构体实例化，再引用赋值
	bc := BasicColor{Red: 255}
	color3 := AdvancedColor{
		BasicColor: bc,
		Alpha:      0.89,
	}

	// 输出完整的 AdvancedColor 对象信息
	fmt.Printf("%+v\n", color) // 输出：{BasicColor:{Red:255 Green:200 Blue:100} Alpha:0.5}

	// 访问和输出内嵌的 BasicColor 字段，可不带内层类型字段名
	fmt.Printf("Color RGB: (%d, %d, %d)\n", color2.Red, color2.Green, color2.BasicColor.Blue) // 输出：Color RGB: (255, 200, 255)

	// 输出透明度，格式化输出保留一位小数
	fmt.Printf("Alpha: %.1f\n", color3.Alpha) // 输出：Alpha: 0.9
}

递归结构体

结构体可以通过引用自身来定义，这在定义链表或二叉树的元素（也叫节点）时特别有用，此时节点包含指向临近节点的链接（地址）：

type Node struct {
	data float64 // 存放有效数据
	su   *Node   // 指针指向后继节点
}

链表中的第一个元素叫做 head，它指向第二个元素；最后一个元素叫做 tail，它没有后继元素，值为 nil。

同样可以定义一个双向链表，增加一个前趋节点：

type Node struct {
	pr   *Node   // 指向前趋节点
	data float64 // 存放有效数据
	su   *Node   // 指针指向后继节点
}

二叉树中每个节点最多能链接至两个节点：左节点（le）和右节点（ri），这两个节点本身又可以有左右节点，以此类推。树的顶层节点叫做根节点（root），底层没有子节点的节点叫做叶子节点（leaves），叶子节点的左右节点指针值为 nil：

type Tree struct {
	le   *Tree   // 指向左节点
	data float64 // 存放有效数据
	ri   *Tree   // 指针右节点
}

比较结构体

对于类型相同的结构体实例，可以使用相等性运算符来进行比较：

package main

import "fmt"

type Person struct {
	Name string
	Age  int
}

func main() {
	// 初始化两个 Person 类型的变量 p1 和 p2
	p1 := Person{Name: "Alice", Age: 20}
	p2 := Person{Name: "Alice", Age: 20}

	compare(p1, p2) // 比较两个结构体变量
	p2.Age = 30     // 修改 p2 的 Age 字段，使其与 p1 不同
	compare(p1, p2) // 再次比较 p1 和 p2 是否相等
	p2 = p1         // 结构体相同的变量，可以用等号直接复制赋值
	compare(p1, p2)

}

// 使用 == 运算符比较 p1 和 p2 是否相等
func compare(p1, p2 Person) {
	if p1 == p2 {
		fmt.Println("p1 and p2 are equal")
	} else {
		fmt.Println("p1 and p2 are not equal")
	}
}

如果结构体类型不同，编译会报错。即使有完全相同字段但命名不同的结构体类型，也无法进行比较。比较之前可以使用 reflect.TypeOf 函数来检查。

转换结构体

如果两个结构体具有相同的字段名、字段类型和字段顺序，那么它们之间可以进行显式转换：

package main

import "fmt"

type ProductA struct {
	Name  string
	Price float64
}

type ProductB struct {
	Name  string
	Price float64
}

func main() {
	// 初始化 ProductA 类型的变量
	a := ProductA{Name: "Laptop", Price: 1500.00}

	// 转换为 ProductB 类型
	b := ProductB(a) // 显式转换，因为字段和类型相匹配

	fmt.Println(b) // 输出：{Laptop 1500}
}

传递结构体

将结构体类型作为参数传递给函数时，可以选择值传递或指针传递两种方式：

package main

import "fmt"

// Rectangle 为记录正方形参数的结构体
type Rectangle struct {
	Width, Height int
}

// 使用值传递的函数，函数内对副本的修改不会影响原始数据
func enlargeByValue(rect Rectangle, factor int) Rectangle {
	rect.Width *= factor
	rect.Height *= factor
	return rect
}

// 使用指针传递的函数，函数内的修改会直接影响到原始数据
func enlargeByPointer(rect *Rectangle, factor int) {
	rect.Width *= factor
	rect.Height *= factor
}

func main() {
	rect := Rectangle{Width: 10, Height: 5}

	// 调用值传递函数
	newRect := enlargeByValue(rect, 2)
	fmt.Println("After enlargeByValue:", rect)      // 原始对象不变
	fmt.Println("Returned new Rectangle:", newRect) // 返回新对象

	// 调用指针传递函数
	enlargeByPointer(&rect, 2)
	fmt.Println("After enlargeByPointer:", rect) // 原始对象被修改
}

结构体标签

结构体字段还有一个可选的附属于字段的标签（tag），属于类型组成部分。标签内容用反引号「`」括起来，用于为数据交换格式（json、xml、yaml）提供序列化、反序列化、验证等信息。

标签的格式通常是一个或多个键值对，每个键值对以空格分隔，键和值之间用冒号分隔。键值对之间用分号分隔，例如：

package main

import (
	"fmt"
	"reflect"
)

type Person struct {
	Name string `json:"name" validate:"required"`
	Age  int    `json:"age" validate:"min=18"`
}

func main() {
	bob := Person{"bob", 11}
    // 通过 Field 来索引结构体字段，获取 Tag 属性。输出：json:"name" validate:"required"
	fmt.Println(reflect.TypeOf(bob).Field(0).Tag) 
}

在这个例子中，Name 字段的标签包含两个键值对：

  • json:"name" 用于指定在将结构体序列化为 JSON 格式时，使用 name 作为该字段的键名。
  • validate:"required" 用于指定在验证结构体时，要求 Name 字段是必填项。

同样，Age 字段的 tag 也包含两个键值对：

  • json:"age" 用于指定在将结构体序列化为 JSON 格式时，使用 age 作为该字段的键名。
  • validate:"min=18" 用于指定在验证结构体时，要求 Age 字段的值不小于 18。
