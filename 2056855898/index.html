Go 语言代码测试

发表于 2024-05-31 更新于 2024-07-23 分类于 Go ， 4.基本应用 阅读次数： 本文字数： 1.2k 阅读时长 ≈ 4 分钟

测试类型

常用的测试分为单元测试、功能测试和集成测试：

  • 单元测试：又称为模块测试，针对一小部分代码，独立对其进行测试。通常测试对象是单个函数，测试其输入是否复合期望输出。
  • 功能测试：又被称为端到端测试或外向内的测试，测试从用户角度核查程序是否按期望那样工作。
  • 集成测试：测试应用程序各部分协同工作的情况。

基本测试

为了写成良好的测试代码，需要遵守一些设计约定：

  • 文件名：测试代码与被测试代码放在一起，并以要测试文件的名称加上后缀 _test 作为文件命名。例如，要测试的文件名为 strings.go，则相应的测试代码文件名为 strings_test.go。
  • 函数名：测试函数以 Test 开头命名。
  • 变量名：以 got 表示要测试的值，以 want 表示期望的值。

下面是一个简单的函数：

/*
Package hello 包函数文件名为 world.go
*/
package hello

import "fmt"

// World 函数的说明
func World(s string) string {
	result := "hello" + s
	fmt.Println(result)
	return result
}

对应的测试代码和被测试代码属于同一个包，可以方便访问未导出的类型或函数，但非硬性规定：

/*
Package hello 包测试文件名为 world_test.go
*/
package hello

import "testing" // 必须导入 testing 包

func TestWorld(t *testing.T) { // 接受单个参数 *testing.T
	got := World("words")
	want := "hello words"
	if got != want {
        // 使用 t 的方法来报告失败，如果没有 Error 方法被调用，则测试通过
        // 此外 t 还有 Fail、Skip、Log 等方法，用于失败时不同的后续动作
		t.Errorf("结果： %q, 期望： %q", got, want) 
	}
}

使用go test 命令运行测试，会找到目录下所有 _test.go 结尾的文件，并运行名字以 Test 开头的每个函数：

PS D:\Software\Programming\Go\new> go test .\internal\hello\
hellowords
--- FAIL: TestWorld (0.00s)
    world_test.go:12: 结果： "hellowords", 期望： "hello words"
FAIL
FAIL    new/internal/hello      0.664s
FAIL

如果想列出更详细的信息，比如通过的测试，可以在 go test 命令中加入 -v 参数。此外还能使用 -run 参数指定要单独运行的测试函数。

表格驱动

表格驱动用于同时测试多个条件，可以将测试参数、结果和期望值放入到一个结构体中：

package hello

import "testing"

func TestWorld(t *testing.T) {
	type worldTest struct {
		name string
		got  string
		want string
	}

	var worldTests = []worldTest{
		{"hello", "", "hellohello"},
		{"word", "", "hello word"},
		{"mama", "", "hellomama"},
	}

	for _, test := range worldTests {
		test.got = World(test.name)
		if test.got != test.want {
			t.Errorf("测试不通过。参数：%q，返回结果：%q, 期望值：%q", test.name, test.got, test.want)
		} else {
			t.Logf("测试通过。参数：%q", test.name)
		}
	}
}

基准测试

testing 包含一个基准测试框架，用于性能测试。基准测试函数名以 Benchmark 开头：

package hello

import (
	"testing"
)

// 示例函数，正常是写在另外一个包文件中
func Sum(nums []int) int {
	total := 0
	for _, num := range nums {
		total += num
	}
	return total
}

// 性能测试函数
func BenchmarkSum(b *testing.B) {
	// 创建一个足够大的测试数据
	nums := make([]int, 1000)
	for i := 0; i < 1000; i++ {
		nums[i] = i
	}

	// 重置定时器，避免准备数据的时间影响测试结果
	b.ResetTimer()

	// 执行性能测试。b.N 表示循环次数，会自动调整
	for i := 0; i < b.N; i++ {
		Sum(nums)
	}

	// 停止计时器，测试代码放重置和停止计时器之间
	b.StopTimer()
}

在命令提示符中使用 -bench= 参数来运行：

D:\Software\Programming\Go\new\internal\hello>go test -bench=.
goos: windows
goarch: amd64
BenchmarkSum-64          5000000               341 ns/op
PASS
ok      _/D_/Software/Programming/Go/new/internal/hello 2.731s

结果表示总共运行了 5000000 次，每次耗时 341 纳秒，总共运行时长 2.731 秒。

另外还能加入 -benchmem 参数来输出每次操作内存分配信息（字节数）和从堆上分配内存的次数。

测试覆盖率

测试覆盖率是代码测试详尽的指标，指测试执行了代码的百分比。在测试命令加入 -cover 参数来展示测试覆盖率：

D:\Software\Programming\Go\new\internal\hello>go test -bench=. -benchmem -cover
goos: windows
goarch: amd64
BenchmarkSum-64           200000              6456 ns/op               0 B/op          0 allocs/op
PASS
coverage: 57.1% of statements
ok      _/D_/Software/Programming/Go/new/internal/hello 2.023s

上面结果显示，测试只覆盖了 57.1% 的代码。一般通过标准为 80%。

在实际开发中，可能要将测试覆盖率导出成报告分析。可以使用 -coverprofile 来生成报告，并使用 go tool cover 转为 HTML 格式展示：

PS D:\Software\Programming\Go\new\internal\hello> go test -v -coverprofile="c.out"       
=== RUN   TestWorld
hellowords
    world_test.go:14: 结果： "hellowords", 期望： "hello words"
--- FAIL: TestWorld (0.00s)
FAIL
coverage: 75.0% of statements
exit status 1
FAIL    new/internal/hello      0.709s
PS D:\Software\Programming\Go\new\internal\hello> go tool cover -html="c.out" -o="c.html"
