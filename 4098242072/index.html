Go 命令行基本命令

发表于 2024-01-14 更新于 2024-07-23 分类于 Go ， 1.入门基础 阅读次数： 本文字数： 2.5k 阅读时长 ≈ 8 分钟

命令列表

go 命令是用于管理 Go 语言源代码的工具。单独输入 go 时，可以得到关于命令使用的帮助信息。命令的格式为 go 命令 [参数]。以下是一些常见的命令：

  • build：编译源代码包及其依赖。对于 main 包将生成一个可执行文件。
  • clean：删除源码包中编译生成的文件。
  • doc：显示 Go 包的文档。可以支持第三方包，例如：go doc -all github.com/mohae/deepcopy，将展示 deepcopy 的完整文档。
  • env：打印 Go 语言环境变量的信息。
  • fix：将在旧版本的 Go 语言中编写的代码升级至新版本。
  • fmt：格式化源代码包中的代码。虽然许多 IDE 都提供了代码自动格式化的功能，但它在持续集成（CI）流程中仍需要手动配置。
  • get：下载并安装指定的包到本地，支持 GitHub 地址。从 Go 1.11 版本开始，默认启用模块模式（GO111MODULE=on），主要用于添加依赖到当前模块并更新 go.mod 文件，以及下载并安装特定的包或命令。
  • install：编译并安装指定的包，生成的可执行文件位于 bin 目录下，包文件位于 pkg 目录下。从 Go 1.16 版本开始，更推荐使用 go install <some/package>@version 来安装可执行文件，避免修改项目的 go.mod 文件。例如，使用 go install example.com/cmd@latest 会安装 example.com/cmd 的最新版本。
  • list：列出当前安装的包。
  • mod：从 Go 1.11 版本开始，go mod 命令为模块管理提供了多个子命令，如 go mod tidy、go mod download 等。
  • run：编译并运行程序。它会构建临时可执行文件，并输出程序执行的结果。
  • test：测试源码包。自动识别源码目录下以 _test.go 结尾的文件，并执行测试，展示结果。
  • tool：运行指定的 go 工具。
  • version：显示 Go 语言的版本信息。
  • vet：检查源码包中可能存在的错误。

这些命令的可执行文件实际上存放在 $GOROOT/pkg/tool/<platform> 目录下。可以通过使用 go help [命令] 来查看每个命令的详细用法。

编译代码

如果不带任何参数运行 go build 命令，那么将编译当前目录下的代码包。生成的文件名将根据包名来确定。例如，在 src 目录下执行该命令会生成名为 src.exe 的可执行文件：

go build

对于命令源代码文件，可以指定单个文件进行编译。生成的可执行文件将与源码文件同名，并且会被创建在源码文件所在的同级目录中：

go build hello.go

可选参数

go build 命令带有一些可选参数，常用的如下：

  • -a：强制重新编译所有相关的 Go 语言源代码包，包括标准库中的包。
  • -n：打印将要执行的命令，但不实际执行它们。
  • -o：允许用户指定输出的可执行文件的名称和路径。
  • -p：设置在构建或测试时并行运行的程序数量，默认值是 CPU 的线程数。
  • -race：启用数据竞争检测，常用于并发编程。
  • -v：打印出编译过程中的详细信息。
  • -work：打印编译时使用的临时工作空间的路径，并在编译完成后保留它。

交叉编译

默认情况下，编译生成的文件是为编译所在的系统平台准备的。可以通过设置环境变量 $GOOS 和 $GOARCH 来改变目标操作系统和处理器架构，以实现交叉编译：

set GOOS=linux
set GOARCH=amd64
go build -o hello_linux_amd64 main.go

上面在 Windows 下的命令在源码文件的同级目录中生成了名为 hello_linux_amd64 的可执行文件，该文件可以直接在 Linux 64 位操作系统上运行。

压缩体积

通过指定一些编译标志，可压缩编译得到的二进制文件大小：

go build -ldflags="-s -w" main.go

此时生成的文件比原先小了三分之一体积。要进一步压缩生成物大小，可以使用 upx 压缩。下载解压后将路径添加到系统变量：

setx PATH "%PATH%;D:\Software\Program\Upx"

跳转到项目目录下，对文件进行进一步处理：

D:\Software\Programming\Go\new>upx -9 main.exe
                       Ultimate Packer for eXecutables
                          Copyright (C) 1996 - 2024
UPX 4.2.4       Markus Oberhumer, Laszlo Molnar & John Reiser    May 9th 2024

        File size         Ratio      Format      Name
   --------------------   ------   -----------   -----------
   1338368 ->    539648   40.32%    win64/pe     main.exe                                                                                                                                                                                Packed 1 file.

可以看到文件体积被进一步压缩到 40%，最终体积从 2 MB 压缩到 0.5 MB 大小。

运行代码

除了在编译代码后，执行生成的可执行文件外，还可使用 go run 命令编译并执行源码，但不保留可执行文件：

go run hello.go

其中 hello.go 是包含 main 函数的 Go 源文件。也可以在命令中传入参数：

go run hello.go arg1 arg2

加入 -work 参数可以显示临时工作目录路径，并在命令完成后保留它：

PS D:\Software\Programming\Go\new> go run -work main.go 1194
WORK=C:\Users\ADMINI~1\AppData\Local\Temp\go-build3687578180
false
false

go run 常用于快速测试代码，而无需手动执行编译和运行可执行文件的繁琐步骤。

安装工具

go install 命令用于编译并安装 Go 程序和库。通过编译指定的源代码包，生成可执行文件或库文件，并将这些文件安装到 Go 的 bin 或 pkg 目录下。

从 Go 1.11 版本开始，随着模块功能的引入，推荐在模块内部使用 go install 命令。此时 go install 主要用来安装工具，命令格式为：go install some/package@version ，可执行文件会被安装到 $GOPATH/bin 或 $GOBIN 目录中。

例如使用 go install 命令来安装 godoc：

go install golang.org/x/tools/cmd/godoc@latest

安装完成后，可以通过命令行直接运行 godoc 命令。

安装模块

使用 go get 命令可以方便地从网上拉取各种代码包，支持各种储存库：

go get gopl.io/ch1/helloworld

下载的代码包保存在 $GOPATH 环境变量中第一个路径的 src/gopl.io 目录下。除了 Git，go get 还支持 Mercurial 和 Subversion 等版本控制系统。

从 Go 1.11 版本引入 Go 模块（module）开始，$GOPATH 模式开始逐渐被模块模式取代。在模块模式中，依赖包会被下载到模块缓存中，而非直接放在 $GOPATH/src 下。

从 Go 1.17 版本开始，go get 命令用于添加和更新模块依赖，不再推荐用于安装工具。Go 模块允许在任何目录下工作，不再强制要求代码必须在 GOPATH 目录结构中。此时 go get 会自动更新项目依赖跟踪 go.mod 文件。

go get 命令也支持一些选项和参数：

  • -d：只下载不安装。
  • -fix：下载代码包后，先执行 fix 命令来修复代码的兼容性问题，然后再编译安装。
  • -insecure：允许使用 HTTP 协议下载。
  • -t：同时下载安装指定代码包测试源码文件中的依赖。
  • -u：更新已有的代码包及其依赖包。
  • -v：打印详细的操作信息。

如果要升级模块需要注意，一般不同主版本号使用不同的模块路径，例如 gopl.io/ch1/helloworld/v2。如果是小版本升级，则直接使用 go get -u 即可。

清理缓存

为了清除当前源码包和关联源码包中编译生成的文件，可以使用 go clean 命令来快速完成清理。主要用于移除不需要的文件，如可执行文件和编译生成的中间文件（.a、.o 等）。

删除当前目录缓存文件：

go clean

清理所有的缓存：

go clean -cache -modcache -testcache

这将清理 Go 命令使用的所有缓存，包括测试和模块缓存。

格式化代码

虽然在编译时，Go 语言编译器会自动格式化所有源代码到标准格式，但也可以通过 go fmt 工具手动格式化当前目录及其子目录中的所有 Go 文件：

go fmt ./...

go fmt 实际上是在内部调用 gofmt 工具的 -l -w 选项来对指定的文件进行格式化。如果使用 gofmt 工具直接格式化代码，命令类似于：gofmt -w hello.go。命令支持的参数包括：

  • -w：直接覆盖源文件以保存格式化的结果，而不是将其输出到标准输出。
  • -r：应用自定义的重写规则，例如：gofmt -r "(a) -> a" -w *.go。
  • -s：尝试简化代码中的构造。
  • -d：显示格式化前后的差异，而不会写入文件。
  • -e：打印所有的语法错误，即使在标准输出中也能看到。

导入维护

格式化代码命令无法对源文件中导入的包进行检查，可以通过 goimports 工具来自动维护导入包列表：

go install golang.org/x/tools/cmd/goimports@latest

goimports 的用法和 gofmt 类似，可以直接套用参数。例如直接检查并修改源文件：

goimports -w main.go

代码审查

一般 IDE 中都带了代码分析与审查功能，否则可以尝试使用 golint 来检查代码规范。需要通过 go install 来安装：

go install golang.org/x/lint/golint@latest

一般而言，源代码会下载到 pkg\mod\golang.org\x\lint@v0.0.0 目录，可执行文件 golint.exe 生成到 go\bin 目录。

安装好之后通过 golint [文件名] 来对具体的文件或整个项目执行代码审查：

D:\Software\Programming\Go\new>golint
main.go:11:2: don't use underscores in Go names; var replace_str should be replaceStr

golint 只是检查语法，如果是其他类型错误，可以尝试用 go vet 命令检查：

D:\Software\Programming\Go\new>golint

D:\Software\Programming\Go\new>go vet
# new
# [new]
vet.exe: .\main.go:12:2: newstr declared and not used
