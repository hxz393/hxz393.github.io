当您的应用程序中有多个组件都需要访问相同的数据（如 lang_dict）时，合理的传递方式应该是既减少耦合又方便管理。在这种情况下，可以考虑以下几种方法：

1. 单例模式

如果 lang_dict 在整个应用程序中是共享且唯一的，您可以考虑使用单例模式来管理它。这种方式确保了整个应用程序中 lang_dict 只有一个实例，且易于访问。但请注意，单例模式可能会增加代码的复杂性，并在一定程度上违反了面向对象编程的原则。

2. 全局变量

虽然通常不推荐使用全局变量，但如果 lang_dict 在应用程序启动时就已确定，并且在整个应用程序生命周期中保持不变，那么将其作为全局变量可能是一种简单有效的方法。

3. 服务类或配置类

创建一个专门的服务类或配置类来管理 lang_dict 和其他类似的应用程序级别配置。这个类可以作为一个中心点，负责加载、更新和提供配置数据。其他组件可以通过依赖注入或直接请求这个服务类来获取所需的配置。

4. 信号和槽机制

在 PyQt 应用程序中，您可以使用信号和槽机制进行跨组件的通信。您可以在一个中心组件中发出一个信号，携带 lang_dict 数据，并在其他需要这些数据的组件中连接到这个信号。

5. 传递参数（构造器注入）

正如前面所述，您可以在初始化组件时将 lang_dict 作为参数传递。这种方法简单直接，但当组件数量较多时，可能会导致构造函数参数过多，难以管理。

推荐方法

考虑到易用性和减少耦合，服务类或配置类 方法可能是最佳选择。这样，您只需在应用程序启动时初始化这个类，并在需要的地方请求所需的配置数据。这种方式既保持了代码的清晰结构，又易于维护和扩展。同时，它也提供了一定程度的灵活性，比如在未来需要更改配置管理方式时，您只需修改这个服务类，而不需要触及使用这些配置的各个组件。

好的，我将为您提供使用不同方法传递 lang_dict 的代码示例。请注意，这些代码仅作为示例，您可能需要根据您的应用程序的具体需求进行调整。

1. 单例模式

class LangSingleton:
    _instance = None
    lang_dict = {}

    @classmethod
    def get_instance(cls):
        if cls._instance is None:
            cls._instance = LangSingleton()
        return cls._instance

    @classmethod
    def set_lang_dict(cls, lang_data):
        cls.get_instance().lang_dict = lang_data

    @classmethod
    def get_lang_dict(cls):
        return cls.get_instance().lang_dict

# 使用示例
LangSingleton.set_lang_dict(your_lang_data)
lang_dict = LangSingleton.get_lang_dict()

2. 全局变量

# 全局变量定义
GLOBAL_LANG_DICT = {}

# 在程序启动时初始化
GLOBAL_LANG_DICT = load_lang_data()

# 在需要的地方使用
lang_dict = GLOBAL_LANG_DICT

3. 服务类或配置类

class LangService:
    def __init__(self):
        self.lang_dict = {}

    def load_lang_data(self, lang_data):
        self.lang_dict = lang_data

    def get_lang_data(self):
        return self.lang_dict

# 初始化服务并在应用程序中使用
lang_service = LangService()
lang_service.load_lang_data(your_lang_data)
lang_dict = lang_service.get_lang_data()

4. 信号和槽机制

from PyQt5.QtCore import QObject, pyqtSignal

class LangManager(QObject):
    lang_updated = pyqtSignal(dict)

    def update_lang(self, lang_data):
        # 更新语言字典并发出信号
        self.lang_updated.emit(lang_data)

# 在其他组件中连接信号

class YourComponent:
    def __init__(self, lang_dict):
        self.lang_dict = lang_dict
        # 其他初始化代码

# 在创建组件实例时传递 lang_dict
your_component = YourComponent(your_lang_data)

这些示例展示了如何使用不同的方法在您的应用程序中管理和传递 lang_dict。每种方法都有其适用场景和优缺点，您可以根据您的实际需求选择最合适的方法。

服务类（方法3）

使用服务类可以很好地管理和更新语言设置，但要使应用程序中的其他部分响应这些更改，您需要实现一些额外的机制。一种方法是在服务类中定义一个更新语言字典的方法，并在设置更改时调用它。但是，这本身并不会自动通知其他组件语言已更改。为此，您可以结合使用观察者模式，让其他组件订阅语言更改事件，并在事件发生时更新他们的显示。

class LangService:
    def __init__(self):
        self.lang_dict = {}
        self.observers = []

    def load_lang_data(self, lang_data):
        self.lang_dict = lang_data
        self.notify_observers()

    def get_lang_data(self):
        return self.lang_dict

    def register_observer(self, observer):
        self.observers.append(observer)

    def notify_observers(self):
        for observer in self.observers:
            observer.update_lang()

# 在其他组件中注册并响应更新
class YourComponent:
    def __init__(self, lang_service):
        self.lang_service = lang_service
        self.lang_service.register_observer(self)
        self.update_lang()

    def update_lang(self):
        self.lang_dict = self.lang_service.get_lang_data()
        # 更新组件的显示等

信号和槽机制（方法4）

信号和槽机制天然适合于事件驱动的应用程序，例如 PyQt 应用程序。您可以在语言服务类中定义一个信号，当语言设置更改时发出该信号。其他组件可以连接到这个信号并在语言更改时更新他们的显示。

from PyQt5.QtCore import QObject, pyqtSignal

class LangManager(QObject):
    lang_updated = pyqtSignal()

    def __init__(self):
        super().__init__()
        self.lang_dict = {}

    def update_lang(self, lang_data):
        self.lang_dict = lang_data
        self.lang_updated.emit()

# 在其他组件中连接信号
class YourComponent:
    def __init__(self, lang_manager):
        self.lang_manager = lang_manager
        self.lang_manager.lang_updated.connect(self.update_lang)
        self.update_lang()

    def update_lang(self):
        self.lang_dict = self.lang_manager.lang_dict
        # 更新组件的显示等

在这两种情况下，当设置窗口更改语言设置并相应地更新 LangService 或 LangManager 中的语言字典时，所有已注册或连接的组件都会收到通知并更新它们的显示，以反映新的语言设置。这种方式确保了应用程序内的语言显示始终是同步的，并且对于更改反应灵敏。

为了使用信号和槽的方式来处理语言更新，您需要在您的应用程序中实现一个中央管理器，该管理器负责维护和更新语言字典，并通过信号通知其他组件更新。下面是如何修改您的代码以实现这一点：

1. 创建语言管理器

首先，创建一个新的语言管理器类，这个类将负责存储语言字典，并在语言更改时发出信号。

from PyQt5.QtCore import QObject, pyqtSignal

class LangManager(QObject):
    lang_updated = pyqtSignal()

    def __init__(self):
        super().__init__()
        self.lang_dict = get_lang_dict()

    def update_lang(self, new_lang):
        self.lang_dict = LANG_DICTS[new_lang]
        self.lang_updated.emit()

    def get_lang(self):
        return self.lang_dict

3. 修改其他组件以使用 LangManager

在其他需要使用语言字典的组件中，您现在应该连接到 LangManager 的 lang_updated 信号，并在收到信号时更新语言。

class ActionExit(QObject):
    status_bar_message = pyqtSignal(str)

    def __init__(self, lang_manager: LangManager):
        super().__init__()
        self.lang_manager = lang_manager
        self.lang_manager.lang_updated.connect(self.update_lang)
        self.update_lang()

        self.action_exit = QAction(QIcon(get_resource_path('media/icons8-exit-26.png')), '', None)
        self.action_exit.setShortcut('Ctrl+Q')
        self.action_exit.triggered.connect(self.exit)
        self.update_action_text()

    def update_lang(self):
        self.lang = self.lang_manager.get_lang()
        self.update_action_text()

    def update_action_text(self):
        self.action_exit.setText(self.lang['ui.action_exit_1'])
        self.action_exit.setStatusTip(self.lang['ui.action_exit_2'])

    # 其他代码...

在这种方式下，当语言设置发生变化时，LangManager 会发出 lang_updated 信号，所有连接到这个信号的组件将更新他们的语言字典，并且根据新的语言设置更新显示。

这样的设计不仅使得语言更新的处理更加集中和统一，而且也使得组件之间的耦合减少，便于维护和扩展。
