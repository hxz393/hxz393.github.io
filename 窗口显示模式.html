在您的 DialogAbout 类中，对话框是作为模态窗口（modal window）打开的，这意味着在关闭对话框之前，用户无法与其他窗口交互。如果您想要允许用户在打开关于窗口时同时与主界面交互，您需要将对话框设置为非模态（non-modal）。

为了实现这个功能，您需要在 DialogAbout 类的构造函数中设置对话框的模态类型。您可以通过调用 setModal(False) 或直接使用 Qt.NonModal 标志来实现这一点。

修改后的 DialogAbout 构造函数如下所示：

class DialogAbout(QDialog):
    def __init__(self, lang_manager: LangManager):
        super().__init__(flags=Qt.Dialog | Qt.WindowCloseButtonHint)
        self.setModal(False)  # 设置对话框为非模态
        self.lang_manager = lang_manager
        self.lang_manager.lang_updated.connect(self.update_lang)
        # ... 其余代码 ...

在这个修改中，通过调用 self.setModal(False)，我们将对话框设置为非模态，这样用户就可以在对话框打开的同时与应用程序的其他部分交互。这通常用于提供信息、设置或帮助窗口，这些窗口不需要阻止用户与应用程序的其余部分交互。

如果调用 setModal(False) 或使用 Qt.NonModal 标志没有使 DialogAbout 对话框成为非模态，原因可能是由于您是如何显示对话框的。在 PyQt 中，有两种主要的方法来显示一个对话框：exec_() 和 show()。

  1. 使用 exec_() 方法： 当使用 exec_() 显示对话框时，它总是以模态方式显示，即使您已经设置了 setModal(False)。这意味着主窗口将无法响应任何操作，直到对话框关闭。

  2. 使用 show() 方法： 为了以非模态方式显示对话框，您应该使用 show() 方法。这允许对话框独立于主窗口显示，同时主窗口保持可交互。

根据您的需求，您可能需要修改 ActionAbout 类中显示对话框的代码。如果您目前使用的是 exec_()，请将其改为 show()。

例如：

def open_dialog(self):
    try:
        if not self.dialog_about:
            self.dialog_about = DialogAbout(self.lang_manager)

        self.dialog_about.show()  # 修改此处使用 show() 而不是 exec_()
    except Exception:
        # ... 异常处理 ...

使用 show() 后，对话框将以非模态方式显示，允许用户同时与主窗口和对话框交互。

在 PyQt 中，显示对话框的两种主要方法——exec_() 和 show()——有以下区别：

  1. exec_() 方法：

    • 模态操作： 当您使用 exec_() 显示一个对话框时，它会以模态方式运行。这意味着对话框打开时，用户无法与程序的其它部分交互。用户必须先处理完这个对话框，才能返回到程序的其它部分。
    • 阻塞调用： exec_() 是一个阻塞调用。这意味着当对话框打开时，程序的其它部分会停止执行，直到对话框关闭。
    • 返回值： exec_() 方法返回一个整数，通常用于表示用户如何关闭对话框（例如，点击了哪个按钮）。
    • 用途： 这种方法通常用于需要用户做出决定或输入信息才能继续的情况，如文件保存、确认操作等。

  2. show() 方法：

    • 非模态操作： 使用 show() 显示对话框时，对话框是非模态的。这意味着对话框打开时，用户仍然可以与程序的其他部分交互。
    • 非阻塞调用： show() 是一个非阻塞调用。当对话框显示时，程序的其它部分将继续执行。
    • 无返回值： show() 方法不返回特定值，因为它不阻塞程序的执行。
    • 用途： 这种方法通常用于提供额外信息、辅助工具或设置，用户可以在不关闭对话框的情况下继续与程序的其它部分交互。

选择哪种方法取决于您的具体需求。如果您需要用户在继续之前必须响应对话框，比如确认删除文件或保存更改，那么使用 exec_() 是合适的。而如果您想让对话框提供额外信息或辅助功能，同时允许用户继续使用程序的其他部分，那么使用 show() 更为适合。

当您使用 show() 显示一个对话框，且该对话框似乎“一闪而过”后就关闭了，这通常是因为对话框对象在作用域外被垃圾回收器回收了。在 PyQt 中，当一个变量（如对话框）离开其定义的作用域，Python 的垃圾回收器会自动清理这个对象。如果您在一个方法内创建并显示对话框，当该方法执行结束后，对话框对象可能会被销毁，从而导致对话框关闭。

为了防止这种情况发生，您需要确保对话框对象在整个应用程序生命周期内保持活动。一种常见的方法是将对话框对象作为类的属性。这样，只要类的实例存在，对话框对象就不会被垃圾回收。

根据您之前的描述，您似乎已经在 ActionAbout 类中这样做了。确保在调用 show() 时使用的是这个类属性中的对话框实例。

示例代码如下：

class ActionAbout(QObject):
    # ...

    def open_dialog(self):
        try:
            if not self.dialog_about:
                self.dialog_about = DialogAbout(self.lang_manager)

            self.dialog_about.show()  # 使用类属性中的对话框实例
        except Exception:
            # ... 异常处理 ...

在这个示例中，self.dialog_about 是 ActionAbout 类的一个属性，因此只要 ActionAbout 的实例存在，对话框就不会被垃圾回收。这应该解决对话框“一闪而过”然后关闭的问题。
