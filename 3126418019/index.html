Go 语言数据类型

发表于 2024-03-09 更新于 2024-07-23 分类于 Go ， 2.类型系统 阅读次数： 本文字数： 3.5k 阅读时长 ≈ 12 分钟

数据类型

数据类型定义了编程语言、编译器、数据库以及代码执行环境如何操作和处理数据的方式，同时也能提高内存使用效率。例如，当数据类型被定义为数字时，表示可以对其执行数学运算。

下面是 Go 语言中内置类型表：

类型 说明
int/uint 整数类型，支持无符号（unsigned） uint 和带符号（signed） int 两种整形。Go 语言也有己经定义了位数的整型：int8、int16、int32（rune 字符）、int64 和 uint8（byte 字节）、uint16、uint32、uint64 。8 位整型长度为 1 字节，64 位整型长度为 8 字节。还有 1 种用于储存指针的类型 uintptr，长度等同于 uint32 或 uint64.
float32/float64 浮点型，类型有 float32 和 float64（默认） 。由于 math 包中所有有关数学运算的函数都要求 float64 类型，所以尽可能使用 float64。字节长度分别为 4 字节和 8 字节。分别精确到小数点后 7 位和 15 位。
complex64/complex128 复数类型。字节长度分别为 8 字节和 16 字节。
string 字符串类型，字符串是一串固定长度的字符序列。Go 语言字符串是由单字节连接而成，所有字节默认使用 UTF-8 编码标识 Unicode 文本。在 Go 语言中字符串是不可变的。
boolean 布尔型，值为 true 或 false。类型长度为 1 字节。
复合类型 包括切片（slice）、映射（map）、通道（channel）、指针（pointer）、数组（array）、结构体（struct）、函数（function）、接口（interface）类型。
特殊类型 error 类型，用于处理错误信息的接口。

类型系统

在 Go 语言中，数据类型根据数据在内存中的存储和访问方式，以及它们在程序中如何被传递，可以分为以下几种：

  • 值类型（Value-type）：值类型直接存储数据值本身。当这些类型的变量被赋值给另一个变量或作为参数传递给函数时，实际上是创建并传递了原始数据的副本（值传递）。因此，修改副本不会影响原始数据。值类型包括：基本数据类型、数组、字符串和结构体。

  • 引用类型（Reference-type）：引用类型的变量存储的是数据的引用（即内存地址），而不是数据值本身。为变量分配引用类型的值时，会复制内存地址的副本，而不是原始数据的副本，对引用类型的修改会影响所有引用该数据的变量。引用类型包括：切片、映射、通道和指针。

  • 接口类型（Interface types）：接口类型定义了一套方法的集合，但不会暴露对象的内部表示。它支持 Go 语言中的多态性，不同类型的对象可以被以相同的方式处理，只要它们实现了接口定义的方法。

  • 函数类型（Function types）：函数也是一种类型。可以作为参数传递给其他函数，也可以作为其他函数的返回值，用于实现回调和高阶函数。

对引用类型使用上，语法和值类型一样，不必获取引用类型的地址，也无需解引用引用类型来取值。

复合类型

复合类型（Compound-type），也称为构造类型（Structure-type）或导出类型，是指基于一种或多种已有的数据类型，通过特定的方式组合和构造出的新的数据类型。在 Go 语言中，复合类型包括以下几种：

  • 数组类型（Array Types）：通过确定的元素类型和固定的长度来定义，能够存储一系列相同类型的元素。
  • 切片类型（Slice Types）：基于数组类型构造，提供更灵活的序列表示形式，可以动态地调整大小。
  • 结构体类型（Struct Types）：将多个不同类型或相同类型的数据项组织在一起，形成一个复合数据类型。
  • 指针类型（Pointer Types）：存储另一种类型变量的内存地址，允许间接读取或修改该变量值。
  • 函数类型（Function Types）：由函数的参数类型和返回值类型定义，允许将函数作为参数或返回值使用。
  • 接口类型（Interface Types）：定义了一套方法集，任何实现了这些方法的类型都被认为实现了该接口。
  • 映射类型（Map Types）：通过键值对存储数据，每个键映射到一个值。
  • 通道类型（Channel Types）：用于在不同的协程间之间进行通信的类型，可以发送和接收值。

Go 语言支持用户根据这些构造类型定义自己的类型。

静态与动态类型

在静态类型语言中，变量一经声明便拥有了确定的类型，此后无法更改。任何类型不匹配的值都会在编译时引发错误；而在动态类型语言中，类型转换在执行时进行，错误或异常直到运行时才会被发现。通过对比静态和动态语言，可以展示两者之间的主要区别。首先看一个 JavaScript 函数的例子：

var addition = function (a, b) {
    return a + b;
};

函数 addition 本意接受两个数字参数，并返回它们的和。例如：addition(1, 2) 返回 3。但如果传递给函数的是一个数字和一个字符串，如 addition(1, "2")，JavaScript 会将数字转换为字符串，然后进行字符串拼接，返回 "12"，而不是将 "2" 转为数字 2 再与数字 1 求和。

相比之下，Go 语言的实现方式如下：

package main

import "fmt"

func addition(x int, y int) int {
	return x + y
}

func main() {
	fmt.Println(addition(1, 2)) // 输出: 3
	//fmt.Println(addition(1, "2")) // 编译错误：无法将 '"2"' (类型 string) 用作类型 int
}

这段代码同样定义了一个名为 addition 的函数，它接受两个整数参数并返回一个整数。然而在 main 函数调用时，第二个参数是一个字符串，因此会在编译时报错：cannot use "a" (type untyped string) as type int in argument to addition，指出类型不匹配。这种类型检查发生在编译时，而不是运行时，只有当错误被修正后，编译过程才能继续，从而确保错误不会暴露给用户。

类型实例初始化

Go 语言提供两种函数，用于不同的内存分配和初始化机制。

new

new 函数用于分配任何类型的内存空间，包括值类型（如整型、浮点型）和复合类型（如结构体、数组），返回一个指向被分配空间的指针，该空间被初始化为类型的零值。

new 函数适用场景：

  • 初始化零值指针：用于获取一个指向某类型的指针，期望初始值为类型零值。通常用于基本数据类型。

  • 明确的指针语义：需要在多个函数之间共享数据，或者需要传递指针给函数时，使用 new 有助于减少数据复制并明确表达要使用指针。

  • 与结构体配合使用：当想要一个指向结构体的指针，使用 new 很方便。可以顺便把结构体的字段初始化为零值。

当需要指针且想明确初始化为零值时，new 是一个合适的选择。其他时候一般直接初始化变量或使用字面量语法。

make

make 函数用于引用类型（包括切片、映射和通道）分配内存和初始化，返回初始化后的类型的实例：

  • 初始化切片：只有使用 make 初始化切片时，才能设定切片长度和容量参数。
  • 初始化映射：同样，想要指定映射容量参数时，必须使用 make 来初始化。
  • 初始化通道：make 是初始化通道唯一的方式，而通道只有初始化后才能使用。

make 用于初始化内建的复杂数据结构，这些结构需要在使用前有明确的初始化步骤。

自定义类型

自定义类型（Custom Types）通过 type 关键字创建，允许根据已有的数据类型定义新的数据类型。自定义类型不仅可以用来简化复杂类型的使用，还可以在基本类型或复合类型之上添加额外的方法。

自定义类型在类型系统中是独立的，即使底层类型相同，也不能将一个自定义类型的值赋给另一个自定义类型的变量，得先通过显式类型转换。

基于基本类型的自定义类型：

type UserID int
type UserName string

// IsValid 为 UserID 的一个方法
func (u UserID) IsValid() bool {
    return u > 0
}

// 类型转换示例
var uID UserID = 42
var n int = int(uID)  // 将 UserID 转换为 int

UserID 和 UserName 是 int 和 string 的自定义类型。

基于复合类型的自定义类型：

type UserMap map[string]int
type CallbackFunc func(int) bool

// GetUserScore 为 UserMap 的一个方法
func (um UserMap) GetUserScore(name string) int {
	return um[name]
}

UserMap 是基于映射的自定义类型，而 CallbackFunc 是基于函数的自定义类型。

类型检测

在 Go 语言中，可以利用标准库中的 reflect 包来检测变量的具体类型。reflect.TypeOf 函数返回一个 reflect.Type 对象，表示被检查变量的类型。如果想更简洁地只打印数据类型，可以直接使用 fmt.Printf 函数的 %T 动词来输出变量的类型：

package main

import (
	"fmt"
	"reflect"
)

// 检测并打印变量的类型
func printType(v interface{}) {
	// 使用 reflect.TypeOf() 获取变量的类型
	t := reflect.TypeOf(v)
	fmt.Printf("Reflect type: %s\n", t)

	// 使用 fmt.Printf 中的 %T 动词直接打印变量的类型
	fmt.Printf("Fmt type: %T\n", v)
}

func main() {
	// 定义不同类型的变量
	number := 42
	text := "Hello, Go!"
	floatingPoint := 3.14

	// 打印各变量的类型
	printType(number)
	printType(text)
	printType(floatingPoint)
}

类型转换

Go 语言是静态类型语言，因此所有类型转换必须显式进行。强制类型转换的语法格式为：类型名(变量名或表达式)。

例如，进行整型数值类型转换：

package main

import "fmt"

func main() {
	intNumber := 123

	// 调用 float32 函数进行类型转换
	floatNumber := float32(intNumber)

	// 打印转换后的结果
	fmt.Printf("整型值 %d 转换为 float32 类型的结果为 %f\n", intNumber, floatNumber)
}

转换过程中可能会产生值溢出或精度损失，需要小心进行：

package main

import (
	"fmt"
)

func main() {
	var a int32 = 65537
	var b float32 = 10.2

	// 将 int32 类型的 a 转换为 int16 类型，导致数据溢出，因为 65537 大于 int16 的最大值 32767
	fmt.Println("int32 to int16, 溢出情况:", int16(a)) // 发生数据溢出，输出：1

	// 将 float32 类型的 b 转换为 int16 类型，导致精度损失，因为浮点数 10.2 无法完全转换为整数
	fmt.Println("float32 to int16, 精度损失情况:", int16(b)) // 发生精度损失，输出：10
}

自定义类型转换方式也一样：

package main

import "fmt"

// UserID 定义用户 ID 的类型
type UserID int

// UserIDStarter 定义初始用户 ID 的类型
type UserIDStarter int

func main() {
	// 初始化 UserID 类型的变量 id
	var id UserID = 1

	// 初始化 UserIDStarter 类型的变量 start，用于指定起始的用户 ID
	var start UserIDStarter = 10000

	// 显式类型转换：将 UserIDStarter 类型的 start 转换为 UserID 类型
	id += UserID(start) // 必须显式转换 start 成同样的 UserID 类型后才能进行运算

	// 打印转换和计算后的结果
	fmt.Printf("新的 UserID 是：%d\n", id)
}

类型别名

除了官方定义的别名「字符型」和「字节型」，也可以对本地类型自定义类型别名（Type-alias）：

package main

import (
	"fmt"
	"reflect"
)

// 定义类型别名组，也可以单独定义类型别名，例如：type Boolean = bool
type (
	Boolean = bool // Boolean 是 bool 的类型别名
	BoolExt bool   // BoolExt 是基于 bool 的新类型
)

func main() {
	var boolAlias Boolean = true
	fmt.Printf("boolAlias 值：%v，类型：%v\n", boolAlias, reflect.TypeOf(boolAlias)) // 输出实际类型：bool

	var customBool BoolExt = false
	fmt.Printf("customBool 值：%v，类型：%v\n", customBool, reflect.TypeOf(customBool)) // 输出自定义类型：main.BoolExt
}

和自定义类型声明不同的是，定义类型别名必须要有等号 type 类型别名 = 原始类型。类型别名的主要用途是帮助代码重构，使得旧类型可以被新类型别名无缝替换。

类型复制

由于将引用类型传递给函数，可能会因为函数内操作导致原始数据被修改。因此在需要修改数据时，尽量通过创建副本操作并返回新副本，来保持原始数据的不变性。

根据数据类型的不同，复制操作也有所不同，下面总结各种类型常用的复制方法。

基本数据类型

对于基本数据类型，因为是值传递的，所以复制很简单，直接赋值就行：

x := 5
y := x  // y 现在是 x 的一个副本

切片

对于切片，通常先使用 make 函数创建新切片，再用 copy 函数来复制元素到新的切片中：

original := []int{1, 2, 3}
copyOfOriginal := make([]int, len(original))
copy(copyOfOriginal, original) // 将 original 的元素复制到 copyOfOriginal

结构体

对于结构体，如果结构体中都是基本数据类型，可通过直接赋值来复制。如果结构体中包含引用类型，则需要单独对字段深拷贝：

type Person struct {
    Name string
    Age  int
    Tags []string
}

// 浅拷贝
p1 := Person{Name: "Alice", Age: 30, Tags: []string{"friendly", "happy"}}
p2 := p1  // p2 是 p1 的副本，但 Tags 切片仍然指向同一个底层数组

// 深拷贝（需要手动复制所有引用类型）
p2.Tags = make([]string, len(p1.Tags))
copy(p2.Tags, p1.Tags)

映射

对于映射，需要手动创建一个新映射并复制每个键值对：

originalMap := map[string]int{"a": 1, "b": 2}
copyOfMap := make(map[string]int)
for key, value := range originalMap {
    copyOfMap[key] = value
}
