针对您提到的 format_query_results 函数中的 Magic Strings 和配置数据处理问题，可以采取以下改进措施：

改善 Magic Strings

Magic Strings 是指在代码中直接使用的硬编码字符串，这些字符串通常关联着特定的逻辑或配置。在您的函数中，例如使用 'Apollo' 来判断配置中心的类型。为了改善这个问题，可以采用以下方法：

  1. 使用枚举（Enumerations）:

    • 创建一个枚举类来定义所有配置中心的类型。这样就可以在代码中使用这些枚举值，而不是直接使用硬编码的字符串。

    • 例如：

      from enum import Enum

class ConfigCenter(Enum):
    APOLLO = "Apollo"
    NACOS = "Nacos"
    # 可以根据需要添加其他配置中心

    • 在函数中使用枚举值而不是硬编码字符串：

      if config_main['config_center'] == ConfigCenter.APOLLO.value:
    # 处理 Apollo 相关逻辑

改善配置数据处理

对于配置数据的处理，目前您的函数直接从 config_main 字典中提取数据。可以通过以下方式进行改进：

  1. 使用数据类（Data Classes）或命名元组（Named Tuples）:

    • 定义一个数据类或命名元组，将配置数据封装成一个对象。

    • 这样可以更清晰地定义每个配置项的类型和默认值，同时也提高了代码的可读性。

    • 例如，使用 dataclass：

      from dataclasses import dataclass

@dataclass
class Config:
    fix_name_left: str
    fix_name_right: str
    fix_name_before: str
    fix_name_after: str
    config_center: ConfigCenter  # 使用前面定义的枚举类

    def get_replacements(self):
        return {search: replace for search, replace in zip(self.fix_name_before.split(), self.fix_name_after.split())}

    • 在函数中使用这个数据类：

      def format_query_results(query_results: QueryResult, env_name: str, config: Config, formatted_results: Dict[str, Dict[str, str]]) -> None:
    # 使用 config 对象
    replacements = config.get_replacements()
    # ... 其他逻辑

通过这些改进，您的代码将更加清晰、可维护，同时也减少了硬编码和潜在的错误。
