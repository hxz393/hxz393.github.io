Go 语言创建 HTTP 服务器

发表于 2024-06-27 更新于 2024-07-23 分类于 Go ， 5.网络编程 阅读次数： 本文字数： 1.6k 阅读时长 ≈ 5 分钟

启动服务

要创建一个简单的 HTTP 服务器端，首先要使用 http.HandleFunc 函数注册路由，然后通过 http.ListenAndServe 函数绑定地址和端口，开启对客户端的监听。http.ListenAndServe 函数的第二个参数通常为空，表示调用 http.DefaultServeMux 进行处理，在服务端编写的业务逻辑处理程序 http.Handle 或 http.HandleFunc 会默认注入其中。：

package main

import "net/http"

// hello 函数接受一个 http.ResponseWriter 和一个指向请求的指针
func hello(w http.ResponseWriter, r *http.Request) {
	w.Write([]byte("Hello!\n")) // 使用方法 Write 来写入字节切片到响应
}

func main() {
	http.HandleFunc("/", hello)      // 创建根路由，由 hello 函数响应
	http.HandleFunc("/world", hello) // 创建路由/world，同样的功能

	http.ListenAndServe(":8080", nil) // 启动服务器，监听 8080 端口
}

运行程序后，可以通过浏览器访问地址：http://127.0.0.1:8080/ 和 http://127.0.0.1:8080/world 来获得响应。

自定义服务器

http.ListenAndServe 启动的服务器只能配置监听端口，可以自定义 http.Server 对象来设置超时参数、TLS 配置、以及更复杂的行为如请求路由、中间件处理等。一些常见设置：

  • 端口和地址：通过设置 Addr 字段，可以定义服务器监听的 IP 地址和端口。

  • 超时设置：可以设置读、写以及空闲连接的超时时间。

  • Handler：指定一个根处理器，处理进入的所有 HTTP 请求。通常是一个路由处理器或一个中间件链。

  • TLS 配置：如果需要 HTTPS 服务器，可以通过配置 TLSConfig 字段来实现。

一个创建自定义 http.Server 并启动：

package main

import (
	"fmt"
	"net/http"
	"time"
)

func main() {
	mux := http.NewServeMux() // 创建 HTTP 处理器
	mux.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintln(w, "Hello, World!")
	})

	// 创建自定义的 http.Server
	server := &http.Server{
		Addr:         ":8080",          // 监听地址和端口
		Handler:      mux,              // 使用自定义的多路复用器。要使用默认 http.DefaultServeMux 则设为 nil
		ReadTimeout:  10 * time.Second, // 设置读取超时
		WriteTimeout: 10 * time.Second, // 设置写入超时
		IdleTimeout:  15 * time.Second, // 设置空闲连接超时
	}
	
	// 默认开启会话保持，可以手动设置关闭
	server.SetKeepAlivesEnabled(false)

	fmt.Println("Server is running at http://localhost:8080/")
	if err := server.ListenAndServe(); err != nil {
		fmt.Println("Error starting server:", err)
	}
}

文件服务器

可以通过 http.FileServer 方法来创建一个静态文件服务器：

package main

import (
	"log"
	"net/http"
)

func main() {
	// 指定目录路径
	h := http.FileServer(http.Dir("./internal"))
	// 启动文件服务器，监听 8080 端口
	log.Fatal(http.ListenAndServe(":8080", h))
}

路由

HandleFunc 用于注册对 URL 地址映射进行响应的函数，其中路由地址必须完全匹配。例如对 /world/ 发起请求，不会响应 /world，而会因为没有注册重定向到根目录 /。

路由器不关心请求类型，只管将与路由匹配的请求传递给对应的函数。

路由默认使用 ServeMux 来处理，因此不支持变量和正则表达式。

处理程序

在路由器将路由映射到函数后，处理请求及返回响应由处理程序函数定义。具体来说，处理程序负责完成下面功能：

  • 读写报头；
  • 查看请求类型；
  • 从数据库中取回数据；
  • 分析请求数据；
  • 验证身份。

处理完请求后，最后发送响应给客户端。

返回 404

默认情况下，没有注册的路由地址会重定向到根路径，可以对此种情况返回 404 错误。修改根路径对应的函数，添加判断条件：

package main

import "net/http"

// hello 函数接受一个 http.ResponseWriter 和一个指向请求的指针
func hello(w http.ResponseWriter, r *http.Request) {
	if r.URL.Path != "/" { // 如果路径不匹配，调用 NotFound 方法，将返回 404 响应
		http.NotFound(w, r)
		return
	}
	w.Write([]byte("Hello!\n")) // 使用方法 Write 来生成响应
}

func main() {
	http.HandleFunc("/", hello)       // 创建根路由，由 hello 函数响应
	http.ListenAndServe(":8080", nil) // 启动服务器，监听 8080 端口
}

测试效果：

HTTP/1.1 404 Not Found
Content-Type: text/plain; charset=utf-8
X-Content-Type-Options: nosniff
Date: Thu, 27 Jun 2024 03:14:27 GMT
Content-Length: 19

404 page not found

设置报头

可以通过 Header().Set 方法来设置报头：

package main

import "net/http"

// hello 函数接受一个 http.ResponseWriter 和一个指向请求的指针
func hello(w http.ResponseWriter, r *http.Request) {
	w.Header().Set("Content-Type", "application/json; charset=utf-8") // 设置响应头 Content-Type，声明响应体数据类型为 json
	w.Write([]byte(`{"hello": "world"}`))                             // 写入 json 格式内容
}

func main() {
	http.HandleFunc("/", hello)       // 创建根路由，由 hello 函数响应
	http.ListenAndServe(":8080", nil) // 启动服务器，监听 8080 端口
}

测试效果：

root@k8s-204:~# curl -is http://192.168.2.101:8080
HTTP/1.1 200 OK
Content-Type: application/json; charset=utf-8
Date: Thu, 27 Jun 2024 03:18:58 GMT
Content-Length: 18

{"hello": "world"}

响应方法

对于不同类型的请求方法，可以使用 switch 来检查处理：

package main

import "net/http"

// hello 函数接受一个 http.ResponseWriter 和一个指向请求的指针
func hello(w http.ResponseWriter, r *http.Request) {
	switch r.Method {
	case "GET":
		w.Write([]byte("响应 GET 请求\n"))
	case "POST":
		w.Write([]byte("响应 POST 请求\n"))
	default:
		w.WriteHeader(http.StatusNotImplemented) // 发送 501 状态码
		w.Write([]byte(http.StatusText(http.StatusNotImplemented) + "\n"))
	}
}

func main() {
	http.HandleFunc("/", hello)       // 创建根路由，由 hello 函数响应
	http.ListenAndServe(":8080", nil) // 启动服务器，监听 8080 端口
}

通过 curl -X 来指定请求方法，测试结果：

root@k8s-204:~# curl -is -X PUT http://192.168.2.101:8080
HTTP/1.1 501 Not Implemented
Date: Thu, 27 Jun 2024 03:27:30 GMT
Content-Length: 16
Content-Type: text/plain; charset=utf-8

Not Implemented

获取请求数据

对于 GET 请求，需要获取请求 URL 中包含的查询字符串参数，而在 POST 请求中，数据在请求体中发送：

package main

import (
	"fmt"
	"io"
	"net/http"
)

// hello 函数接受一个 http.ResponseWriter 和一个指向请求的指针
func hello(w http.ResponseWriter, r *http.Request) {
	switch r.Method {
	case "GET":
		// curl -is "http://192.168.2.101:8080/?q=33&p=22" 将获取到
		//q: [33]
		//p: [22]
		for k, v := range r.URL.Query() {
			fmt.Printf("%s: %s\n", k, v)
		}
		w.Write([]byte("响应 GET 请求\n"))
	case "POST":
		// 测试命令 curl -X POST -d "POST DATA" -is http://192.168.2.101:8080/
		reqBody, err := io.ReadAll(r.Body)
		if err != nil {
			w.WriteHeader(http.StatusInternalServerError) // 发送 500 状态码
			w.Write([]byte(http.StatusText(http.StatusInternalServerError) + "\n"))
			return
		}
		fmt.Printf("%s\n", reqBody)
		w.Write([]byte("响应 POST 请求\n"))
	default:
		w.WriteHeader(http.StatusNotImplemented) // 发送 501 状态码
		w.Write([]byte(http.StatusText(http.StatusNotImplemented) + "\n"))
	}
}

func main() {
	http.HandleFunc("/", hello)       // 创建根路由，由 hello 函数响应
	http.ListenAndServe(":8080", nil) // 启动服务器，监听 8080 端口
}

模板

HTML 模板由 html/template 提供，通过 template.ParseFiles 读取 HTML 文件创建模板，然后通过 Execute 方法来将数据写入到模板：

package main

import (
	"html/template"
	"log"
	"net/http"
)

func check(err error) {
	if err != nil {
		log.Fatal(err)
	}
}

func viewHandler(w http.ResponseWriter, r *http.Request) {
	html, err := template.ParseFiles("view.html") // 读取 html 内容创建一个新模板
	check(err)
	err = html.Execute(w, "nil?") // 将模板内容写入到响应
	check(err)
}

func main() {
	http.HandleFunc("/v", viewHandler)
	err := http.ListenAndServe(":8080", nil)
	log.Fatal(err)
}
