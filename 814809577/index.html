Go 语言程序结构

发表于 2024-02-27 更新于 2024-07-23 分类于 Go ， 1.入门基础 阅读次数： 本文字数： 3.7k 阅读时长 ≈ 12 分钟

项目结构

使用 Go Modules 的项目，其结构可以相对简单且灵活，一切根据项目需求进行调整。

源码分类

在 Go 语言中，源代码文件分为命令源代码、库源代码和测试源代码，它们功能和内容结构不同：

  • 命令源代码：作为可执行程序的入口，能通过 go run 命令直接启动运行。
  • 库源代码：主要包含全局变量、常量、接口、结构体和函数等程序实体。
  • 测试源代码：对上面两种源码文件中的程序实体功能与性能进行测试，通过 go test 命令运行。

源码默认使用 UTF-8 编码。

基本文件

项目根目录下 go.mod 和 main.go 文件，是一个项目能运行最基础配置：

  • go.mod： Go 模块核心文件，定义了项目的模块路径以及依赖关系。

  • main.go：程序的入口文件，包含 main 函数。

其中，go.mod 文件由 go mod init <项目名> 命令自动生成，之后下载依赖包时会自动更新内容。main.go 文件也可叫其他名字，但会显得不专业。

常用结构

随着项目的发展，可能建立以下目录，来存放不同功能和类型的文件：

  • cmd：用于存放项目的可执行文件入口点。如果项目生成多个二进制文件，可以用子目录划分。例如：cmd/api/main.go，cmd/cli/main.go。
  • pkg：用于存放可以被外部应用使用的库代码。这些代码应该是可导出的，支持被其他 Go 项目导入。
  • internal：存放私有的应用代码和库代码，不能被外部项目导入，只允许同一模块的其他包导入。这里的模块指的是包含 go.mod 文件的根目录所定义的模块。
  • config：存放配置文件模板或默认配置。
  • assets：存放需要使用的媒体文件。
  • scripts：存放用于编译、部署、测试等操作的脚本。
  • test：存放外部测试的包和数据。

规范参考来源于：https://github.com/golang-standards/project-layout

包

Go 程序以包（package）的形式来封装、组织和重用代码，每个包可含一个到多个 .go 文件，这些文件都位于同一个以包名为目录名的目录下，并且这些文件的第一行声明了相同的包名。

一个简单的包结构通常包括三部分：包声明、包引入和包内容。

包定义

包是 Go 代码的一个集合，通过把一组有相关特性的函数和数据放进一个单元中，方便使用和更新相应的模块。

包系统使得每个模块都与程序和其他包保持一定的独立性，每个模块可以被应用到不同的程序中，通过包的利用提高了整体的开发效率。

与大部分编译语言类似，当改动源文件时，必须重新编译该源文件，以及它对应和依赖的包。但和其他语言比起来，编译速度明显快得多，原因如下：

  • 每个源文件在开头显式地列出所有依赖的包，这样编译器不必读取整个文件来确定其依赖的包列表。
  • 包之间禁止循环依赖，所以包可以被单独编译，也支持并行编译。
  • 每个包仅需编译一次然后缓存，当代码没变化时，编译器会重用之前的编译结果缓存而非重新编译。

包作用域

不同包之间为了实现某个类似功能，可能会采用相同名字的函数，为此 Go 语言引入命名空间概念，来消除调用函数时产生的歧义。

包级变量对属同一个包的所有文件可见，就像包的所有源代码写在同一个文件中一样。

但包中一个文件导入的第三方包仅对该文件有效，包中其他文件如果需要也必须显式地进行导入。这是因为 Go 的每个文件都被设计为可以独立编译，所以每个文件必须独立声明它自己所依赖的包。

包声明

所有导入的包必须在文件的开头显式声明，表示该文件属于哪个包。包声明格式如下：

package <pkgName>

其中 pkgName 为当前文件所属的包名，包名使用小写形式的单个单词命名。每个 Go 文件都属于且仅属于一个包，因此一个包可以由许多个 .go 源文件组成，一个应用程序可以由多个包组成。

属于同一个包的源文件必须全部被一起编译，所以每个目录都只能包含一个包，也不能把同个包的文件拆分到多个目录中。

当要生成可执行程序时，必须建立 main 包，并且该包中应包含 main 函数作为程序运行入口。main 包和 main 函数在一个可执行程序中只能有一个。

对非 main 包运行 go build 命令，编译器会编译该包中的所有源文件来检查错误，但不会产生任何输出文件。也可以使用 go install 命令生成一个归档文件（.a 文件），编译后的包文件安装到 $GOPATH/pkg 目录下。

包导入

无论需要调用标准库函数，还是本地包，都要使用导入来使用。Go 语言中，使用关键字 import 来导入包导出的变量、常量和函数。导入包有三种模式：

  • 正常模式：import <pkgName> 。导入 pkgName 之后，可以使用 <pkgName>.<funcName> 对包中的函数或类型进行调用。
  • 别名模式：import 别名 <pkgName>。当导入第三方包时，可能遇到包名相同或相近的情况，此时可以使用别名来进行区别和调用。别名模式还可以解决包名过长、避免与变量或常量命名冲突等情况。注意，一旦定义了别名，就不能再使用包原来的名字来调用它。
  • 简便模式：import .<pkgName>。在简便模式中，可直接使用函数名进行第三方包的调用，而不用带上包名。简便模式和别名模式同时使用时，可能引起的命名冲突和在代码中引用时的不明确性，通常不会使用此导入方式。

导入的包一定得使用，否则编译不通过。想绕过编译器检查匿名导入，可以使用下划线 _ 作为包的别名，这样可以编译导入包但不使用，常用于执行引入包初始化（init）函数。

多个导入包时，可以使用导入块来包装，并通过空行分隔进行分组。

所有非标准库包的导入路径以所在组织的互联网域名为前缀，例如：github.com/go-sql-driver/mysql，默认的包名就是包导入路径的最后一段。本地包则通过 模块路径/<pkgName> 来导入，路径分隔符统一使用 /。

注意，不能导入 main 包。

包内容

包可以包含函数、类型（如结构体和接口）和变量的声明。其中全局变量初始化最先执行，然后再执行初始化函数。

包使用示范

假设要创建一个示范项目 new ，有一个名为 hello 的包，该包有一个函数 World，用来返回问候语。

下面是项目的目录结构：

new/
├── internal/
│   ├── hello/
│   │   └── world.go
├── go.mod
├── main.go

主函数文件 main.go 的内容：

package main

import (
	"fmt"                          // 导入自带模块方式
	dp "github.com/mohae/deepcopy" // 导入第三方模块方式，使用别名
	"new/internal/hello"           // 导入本地模块方式
)

func main() {
	type Config struct {
		Version string `json:"version"`
		Name    string `json:"name"`
	}

	var config Config
	config.Version = "1.0.0"
	config.Name = "NewConfig"
	fmt.Println(config)

	cpy := dp.Copy(config) // 调用第三方包
	fmt.Printf("cpy pointer: %p\n", &cpy)
	fmt.Printf("config pointer: %p\n", &config)

	result := hello.World() // 调用本地包方式一样
	fmt.Println(result)
}

包 hello 所在文件 world.go 内容：

/*
Package hello 包最顶部说明
*/
package hello

import "fmt"

// World 函数的说明
func World() string {
	fmt.Println("hello world")
	return "world"
}

包可以由很多个 .go 源文件组成，因此对源文件命名没有规定，只要保证包名和目录名一致即可。

模块

模块是包的集合，提供包含多个相关包的更高级的组织结构。模块使用 go.mod 来管理模块的路径和依赖关系。

初始化模块

在项目根目录下，打开终端使用以下命令来初始化一个新的模块，同时创建 go.mod 文件：

go mod init <module-name>

其中 <module-name> 为项目名，可使用项目的 GitHub 路径，例如 ：github.com/hxz393/projectname，这样能保证模块名的全球唯一性。

使用模块

一个 go.mod 示例文件内容如下：

module github.com/hxz393/myproject

go 1.22.0

require (
    github.com/gin-gonic/gin v1.6.3
    github.com/mohae/deepcopy v0.0.0-20170929034955-c48cc78d4826 // indirect
)

replace (
    github.com/gin-gonic/gin v1.6.3 => github.com/gin-gonic/gin v1.8.0
    github.com/mohae/deepcopy => ../myproject/deepcopy
)

exclude (
    github.com/gin-gonic/gin v1.7.0
)

retract (
    v0.0.1 // 不再支持
    v0.2.3 // 有安全问题
)

一般 go.mod 文件包含以下几个部分：

  • module：声明模块的路径，也就是使用 go mod init 命令时输入的项目名。
  • go：声明项目使用的 Go 语言版本，编译时不得低于指定版本。
  • require：列出项目直接依赖的模块及其版本号。没有版本号的情况下，由生成的提交号代替。
  • exclude：可以指定排除特定模块的特定版本，避免代码缺陷或版本不兼容。
  • replace：可以替换依赖的模块为本地的模块或指定版本的模块。
  • retract：标记本模块的特定版本为弃用。

管理模块

当使用 go get 命令安装新的依赖时，go.mod 文件的内容会自动更新，并同时创建 go.sum 文件用来存储依赖的哈希值。其他时候，使用 go mod 命令来管理模块：

  • 整理依赖：运行 go mod tidy 命令可以清理未使用的依赖。
  • 查看依赖图：使用 go mod graph 命令输出项目的依赖关系图。
  • 下载依赖：运行 go mod download 可以下载 go.mod 文件中列出的所有依赖。
  • 验证依赖：运行 go mod verify 可以验证当前模块的依赖是否正确。

如果要查看当前模块所依赖的模块，可以使用 go list 命令：

PS D:\Software\Programming\Go\new> go list -m all
new
cloud.google.com/go v0.26.0
filippo.io/edwards25519 v1.1.0
github.com/BurntSushi/toml v1.4.0

程序语句

无论是运算操作还是流程控制，都由相应的程序语句完成。

简单语句

组成顺序结构程序的简单语句有下面几种：

  • 表达式语句：由表达式加分号 ; 组成。通常一行只有一条表达式语句时可以省略分号，如果一行包含多条语句，则必须使用分号将不同语句隔开，但编译器运行时会自动把一行中多条语句拆成多行。

  • 函数调用语句：基本格式为函数名(参数列表)。如果函数从外部包中导入，需要带上包名 包名.函数名(参数)。

  • 空语句：空语句用一个分号 ; 表示，空语句在语法上占有一条简单语句的位置，而执行该语句不调用任何操作，通常用来占位。

复合语句

复合语句（Compound Statement）是由一对大括号将多条语句组合在一起而构成的，在语法上相当于一条语句，又称为分程序：

package main

import "fmt"

func main() {
	a := 1
	fmt.Println(a) // 打印 a 的值 1
	{
		a := 2
		fmt.Println(a) // 打印复合语句中 a 的值 2
		{
			a := 3
			fmt.Println(a) // 打印嵌套的复合语句中 a 的值 3
		}
	}
}

上面的 main 函数只有一条复合语句，这条复合语句中又嵌套了另一条复合语句，各层中定义的 a 都是局部变量，不相影响。

语法规则

下面是一些 Go 程序语法注意事项。

分号

在 Go 语言中，分号 ; 由编译器在扫描源代码过程中使用简单的规则自动插入，通常仅在流程控制语句中手动使用分号，用来分开初始化语句、条件表达式语句或步进表达式语句。

代码块

在 Go 语言中，把一对大括号 {} 中的若干程序语句称为一个代码块。其实一个源代码文件本身就是一个代码块，一个代码包也是一个代码块，甚至整个项目的代码都是一个代码块。因此代码块是互相包含关系，就像复合语句例子中展示的那样。

在 Go 语言中，函数和控制结构的左大括号 { 必须和函数声明或控制结构放在同一行。如果左大括号另起一行，编译器会在左大括号的前方自动插入一个分号，导致编译报错。

注释

注释语句（Comment Statement）是在程序的开始或中间，不具有任何功能，仅对程序进行说明的语句。

Go 语言中注释有两种：单行注释和多行注释（块注释）。编译器遇到单行注释会当作空格；多行注释当作换行符，中间内容会完全忽略。

注释可以用来给包和包的函数添加描述，这些文档信息将可以通过 go doc 命令来展示。

单行注释

单行注释以两个斜杠 // 作为开头，一般对单行语句的含义进行说明。可以单独放一行，也可以跟在代码后面，和代码同一行：

// 注释文字，常见于函数之前说明其用途
a = b + c // 注释文字，常用于解释说明难懂代码

多行注释

多行注释以斜杠加星号 /* 开头，以星号加斜杠 */ 结束，经常用于对一个函数或语句块进行说明，也用于注释大段代码：

/*
可用于注释代码
也可用于多行说明文字
在单行说明文字放不下的时候
*/

在 package 语句之前的块注释，被视为这个包的文档说明。几乎所有全局作用域的类型、常量、变量、函数和被导出对象都应该有一个合理的注释。
