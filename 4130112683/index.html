Go 语言安装配置

发表于 2024-01-14 更新于 2024-07-23 分类于 Go ， 1.入门基础 阅读次数： 本文字数： 1.8k 阅读时长 ≈ 6 分钟

安装

本篇介绍在 Windows 和 Linux 平台上安装 Go 语言开发环境的方法。

版本选择

软件的测试版本通常经历三个阶段：

  • Alpha 版：软件开发早期的版本，通常仅供开发者内部测试。
  • Beta 版：也称为公开测试版，向所有用户开放，以收集反馈。
  • Gamma 版：接近最终产品的版本，也称为版本发布的候选版。

在官网上，可下载的版本分为两种：「Stable versions」表示稳定版本；「Release Candidate」表示测试版本。稳定版本一年发布两次，官方承诺对最新的两个稳定大版本提供支持，并保持向后兼容。因此安装时选择最新版本即可。

环境变量

安装 Go 语言开发环境时，需要设置以下全局的操作系统环境变量：

  • $GOROOT：Go 语言安装路径，需使用绝对路径，例如：D:\Software\Program\Go1.21.6。
  • $GOPATH：Go 语言工作目录，可以设置多个路径，例如默认路径：%USERPROFILE%\go。

通过 go env 命令可以查看的一些其他环境变量包括：

环境变量 含义
$GOARCH 当前 CPU 架构，例如：amd64
$GOOS 当前操作系统，例如：windows
$GOBIN 工作目录下的 bin 文件夹
$GOEXE 可执行文件的后缀，例如 Windows 下为 .exe
$GOHOSTARCH 用于交叉编译的 CPU 架构，例如：amd64
$GOHOSTOS 用于交叉编译的操作系统，例如：windows
$GOPROXY 用于设置模块下载代理，默认为：https://proxy.golang.org,direct

此外，由于国内网络环境不佳，需要修改下载模块代理的值为 https://goproxy.cn：

$env:GOPROXY = "https://goproxy.cn,direct"

Linux 下面将修改环境变量的命令加入用户配置文件：

echo "export GO111MODULE=on" >> ~/.profile
echo "export GOPROXY=https://goproxy.cn" >> ~/.profile
source ~/.profile

Windows

在 Windows 10 下安装 Go 语言的步骤如下：

  1. 访问官方下载页面 https://go.dev/dl/，选择适用于 Microsoft Windows 的版本下载 .msi 安装文件，文件大小约为 50 MB。
  2. 运行下载的安装文件，按提示操作，建议将安装路径设置为包含版本号的文件夹，例如：D:\Software\Program\Go1.21.6。
  3. 完成安装后，在命令提示符 (cmd) 中输入 go env 来检查环境变量，确认安装是否正确。若提示找不到可执行文件，需在系统变量 PATH 中添加 %GOROOT%\bin 和 %GOPATH%\bin。

Linux

在 Ubuntu 22.04.2 LTS 下安装 Go 语言的步骤如下：

  1. 使用 wget 命令下载 Linux 版本的压缩包：wget https://go.dev/dl/go1.21.6.linux-amd64.tar.gz。

  2. 解压缩文件：tar -xvf go1.21.6.linux-amd64.tar.gz。

  3. 将 GOROOT 和 GOPATH 加入到 .profile 或 .bashrc 文件中，示例如下：

    export GOROOT=$HOME/go
export GOPATH=$HOME/Workspace/Go:$HOME/other/Go
export PATH=$PATH:$GOROOT/bin:${GOPATH//://bin:}/bin

    若要为所有用户设置 Go 环境变量，可将上述内容添加到 /etc/profile 文件中。

另外，也可以通过系统包管理器安装 Go，例如使用 apt install golang-go 命令，但这种方式安装的版本不是最新的。此方法默认将 Go 安装到 GOPATH 的第一个目录 $HOME/Workspace/Go 内。

当脚本运行

由于 Go 语言编译速度快，可以配合 gorun 工具，在 Linux 下把 Go 语言当作脚本语言来用。先使用 go install 命令来安装模块，然后确保 $GOPATH/bin 被加入到系统变量 $PATH 中，否则会提示找不到可执行文件：

go install github.com/erning/gorun@latest
export PATH=$PATH:$(go env GOPATH)/bin

新建一个测试脚本，在首行添加 shebang：

#!/usr/bin/env gorun
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}

加上可执行权限，测试运行：

chmod +x script.go
./script.go

配置

在主流 IDE（Integrated Development Environment，集成开发环境）中设置 Go 开发环境。

GoLand

基本设置步骤如下：

  • GOROOT 配置：在设置菜单中选择「Go | GOROOT」，这里配置 Go 语言的安装位置。点击「+」添加新的 GOROOT，选择「Download…」下载并安装特定版本的 Go，例如安装到 D:\Software\Program\go。若已安装 Go，选择「Local…」来指定已存在的安装目录。注意，旧版本的 GoLand 可能无法识别较新版本的 Go 安装路径，导致「找不到 SDK」的错误提示。
  • GOPATH 配置：选择「Go | GOPATH」来设置全局和项目级的 GOPATH。GOPATH 可以指定多个工作目录，例如：D:\Software\Programming\Go 和 D:\Software\Programming\Go\project1。

为了使快捷键与 VSCode 保持一致，可以在设置中通过「按键映射」来自定义快捷键。

VSCode

VSCode 的 Go 开发环境配置步骤如下：

  1. 打开扩展市场，搜索并安装 Go 语言支持插件。
  2. 安装 Code Runner 扩展以增加一键运行代码的功能。
  3. 使用快捷键 Ctrl+Shift+P 打开命令面板，输入 Go: Install/Update Tools，选择并安装必要的 Go 工具，如 gopls（语言服务器），gofmt（格式化工具），delve（调试工具）等。
  4. 打开 Go 项目文件，VSCode 将提示安装任何其他需要的插件。

安装完成后，可以使用 Alt+Shift+F 快捷键进行代码格式化，使用 Ctrl+Alt+N 快捷键执行代码。

Sublime Text

配置 Sublime Text 步骤：

  1. 通过 Package Control 安装 Golang Build 官方插件，用于构建和测试 Go 程序。
  2. 安装 SublimeREPL 插件以支持命令行输入。
  3. 在 Sublime Text 的设置中配置 GOROOT 和 GOPATH 路径。
  4. 使用 Ctrl+B 快捷键进行代码编译。

目录结构

安装 Go 语言后，系统变量会自动添加 $GOROOT 和 $GOPATH。若需要，也可以手动修改这些系统变量。如果有多个工作目录，以分号「;」分隔多个设置值。

GOROOT

Go 语言安装目录下的重要子目录包括：

  • api：包含公开变量、常量、函数等的 API 列表，便于进行 Go 语言 API 检索。
  • bin：存放标准命令的执行文件，如 go、godoc 和 gofmt。
  • doc：包含标准库文档。在 1.10 版本及以下，可以使用命令 godoc -http=:10000 启动本地文档服务并通过浏览器访问。自 1.13 版本起，该方法被废弃，推荐使用在线文档服务 http://pkg.go.dev/ 或安装 golang.org/x/tools/cmd/godoc 来启动本地文档服务。
  • lib：包含一些特殊的库文件。
  • misc：包含辅助工具和说明。
  • pkg：存放编译 Go 语言标准库生成的文件。随着 Go 模块的推广，标准库以外的依赖包不再存放于 $GOPATH/pkg 下，而是放在 $GOPATH/pkg/mod 中，作为模块缓存的默认位置。
  • src：存放 Go 语言的源代码。
  • test：存放测试和验证相关的文件。

GOPATH

在开发工作目录中，一般包含 3 个主要文件夹：

  • bin：存放 go install 命令生成或下载的可执行文件。
  • pkg：存放 Go 编译时生成的 .a 归档（Archive）文件，也称作静态库文件。
  • src：存放 Go 项目的源代码，不同的工程项目按包名进行区分。

在引入了模块（module）后，传统的 pkg, bin, 和 src 目录结构已经不再是必需。
