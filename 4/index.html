语言介绍

发表于 2024-01-13 更新于 2024-07-23 分类于 Go ， 1.入门基础 阅读次数： 本文字数： 2.3k 阅读时长 ≈ 8 分钟

语言介绍

在 2007 年，面对 C 语言以及其衍生语言日益增长的复杂性，以及这些语言在多核处理器、并发编程和网络应用开发中的局限性，Google 的首席软件工程师 Rob Pike、Robert Griesemer 和 Ken Thompson 启动了一个新的编程语言项目 Go 语言（也称为 Golang）。新语言的目标是保持 C 语言的效率和控制力，同时减少其复杂性，提供更加安全、简洁的编程方式。

Go 语言继承了 C 语言的许多优点，如编译式执行、静态类型、效率高，同时通过引入诸如垃圾收集、并发模型（goroutines）、包管理等现代语言特性，解决了 C 和 C++ 在现代软件开发中面临的许多问题（一个有意思的说法：Go = C + GC + Goroutine）。Go 的设计哲学强调简洁和效率，意在提高开发者的生产力，同时保持运行时性能。这些特点使得 Go 在开发高性能网络服务和大型分布式系统时大受欢迎。

著名项目

现在（2023 年），Go 语言在云计算、边缘计算、大数据、微服务、物联网、高并发领域的应用日益广泛，已经成为云计算领域的事实上的标准语言。以下是一些采用 Go 语言开发的著名项目：

  1. Docker：一个开源的容器化平台，使开发者能够轻松构建、测试和部署应用。
  2. Kubernetes：一个开源的容器编排系统，用于自动化部署、扩展和管理计算机应用。
  3. Prometheus：一个开源监控和警报工具集，广泛用于实时度量数据的记录和警报生成，非常适合云原生环境。
  4. Etcd：一个分布式键值存储，主要用于共享配置和服务发现，适用于大规模分布式系统。
  5. InfluxDB：一个开源的时间序列数据库，专为处理高写入和查询负载而设计。
  6. Grafana：一个开源的度量分析和可视化套件。虽然其前端采用 TypeScript 开发，但后端服务是用 Go 语言实现的。

语言特点

Go 语言与其他常用编程语言相比，具有以下特点（或优势）：

  • 强静态类型语言，编译速度快；
  • 原生支持多核心处理器，从程序结构方面实现并发编程；
  • 提供无继承的类型系统，并采用组合设计模式，以降低代码复杂度；
  • 拥有现代化的垃圾回收机制；
  • 拥有健全的包管理机制；
  • 支持交叉编译，可在 Linux 系统上开发 Windows 应用程序。

因此可见，Go 语言在「快速编译、高效执行、易于开发」三者之间实现了最佳平衡。

适用场景

Go 语言主要适用于以下开发场景：

  • 云服务和微服务架构：在构建高性能、可伸缩的微服务方面，Go 语言表现出色，成为许多云平台和服务（如 Google Cloud、AWS）的首选语言。
  • 并发编程：Go 的并发模型通过「goroutines」和「channels」等机制简化了并发编程，非常适合需要处理大量并发连接和任务的应用。
  • 网络编程：标准库提供了丰富的网络库，便于开发者构建各种网络应用，如 HTTP 服务器、客户端应用等。
  • 命令行界面（CLI）工具：Go 的静态编译特性非常适合开发不需要任何外部依赖就能运行的跨平台命令行工具。
  • 分布式系统和容器化技术：在 Docker 和 Kubernetes 等现代容器化和编排工具中得到广泛使用，这些工具的核心组件通常是用 Go 编写的。
  • DevOps 和站点可靠性工程（SRE）工具：Go 语言的快速编译和执行速度、跨平台支持及易于部署的特性，使其成为开发用于自动化运维任务的理想选择。

目前还不太适用的开发场景包括：

  • 桌面 GUI 应用：相比于专为此设计的语言和框架（如 C# 的 WPF、Java 的 Swing），Go 在这一领域的生态和库支持相对较弱。
  • 移动应用开发：尽管 Go 语言支持一些实验性项目（如 golang/mobile）以支持移动开发，但目前并不是开发 Android 或 iOS 应用的主流选择。
  • 游戏开发：对于复杂游戏开发，Go 语言的生态不如 C++ 或 Unity 等专业的游戏开发工具和语言，主要是缺乏丰富的图形处理和游戏引擎支持。
  • 科学计算和大数据分析：虽然 Go 语言擅长处理并发任务，但对于需要复杂数学计算和数据分析的应用，Python 等语言因其成熟的科学计算库（如 NumPy、Pandas）通常是更好的选择。

版本时间线

下面是一些重要的版本发布信息：

  • Go 1（2012 年 3 月）
    正式发布 Go 1 版本：确立了 Go 语言的稳定性，为未来的向后兼容性奠定了基础。
  • Go 1.1（2013 年 5 月）
    性能改进：编译器和运行时性能显著提升。
    新特性：引入了新的语言特性和库改进。
  • Go 1.5（2015 年 8 月）
    编译器重写：Go 编译器由 C（原 gc 编译器）重写为 Go（实现自举），仅保留一些基础的汇编代码。提升了编译速度。
    并发性能提升：引入了改进的垃圾收集和并发性能。
  • Go 1.7（2016 年 8 月）
    context 包正式加入标准库。用于管理 API 边界和进程之间的请求。
  • Go 1.8（2017 年 2 月）
    性能改进：继续提升 HTTP/2 的性能和垃圾回收的效率。
  • Go 1.9（2017 年 8 月）
    类型别名：引入类型别名，帮助大型代码库的重构。
  • Go 1.10（2018 年 2 月）
    缓存编译结果：加快了编译速度。
    测试覆盖率改进：提供了更丰富的测试工具。
  • Go 1.11（2018 年 8 月）
    Go modules：正式引入模块支持，开始摆脱 GOPATH 的依赖。
  • Go 1.13（2019 年 9 月）
    错误处理改进：引入了 errors.Is 和 errors.As 函数，改善了错误处理。
    Go modules 成为默认依赖管理工具。
  • Go 1.14（2020 年 2 月）
    性能改进：在调度器、垃圾收集和 defer 语句的性能上有所提升。
    测试改进：增加了更多的测试选项和特性。
  • Go 1.15（2020 年 8 月）
    更小的二进制文件：改进了链接器，减少了编译后的文件大小。
    改进的 X.509 证书链的验证。
  • Go 1.16（2021 年 2 月）
    嵌入静态文件：引入了 embed 包，允许将静态文件嵌入到 Go 程序中。
    模块行为改进：对 Go modules 的处理进行了进一步的优化。
  • Go 1.17（2021 年 8 月）
    更好的核心库：对标准库进行了改进，提高了性能和兼容性。
    对 ARM 架构的优化：增强了对 ARM 架构的支持。
  • Go 1.18（2022 年 3 月）
    泛型支持：引入了泛型编程，这是 Go 语言最受期待的特性之一。
    工作空间模式：增加了对工作空间的支持，方便管理多个模块。
  • Go 1.19（2022 年 8 月）
    改进：引入了对泛型的进一步优化和改进，继续提高 Go 泛型的性能和稳定性。
    调度器性能：对调度器进行了优化，提高了并发性能。
    内存管理：改善了内存分配器的性能，减少了 GC（垃圾收集）的压力。
    安全性：增加了对更多加密算法和安全协议的支持。
  • Go 1.20（2023 年 2 月）
    工具链改进：进一步优化了编译器性能，提高了编译速度和生成二进制文件的效率。
    核心库增强：标准库中新增加和改进了多个包，增强了网络、文本处理和时间处理等方面的功能。
    安全性和可靠性：增强了代码的安全性和可靠性，包括对安全漏洞的修复和改进。
    模块支持：改进了对 Go 模块的支持，使得依赖管理更加简便和高效。

书籍

Go 语言相关书籍出的不多，总的来说国内作者的书结构良好，质量参差不齐，很有教科书和参考书风格。国外作者的书各有风格，能经过翻译过来的原书质量都很不错。

下面是一些学习过程中阅读过的书籍，评价仅代表个人意见：

  • Go语言趣学指南（Nathan Youngman等著，黄健宏译，人民邮电出版社，2020）
  • Go语言入门经典（George Ornbo，张海燕译，人民邮电出版社，2018）
  • Head First Go语言程序设计（Jay McGavren，刘红泉等译，机械工业出版社，2020）
  • Go语言程序设计（王鹏，清华大学出版社，2014）
