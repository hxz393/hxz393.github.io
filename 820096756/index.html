Go 语言中路由框架

发表于 2024-07-16 更新于 2024-07-23 分类于 Go ， 5.网络编程 阅读次数： 本文字数： 441 阅读时长 ≈ 1 分钟

Gin

Gin 是一个用 Go 语言编写的高性能 HTTP Web 框架，Gin 基于 httprouter 提供一个简洁的 API 用于快速开发。

使用前需要安装 Gin 包：

go get -u github.com/gin-gonic/gin

启动服务

下面使用 Gin 创建一个简单 HTTP 服务器：

package main

import (
	"github.com/gin-gonic/gin"
	"net/http"
)

func main() {
	// 创建初始化一个 Gin 实例
	r := gin.Default()

	// 注册一个路由和处理函数
	r.GET("/", func(c *gin.Context) {
		c.JSON(http.StatusOK, gin.H{"message": "hello world"})
	})

	// 默认监听并在 8080 端口上启动服务
	r.Run()
}

文件服务

gin 对静态资源支持分为文件夹 StaticFS 和文件 StaticFile：

package main

import (
	"github.com/gin-gonic/gin"
	"net/http"
)

func main() {
	r := gin.Default()

	// 静态资源加载
	r.StaticFS("/fs", http.Dir("internal"))
	r.StaticFile("/new.html", "./view.html")

	r.Run()
}

路由与路由组

Gin 支持 RESTful 风格的路由接口，可以自定义各种请求方法的处理函数：

package main

import (
	"github.com/gin-gonic/gin"
)

func main() {
	r := gin.Default()

	r.GET("/books", getBooks)
	r.POST("/books", postBook)

	r.Run()
}

func getBooks(c *gin.Context) {}

func postBook(c *gin.Context) {}

也可以使用路由组来组织相关的路由，在构建大型应用时常常用到：

package main

import (
	"github.com/gin-gonic/gin"
)

func main() {
	r := gin.Default()

	// 路由分组
	v1 := r.Group("/v1")
	{
		v1.GET("/books", getBooks)
		v1.POST("/books", postBook)
	}

	v2 := r.Group("/v2")
	{
		v2.PUT("/books/:id", getBooks)
		v2.DELETE("/books/:id", postBook)
	}

	r.Run()
}

func getBooks(c *gin.Context) {}
func postBook(c *gin.Context) {}

中间件

Gin 允许定义全局和路由级的中间件，用来处理诸如日志记录、用户认证、CORS 等功能：

package main

import (
	"github.com/gin-gonic/gin"
	"log"
	"time"
)

func main() {
	r := gin.Default()

	// 全局中间件
	r.Use(gin.Logger())
	r.Use(gin.Recovery())

	r.GET("/benchmark", MyBenchLogger(), benchEndpoint)

	r.Run()
}

func benchEndpoint(c *gin.Context) {}

// MyBenchLogger 是一个中间件函数
func MyBenchLogger() gin.HandlerFunc {
	return func(c *gin.Context) {
		t := time.Now()

		// 设置 example 变量
		c.Set("example", "12345")

		// 请求前

		c.Next() // 处理请求

		// 请求后
		latency := time.Since(t)
		log.Print(latency)

		// 访问发送的状态
		status := c.Writer.Status()
		log.Println(status)
	}
}
