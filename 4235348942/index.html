Go 语言控制流程

发表于 2024-01-14 更新于 2024-07-23 分类于 Go ， 3.核心概念 阅读次数： 本文字数： 4.1k 阅读时长 ≈ 14 分钟

流程控制

计算机程序设计结构一般可以分为三种：顺序结构（Chronological Structure）、选择结构（Selective Structure）和循环结构。其中顺序结构最简单，执行顺序自上而下，依次执行。

在 Go 语言中没有 do-while 语句，因此 for 语句有更广泛应用。另外 Go 语言还有 3 个特殊关键字：「defer」用于捕获异常和资源回收等工作；「select」配合通道用于多分支选择；「go」用于异步启动 goroutine 并执行特定函数。

条件语句

条件语句可以根据条件表达式来执行代码块中的分支，表达式只能返回布尔类型。

if 判断

如果一个条件选择程序仅有一个条件分支，则称该结构为单分支选择结构（Single-branch Selection Structure）。单分支选择结构使用 if 语句实现：

if 表达式 {
    语句 1
}

if 语句检查表达式的值，为 true 时执行语句 1，否则什么也不执行：

package main

import "fmt"

func main() {
	var score int = 80

	// 使用 if 语句进行条件判断
	if score >= 60 {
		fmt.Println("恭喜，你已经及格了！")
	}

	// 程序继续执行其他任务
	fmt.Println("请执行其他操作")
}

上面函数中，if 语句的判断条件为变量 score 是否大于等于 60，由于条件成立，执行后面大括号内的代码，打印结果，然后继续执行后面代码。

else 判断

如果一个条件选择程序中有两个条件分支，则称该结构为双分支选择结构（Dual-branch Selection Structure）。双分支选择结构可以使用 if...else 语句来实现：

if 表达式 {
    语句 1
} else {
    语句 2
}

else 语句指定到达该分支时将执行的代码。只要前面 if 语句表达式的结果为 true，则执行 语句 1；否则执行 语句 2：

package main

import "fmt"

func main() {
	var score int = 59

	// 使用 if...else 语句进行条件判断
	if score >= 60 {
		fmt.Println("恭喜，你已经及格了！")
	} else {
		fmt.Println("很遗憾，你没有及格。")
	}

	fmt.Println("这是考试结果，进入下一个环节")
}

else if 判断

如果一个条件选择程序中有多个分支条件，则称该结构为多分支选择结构（Multi-branch Selection Structure）。多分支选择结构可以使用 if else if 语句来实现：

if 表达式 1 {
    语句 1
} else if 表达式 2 {
    语句 2
     ...
} else if 表达式 n {
    语句 n
} else {
    语句 n+1
}

在多数情况下，需要多个判断条件来决定继续执行的代码，else if 语句能在前面判断条件为 false 时接着判断下一个条件：

package main

import "fmt"

func main() {
	var score int = 88

	// 使用 if, else if 和 else 语句进行条件判断
	if score >= 90 {
		fmt.Println("优秀！")
	} else if score >= 80 {
		fmt.Println("良好！")
	} else if score >= 60 {
		fmt.Println("及格！")
	} else {
		fmt.Println("不及格，请继续努力！")
	}

	fmt.Println("分数评价完毕，进入下一个环节")
}

初始化子语句

if 语句可以有一个子语句，用于初始化局部变量：

package main

import (
	"fmt"
	"strconv"
)

func main() {
	str := "123"

	// 使用 if 语句的初始化子语句来执行操作
	if num, err := strconv.Atoi(str); err != nil {
		fmt.Println("转换错误:", err)
	} else {
		fmt.Println("转换成功，数字为:", num)
	}
    
	// 注意：'num' 和 'err' 变量在这里是不可见的，它们的作用域仅限于 if 和 else 块内部
	//fmt.Println(num) // 报错：undefined: num

	if newInt := 11; newInt < 10 { // 等同于 newInt := 11; if newInt < 10 {}
		fmt.Println("newInt 小于 10")
		//fmt.Println(otherInt) // 流程控制语句为惰性求值，因此不能引用下面语句定义的变量
	} else if otherInt := 12; otherInt < 11 {
		fmt.Println(newInt) // 可以直接调用 if 初始化语句中定义的变量 newInt
	} else {
		fmt.Println(newInt, otherInt) // 最后可以引用上面所有初始化子语句中定义的变量
	}
}

初始化子语句跟在 if 关键字后面，只能有一个表达式。此外，在初始化子语句（包括 else if 分支）定义的变量，能在整个条件语句块使用，不能在代码块之外调用。

选择语句

对于多元判断，使用选择语句比条件语句更简便，可以避免条件语句烦琐的嵌套过程，同时编译代码的效率也更好。Go 语言改进了 switch 的语法设计，每一个 case 是一个独立的代码块，不需要通过 break 语句跳出当前 case 代码块以避免执行到下一行。

switch 语句

switch 语句会根据一个表达式的多种值，选择多个 case 分支，因而也被称为「开关」语句。case 分支语句依据控制表达式的值，选择执行相应的代码，然后退出整个 switch 代码块：

switch 条件表达式 {
case 常量表达式 1:
    语句 1
case 常量表达式 2:
    语句 2
    ...
case 常量表达式 n:
    语句 n
default:
    语句 n+1
}

switch {
case 条件表达式 1:
    语句 1
case 条件表达式 2:
    语句 2
    ...
case 条件表达式 n:
    语句 n
default:
    语句 n+1
}

对于条件判断中的 else 语句，可以使用 default case 代替。它通常放在语句末尾，但也可以放在其他位置。每个 switch 语句只能存在一个 default 分支。

switch 语句的控制类型由条件表达式决定，每个分支常量表达式值的类型必须和控制类型保持一致。如果有两个或以上分支的常量表达式取得相同的值，编译会报错。

如果 switch 条件表达式的值，不匹配任何 case 常量表达式的值，且没有 default 分支，则程序控制会跳转到 switch 语句结尾，继续执行后面代码。

在 Go 语言中 switch 语句有两种类型：「表达式 switch」和「类型 switch」。

表达式 switch

在表达式 switch 中，case 包含与 switch 表达式的值进行比较的表达式。具体到下面例子中，switch 关键词后面是一个变量，选择语句根据这个变量的值判断执行哪个 case 代码块：

package main

import "fmt"

func main() {
	number := 3
	switch number {
	case 1:
		fmt.Println("One")
	case 2:
		fmt.Println("Two")
	case 3, 4, 5: // 每个 case 都可以拥有多个表达式，只要匹配任意一个条件都会执行
		fmt.Println("Three, Four, Five")
	default:
		fmt.Println("Unknown number")
	}
}

更常用另一种写法，在 switch 关键字后没有条件表达式，将完整条件表达式放到 case 中，这种形式的 switch 允许每个 case 执行更复杂的表达式比较，而不仅是简单的值比较：

package main

import (
	"fmt"
	"time"
)

func main() {
	t := time.Now()

	switch {
	case t.Hour() < 12:
		fmt.Println("Good morning!")
	case t.Hour() < 17:
		fmt.Println("Good afternoon!")
	default:
		fmt.Println("Good evening!")
	}
}

类型 switch

类型 switch 用于检查一个接口变量的实际类型，根据具体的类型来执行不同操作。类型 switch 的初始化子语句中需要判断的必须是具有接口类型的变量：

package main

import "fmt"

func typeSwitch(i interface{}) {
    switch v := i.(type) { // 如果没用到变量 v 的值，可以直接写成 switch i.(type) {}
	case int:
		fmt.Printf("%v 乘以 2 等于 %v\n", v, v*2)
	case string:
		fmt.Printf("字符串 %q 长度为 %v 字节\n", v, len(v))
	default:
		fmt.Printf("不能处理的类型：%T\n", v)
	}
}

func main() {
	typeSwitch(21)      // 输出：21 乘以 2 等于 42
	typeSwitch("hello") // 输出：字符串 "hello" 长度为 5 字节
	typeSwitch(true)    // 输出：不能处理的类型：bool
}

类型判断中，不允许使用 fallthrough。

初始化子语句

和 if 语句一样，switch 语句同样将初始化子语句写在关键字之后，且只能有一句：

package main

import (
    "fmt"
    "time"
)

func main() {
    switch t := time.Now(); { // 初始化子语句：获取当前时间并赋值给 t
    case t.Hour() < 12:
        fmt.Println("Good morning!")
    case t.Hour() < 17:
        fmt.Println("Good afternoon!")
    default:
        fmt.Println("Good evening!")
    }
}

初始化变量的作用域限制在 switch 语句内部，能避免在函数的其他部分意外使用这些变量。

fallthrough 语句

关键字 fallthrough 用于执行完一个 case 分支代码后，忽略下个 case 分支的检查条件，无条件地执行分支代码：

package main

import "fmt"

func main() {
	number := 10
	switch {
	case number == 1:
		fmt.Println("one")
	case number == 10:
		fmt.Println("命中!")
		fallthrough // 只要满足这一分支条件，会一并执行下一分支代码
	case number == 11: // 碰到 fallthrough 时，无视条件直接执行里面代码。其他情况依然要判断
		fmt.Println("下降到这一分支!")
	case number == 12: // 但不会影响到这一分支
		fmt.Println("正常分支代码")
	default:
		fmt.Println("都没匹配上")
	}
}

通常用于兼容移植代码，一般不使用。

select 语句

select 语句用于配合通道（channel）的读写操作，用于多个通道的并发读写操作。不同于 switch 语句按顺序从上到下依次执行，select 随机选择一个 case 来判断，直到条件匹配，执行代码：

package main

import "fmt"

func main() {
	a := make(chan int, 100)
	b := make(chan int, 100)

	for i := 0; i < 10; i++ {
		fmt.Printf("第%d次循环：", i+1)
		a <- 1
		b <- 1

		select {
		case <-a:
			fmt.Println("来自a")
		case <-b:
			fmt.Println("来自b")
		}
	}
}

随机选择是为了保证在多个通道操作中提供公平性，防止某个通道因为位置优先或其他原因被频繁选择，也可以避免单个通道的阻塞。

如果 select 语句中的所有 case 都不满足条件，将执行 default 语句（非阻塞的通道操作）。如果没有 default 语句，select 语句将阻塞，直到至少有一个通道操作可以进行。

循环语句

需要重复执行某一段代码时，可以使用循环控制来实现这种设计，这种程序结构叫循环结构（Loop Structure）。在 Go 语言中，没有 while 关键字，循环语句的关键字是 for。

条件循环结构

for 语句能根据指定条件重复执行代码块，这在编程中被称为循环：

package main

import "fmt"

func main() {
	i := 0
	for i < 5 {
		fmt.Println(i) // 分别打印 0 到 4，共 5 次
		i++
	}
}

这种只有条件表达式的循环类似于 while 循环语句，叫做条件循环结构（Conditional-loop Structure）：

for 条件表达式 {
    循环体
    ...
}

如果忽略 for 的条件表达式，会形成无限死循环。

基本循环结构

除了条件表达式外，for 语句还可以指定循环开始时执行的初始化表达式，以及步进表达式，表达式之间使用分号来分隔：

for 初始化表达式; 条件表达式; 步进表达式 {
    循环体
    ...
}

这叫 for 基本循环结构（For-loop Structure），三个语句的顺序不能对换：

  • 初始化表达式：仅在初次迭代时执行，给循环变量赋初值。
  • 条件表达式：每次迭代前都检查的布尔表达式，如果为 true 则执行循环体中的语句；值为 false 则跳出循环。
  • 步进表达式：每次迭代后都将执行修改循环变量的值，然后跳到条件表达式继续判断循环。

包含初始化表达式和步进表达式后，上面的函数可以简化如下：

package main

import "fmt"

func main() {
	for i := 0; i < 5; i++ {
		fmt.Println(i)
	}
}

for 语句还允许有两个或以上循环变量同时参与循环控制，用于解决一些复杂的问题：

package main

import "fmt"

func main() {
	for i, j := 0, 10; i < j; i, j = i+1, j-1 {
		fmt.Println(i, j)
	}
}

循环变量一般习惯使用 i、j、k 等较短的名称命名计数器，不要在循环体内修改计数器。

无限循环结构

另一种比较常见的循环结构是无限循环结构（Infinite-loop Structure），无限循环结构退出的条件放在循环体中，一般由 if 语句和 break 语句构成：

for {
    循环体
    ...
}

下面是一个应用了无限循环结构的简单猜数游戏：

package main

import (
	"fmt"
	"math/rand"
	"time"
)

func main() {
	var input, counts int // 定义输入数字与猜对次数

	rand.Seed(time.Now().UnixNano()) // 设置随机数生成器的种子，当前时间 Uninx 格式整数
	guess := rand.Intn(100) + 1      // 随机生成 0 到 100 间的整数

	fmt.Printf("请猜一个 0 到 100 之间的数字：")
	for {
		counts++
		fmt.Scan(&input)
		if input > guess {
			fmt.Println("猜大了，请继续：")
		} else if input < guess {
			fmt.Println("猜小了，请继续：")
		} else {
			fmt.Printf("猜中了，共猜了 %d 次", counts)
			break
		}
	}
}

range 子语句

for range 语句可以用来遍历数据结构：

package main

import "fmt"

func main() {
	numbers := []int{1, 2, 3, 4}
	for i, n := range numbers {
		fmt.Println("index is:", i)  // 分别打印 0，1，2，3
		fmt.Println("number is:", n) // 分别打印 1，2，3，4
	}
}

range 子语句作用类似于迭代器，针对不同的遍历对象，range 会返回不同的值：

遍历对象 第一个返回值 第二个返回值
字符串（string） 索引（index） 索引对应的值，返回类型为字符
数组/切片（array/slice） 索引（index） 索引对应的值
映射（map） 键（key） 键对应的值（value）
通道（channel） 元素（element），通道内的值

对于空映射或切片、空数组、空字符串等情况，for 语句会直接结束。如果数组指定了长度，则 for 会循环执行长度相等的次数：

package main

import "fmt"

func main() {
	listEmpty := []int{}
	for _, x := range listEmpty {
		fmt.Printf("执行结果：%v\n", x) // 不会执行
	}

	list4 := [4]int{1}
	for i, x := range list4 {
		fmt.Printf("索引%d的执行结果：%v\n", i, x) // 索引0的执行结果：1，后面都是零值返回0
	}
}

跳转语句

当程序需要退出当前循环，或需要转去其他分支继续执行的时候，可以使用跳转语句（Jump Statement）。Go 语言中有三个跳转语句：goto、break 和 continue。

同时 Go 语言中可以使用标签，用于给 for、switch 或 select 等流程控制代码打上一个标签，跳转语句配合标签标识符可以方便地跳转到某一处继续执行。

标签名称建议使用大写字母和数字。

break

break 语句是限定转向语句，它能控制程序流程跳出所在的结构，把流程转向所在结构之后。简单来说，break用于打断当前流程控制，提前结束循环：

package main

import "fmt"

func main() {
	for i := 0; i < 10; i++ {
		if i == 5 {
			break // 到 i 等于 5 时即跳出循环
		}
		fmt.Println(i) // 从 0 打印到 4
	}
}

和标签搭配使用，可以跳出指定循环：

package main

import "fmt"

func main() {
L1:
	for i := 0; i < 20; i++ {
		fmt.Println(i)
		if i > 5 {
			switch {
			case i%7 == 1:
				fmt.Println("到达中断点")
				break L1 // 跳出到标签为 L1 的代码块之外
			default:
				fmt.Println("没达到条件")
			}
		}
	}
}

上面没有标签的情况下，循环到 i 值等于 19，在 i 的值为 8 或 15 时都会响应条件 i%7 == 1。打了和第一层循环同级的标签 L1 以后，将会在 i 的值为 8 时即跳出到第一层循环代码块之外。

continue

continue 语句仅能用于 for 循环结构中，它的作用是结束本次循环，但不会终止整个循环，接着进行下一次循环条件判断：

package main

import "fmt"

func main() {
	for i := 1; i < 8; i++ {
		if i%3 == 0 {
			fmt.Println(i, "能被 3 整除")
			continue
		}
		fmt.Println(i) // 分别输出 1 2 4 5 7，跳过打印能被 3 整除的数
	}
}

配合标签，continue 用于跳转到指定代码块位置继续执行任务：

package main

import "fmt"

func main() {
L1:
	for i := 0; i < 20; i++ {
		fmt.Println(i)
		if i > 5 {
			switch {
			case i%7 == 1:
				fmt.Println("到达中断点")
				continue L1
			default:
				fmt.Println("没达到条件")
			}
		}
	}
}

goto

goto 语句需要和标签配合使用，用于无条件跳转到相同函数中的带标签语句。goto 语句不能跳过变量声明，也不能跳进一个代码块的内部：

package main

import "fmt"

func main() {
	fmt.Println("1")
	goto L2
	fmt.Println("无效代码")
L2:
	fmt.Println("直接跳到此处")
}

使用 goto 语句需要谨慎，因为不当的使用会导致代码难以理解和维护，广泛的共识是应当尽量避免使用 goto，只在某些特定场景下才使用 goto 语句处理。例如：在多个嵌套的条件判断中（本身代码就不合理），如果需要统一处理错误，使用 goto 跳转到错误处理代码块可以减少代码重复，使得错误处理更集中和一致。
