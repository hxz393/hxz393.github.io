Go 语言输入输出

发表于 2024-02-27 更新于 2024-07-23 分类于 Go ， 3.核心概念 阅读次数： 本文字数： 2.1k 阅读时长 ≈ 7 分钟

输入输出

标准输入是提供给命令行程序的数据，一般是文件或文本字符串。标准输出是来自程序运行结果的输出，一般打印到终端。此外还有标准错误，是来自程序运行时的错误。这三者在 Linux 系统中以代码 0、1 和 2 指代。

Go 语言用于输入输出的函数，在 fmt 包中实现。

标准输出

数据输出（Output）是计算机对各类输入数据进行处理后，将结果输出到标准输出设备上。用于标准输出的函数有三个：Print、Printf 和 Println。

此外还有将输出写入到 io.Writer 的函数 Fprint、Fprintf 和 Fprintln。将输出结果返回的函数 Sprint、Sprintf 和 Sprintln。

Print

Print 函数的功能是按照系统默认格式字符串和参照表，生成一个打印字符串，然后将其输出：

package main

import "fmt"

func main() {
	a := 100
	b := "bbb"
	c := 1.20
	fmt.Print("变量 a：", a, "\n") // 输出：变量 a：100
	fmt.Print("变量 b：", b, "\n")  // 输出：变量 b：bbb
	fmt.Print("变量 b 和 c：", b, c)   // 输出：变量 b 和 c：bbb1.2
}

Println

Println 函数和 Print 函数功能基本一致，不同的是会在每个元素之间自动加上空格，输出结束后自动换行：

package main

import "fmt"

func main() {
	a := 100
	b := "bbb"
	c := 1.20
	fmt.Println("变量 a：", a)     // 输出：变量 a： 100
	fmt.Println("变量 bc：", b, c) // 输出：变量 bc： bbb 1.2
}

Printf

使用 Printf 函数可以自定义输出数据的格式：

package main

import "fmt"

func main() {
	a := 10
	b := "bbb"
	c := 111
	fmt.Printf("变量 b 和 a：%d, %s\n", a, b) // 输出：变量 b 和 a：10, bbb
	fmt.Printf("变量 c：%#x", c)          // 输出：变量 c：0x6f
}

具体用法参考字符串格式化。

标准输入

数据输入（Input）是将系统外部原始数据通过标准输入设备传输给系统内部，并将这些数据转为系统内部格式的过程。用于输入函数有三个：Scan、Scanf 和 Scanln。

Scan

Scan 函数从标准输入设备读取数据，并将使用空格分割的连续数据顺序存入到参数里。换行也被视为空格。函数调用成功时，返回正确读取的参数的数量 n，否则返回 err：

package main

import "fmt"

func main() {
	var (
		a int
		b float64
		c string
	)
	fmt.Println("等待输入：")
	result, err := fmt.Scan(&a, &b, &c)
	fmt.Println("调用结果：", result)
	fmt.Println("调用错误：", err)
	fmt.Println("输出结果：", a, b, c)
}

如果有任意错误发生，后续的赋值操作会中断。

Scanln

Scanln 函数与 Scan 和 Scanf 类似，但它只用于从标准输入设备中读取单个行的数据，并根据空格将其分割成多个参数值。等同于使用 Scan 函数并手动判断是否读取到了整行数据，所以输入值必须在同一行，不能分行输入。

Scanf

Scanf 函数使用格式字符串来指定输入应该如何被读取和解析。格式字符串中的格式化占位符指定了期望输入的类型和格式，输入需要完全匹配指定的格式：

package main

import "fmt"

func main() {
	var (
		a int
		b float64
		c string
	)
	fmt.Println("等待输入：")
	result, err := fmt.Scanf("%d+%f+%s", &a, &b, &c)
	fmt.Println("调用结果：", result)
	fmt.Println("调用错误：", err)
	fmt.Println("输出结果：", a, b, c)
}

上面函数接受输入的参数，不再是以空格作为分隔符，而是自定义的加号，任何不符合格式的输入都将报错。

系统输入

另一个比较常见的获取输入方式，是从 os.Stdin 系统输入接受内容，存为 io.Reader 对象，然后使用 bufio 包处理：

package main

import (
	"bufio"
	"fmt"
	"os"
)

func main() {
	fmt.Println("请输入：")
	reader := bufio.NewReader(os.Stdin) // 光标接在上面输出后面，捕获输入到变量
	input, _ := reader.ReadString('\n') // 输入结束标记字符
	fmt.Println("输出为：", input)
}

解析参数

flag 包在 Go 语言中主要用于解析命令行参数。通过使用 flag 包，可以方便地处理命令行传递的参数，包括基本的数据类型（如字符串、整数、布尔值等）的解析，以及自定义的命令行参数：

package main

import (
	"flag"
	"fmt"
	"os"
)

func main() {
	// 定义不同类型命令行参数，分别为参数名、默认值、说明文字。返回指针变量
	ip := flag.String("ip", "127.0.0.1", "IP 地址")
	port := flag.Int("port", 8080, "端口号")
	debug := flag.Bool("debug", false, "启用调试模式")

	// 解析命令行参数，如果解析出错，会输出帮助并退出程序
	flag.Parse()
    
    // os.Args 变量为用户输入的命令行参数字符串切片
	fmt.Println("获取到的输入参数：", os.Args)

	// 通过 * 操作符来取这些参数的值
	fmt.Printf("服务启动在 %s:%d\n", *ip, *port)
	if *debug {
		fmt.Println("调试模式已启用")
	}
}

此时使用命令行运行可传递参数，例如：go run .\main.go -port=80，将会输出：服务启动在 127.0.0.1:80。

输入参数

在 Go 语言中命令行参数支持三种标志语法：

  • -flag：仅支持布尔类型。
  • -flag x：仅支持非布尔类型。
  • -flag=x：支持所有类型。

因此运行上面的例子可以这样指定参数：go run .\main.go -ip="127.0.0.1" --port 8080 -debug。参数值中有特殊符号（如 $、&、*、(、)）和空格，最好使用双引号括起来，否则可能出现参数解析异常。

大多数情况下，使用一个连字符和两个连字符的选项不做区别，例如：-all 和 --all 是一样的参数。但是在 PowerShell 中，点「.」用于属性访问和方法调用，如果在没有引号包围参数值的情况下，命令 go run .\main.go -ip=127.0.0.1 会被解析为 main.exe -ip=127 .0.0.1。这种情况下，使用双横杆「–」作为代替才能正常解析。

最后，没有指定的运行参数值会被默认值代替，而输入的参数值类型不匹配会引发错误，并展示程序帮助信息。

绑定变量

更常见的是将用户输入赋予给变量，这时使用 Var 开头的类型方法来初始化：

package main

import (
	"flag"
	"fmt"
)

func main() {
	// 定义变量后，传入变量指针，用户输入保存其中
	var myint int
	var mystr string
	flag.IntVar(&myint, "myint", 000, "默认为 000")
	flag.StringVar(&mystr, "mystr", "", "默认为空")

	flag.Parse()

	// 模拟对变量的处理
	myint *= 100
	mystr += "_modified"

	// 输出处理后的变量值
	fmt.Printf("MyInt: %v\nMyStr: %v\n", myint, mystr)
}

自定义参数类型

可以通过实现 flag.Value 接口来定义自定义参数类型，也就是自定义的类型必须实现 Set 和 String 方法：

package main

import (
	"flag"
	"fmt"
	"strings"
)

// MyFlag 为自定义数据类型
type MyFlag struct {
	value string
}

// String 用于展示参数
func (f *MyFlag) String() string {
	return f.value
}

// Set 用于设置参数，这里故意做了些修改
func (f *MyFlag) Set(value string) error {
	f.value = value + " 设置"
	return nil
}

func main() {
	// 实例化结构体后，传入变量指针
	var myflag MyFlag
	flag.Var(&myflag, "myflag", "custom flag")

	flag.Parse()
	// 模拟对自定义的数据做些修改
	myflag.value = strings.Replace(myflag.value, "设置", "修改", -1)

	fmt.Printf("MyFlag: %s\n", myflag.value)
}

帮助信息

如果输入参数错误，会显示帮助信息：

PS D:\Software\Programming\Go\my\src> go run .\hello.go -port1=80
Usage of C:\Users\ADMINI~1\AppData\Local\Temp\go-build496526514\b001\exe\hello.exe:
  -debug
        启用调试模式
  -ip string
        IP 地址 (default "127.0.0.1")
  -port int
        端口号 (default 8080)
exit status 2

上面帮助信息是自动生成的，也可以通过 flag.Usage 手动设置帮助信息：

package main

import (
	"flag"
	"fmt"
	"os"
)

func main() {
	// 仅示范使用格式
	flag.Usage = func() {
		usageTxt := `一些多行说明文字`
		// 将消息打印到标准错误输出
		_, _ = fmt.Fprintf(os.Stderr, usageTxt)
	}

	f := flag.Float64("f", 1.0, "输入浮点数")
	flag.Parse()
	fmt.Println("输入为：", f)
}

在输入错误，或者使用 --help 参数运行程序时，都会显示自定义的帮助文本。

子命令

flag 包通过 FlagSets 提供对参数子命令的支持。例如：psCmd := flag.NewFlagSet("ps", flag.ExitOnError)，代表子命令名为 ps，错误时以状态码 2 退出：

package main

import (
	"flag"
	"fmt"
	"os"
)

func main() {
	// 创建一个新的 FlagSet，命名为 ps，并设置常用错误处理策略
	psCmd := flag.NewFlagSet("ps", flag.ExitOnError)
	// 为 ps 子命令定义参数
	all := psCmd.Bool("all", false, "展示所有容器")

	// 检测是否有子命令
	if len(os.Args) < 2 {
		fmt.Println("缺少 ps 子命令")
		os.Exit(1)
	}

	// 根据子命令解析参数
	switch os.Args[1] {
	case "ps":
		psCmd.Parse(os.Args[2:])
		fmt.Println("子命令 'ps':")
		fmt.Printf("  all: %v\n", *all)
	default:
		fmt.Printf("未知命令: %s\n", os.Args[1])
		os.Exit(2)
	}
}
