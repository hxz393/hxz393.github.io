Go 语言类型：浮点型

发表于 2024-04-15 更新于 2024-07-23 分类于 Go ， 2.类型系统 阅读次数： 本文字数： 1.3k 阅读时长 ≈ 4 分钟

浮点型定义

浮点型数据（Floating-point-number）用于表示包含小数点的数字，也称为实数（Real-number）。

表示方法

浮点型数据在 Go 语言中表示方式有两种：

  • 十进制小数：由数字和小数点组成，例如：0.210。
  • 指数形式：由数字和字母 e 组成，例如表示 1234.5 指数形式为：12.345e2 或 1.2345e3。

浮点类型

Go 语言中浮点型分为 32 位浮点型+

类型 字节长度 精确位数
float32 4 小数点后 7 位
float64 8 小数点后 15 位

浮点结构

浮点数在计算机中的表示基于科学记数法，在内存中按指数形式储存，分为三部分：符号位（Sign），指数（Exponent），和尾数（Mantissa）：

  • 符号位：用于表示数值正负的 1 个单独位。在二进制中，0 通常表示正数，1 表示负数。
  • 指数：用于表示浮点数的范围。不是直接存储实际的指数值，而是存储一个偏移量（或称为偏移指数）。例如，在 float32 中，指数部分占用 8 位，有一个固定的偏移量 127。这种表示方法允许表示从 2^-127 到 2^128 的范围，覆盖了非常大的数值区间。例如，指数部分的二进制是 01111111（即 127），它表示的实际指数是 0。如果指数部分是 10000000（即 128），表示的实际指数是 1，以此类推。
  • 尾数：有时也称为有效数字或分数部分，用于表示浮点数的精度。尾数部分包含了数值的实际数字信息，但它通常表示为 1.xxxx 的形式，其中 1 是隐含的（对于规格化的数），不占用存储空间。尾数部分存储的是 xxxx，也就是小数部分。例如，在 float32 中，尾数部分占用了 23 位，加上隐含的 1，总共可以提供 24 位的精度。

在十进制中，科学记数法是这样的形式：±d.d...d × 10^n，其中 d.d...d 是尾数部分，基数为 10，而 n 是指数部分。在二进制中，基数变成了 2。假设有一个 32 位的 IEEE 754 浮点数：

  • 符号位：0
  • 指数：10000011（十进制 131，表示实际指数 4）
  • 尾数：.1010000...（省略后面许多 0，注意隐含的前导 1）

这个数的值是 1.1010000... x 2^4，转换成十进制大约是 1.625 x 16 = 26。

浮点数的这种存储方式使得它能够表示非常宽范围的数值，但由于指数和尾数部分的位数是有限的，有时候会导致精度问题。例如，0.624 的二进制表示开始为 .100111111...，后面的 1 会继续下去，直到达到精度限制。这也是某些十进制小数在计算机中不能被精确表示的原因：当这些数转换为二进制时，可能会产生一个无限循环小数，但由于存储空间的限制，这个无限循环会在某一点被截断，从而导致精度的损失。

赋值操作

浮点数字面量自动类型推断的默认类型为 float64：

package main

import "fmt"

func main() {
	// 定义一个 float32 类型的变量，初始化为 1
	var explicitFloat float32 = 1
	// 自动推导类型为 float64 的变量，初始化为 2.10
	inferredFloat := 2.10

	// 打印两个变量的值
	fmt.Println("显式声明的 float32 类型值：", explicitFloat)
	fmt.Println("推导得到的 float64 类型值：", inferredFloat)

	// 打印两个变量的类型
	fmt.Printf("explicitFloat 的类型为 %T\n", explicitFloat)
	fmt.Printf("inferredFloat 的类型为 %T\n", inferredFloat)
}

舍入误差

注意，当 float64 类型变量值的小数位数超过 15 位时，超出的数字部分会被舍去。同理，float32 类型的变量在小数位数超过 7 位时会发生截断，因此在比较两个浮点数时不应使用 == 或 != 运算符。对于需要高精度的科学计算，应当使用 math 标准库中的相关功能来避免舍入误差：

package main

import "fmt"

func main() {
	// 定义两个 float64 类型的变量，尝试赋予 b 超出 float64 精度的值
	a, b := 1.0, 1.0000000000000001

	// 检查两个浮点数是否相等。由于 float64 的精度限制，b 的实际值会被视为与 a 相同
	if a == b {
		fmt.Printf("a 等于 b\na 的值为：%v\nb 的值为：%v\n", a, b)
	} else {
		fmt.Println("a 不等于 b")
	}
}

这种情况下导致的浮点数错误的极限值称为机器精度（machine epsilon）。要避免这类舍入误差，最好的方法是避免在精确度要求高的场合使用浮点类型。

除数为 0

可以对浮点数进行除了 % 以外的任意数学和比较运算，当浮点数 0 作为除数时，结果如下：

package main

import "fmt"

func main() {
	// 正数除以浮点数 0，结果为正无穷大
	var a float64 = 1.1
	var b float64 = 0
	fmt.Println(a / b) // 输出：+Inf

	// 负数除以浮点数 0，结果为负无穷大
	a = -1.1
	fmt.Println(a / b) // 输出：-Inf

	// 浮点数 0 相除，结果为 Not a Number
	a = 0
	fmt.Println(a / b) // 输出：NaN
}

这些运算结果遵循 IEEE 754 浮点数标准。
