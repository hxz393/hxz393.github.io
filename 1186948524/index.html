Go 语言并发

发表于 2024-06-17 更新于 2024-07-23 分类于 Go ， 3.核心概念 阅读次数： 本文字数： 5.9k 阅读时长 ≈ 20 分钟

并发和并行

并发（Concurrent）和并行（Parallel）是计算机科学中两个关键概念，描述系统如何同时处理多个任务。简单来说，并发是同时处理很多事情，而并行是同时做很多事情。

并发

并发是指系统能够处理多个任务的能力，但不等于这些任务是在同一时刻实际执行。

在并发模型中，单个处理器可能会在多个任务之间迅速切换，使得每个任务都有机会在不同的时间片内执行，只是由于切换速度很快，给人一种同时执行的错觉。常见的 Web 服务器使用并发来处理多个客户端请求。

并行

并行处理是指多个处理器或多核处理器同时执行多个计算任务。

在多核或多处理器系统中，每个核心可以同时运行一个独立的任务。多个任务在同一时刻真正地并行运行，可以显著提高计算效率。常见的科学计算程序，将大型数组的处理分散到不同的核心上，每个核心并行处理数组的一部分以加速计算。

并发基础

从操作系统层面来看，程序并发执行的基础是进程（Process）、线程（Thread）和协程（Coroutine）。

Go 语言的一大特色就是原生支持类协程，由 Go 语言运行时负责全面管理被创建的所有协程并为其分配执行时间。

进程

进程是操作系统分配资源和调度的基本单位，每个进程都有独立的地址空间，包括内存、文件资源和其他执行所需的资源。进程之间资源隔离，通过进程间通信（IPC）机制如管道、信号、共享内存等进行通信和数据交换。

由于进程包含完整的执行环境和资源，管理起来简单，但是系统开销大，能同时创建的进程数量有限。

线程

线程是操作系统能够进行运算调度的最小单位，由进程创建，是进程中的实际运作单位。一个进程可以包含一个以上线程，这些线程可以共享父进程的资源，如内存和文件句柄等。线程之间的通信可以直接通过读写进程中的共享数据来进行，但可能导致数据的同步问题。

与进程比，线程的创建、启动和上下文切换的开销要小很多，可以提高系统吞吐量。

协程

协程是一种用户态的轻量级「线程」，调度完全由用户控制，而非操作系统内核。与线程不同，协程在单线程内执行，并利用任务切换的方式来实现多任务的并发执行，通常不涉及多线程的数据竞态问题，通信更简单、更安全。

协程可以进行非常快速的创建、切换和销毁，开销远小于线程。通常操作系统默认限制线程和进程最多不能超过 1 万个，协程可以轻松创建上百万个而不导致系统资源耗尽。

并发通信

并发程序间最大的问题是通信，最常见的并发通信模型是共享内存（Shared Memory）和消息机制（Message Communication Mechanism）。

Go 语言以并发编程作为核心优势，所以在处理并发模型时，以消息传递机制作为通信方法。Go 语言的并发同步模型叫做通信顺序进程（CSP，Communicating Sequential Processes），goroutine 是实现并发的基本单元，而 channel 是在 goroutine 之间进行通信的主要方式。

共享内存

共享内存是在多个并发执行的线程或进程之间共享和访问内存数据结构的通信方式。共享内存有着非常快速的数据访问和高效的数据交换，因为所有通信的单位都可以直接访问相同的内存地址空间。

但是共享内存需要细致的同步机制来控制对共享数据的访问，防止数据竞争和其他并发问题。此外，共享内存在分布式系统中扩展性不佳，或者非常复杂。

消息传递

消息传递机制是另一种并发通信模型，通过发送和接收消息来进行进程或线程间的通信，而不是直接共享内存。核心特点类似于进程，进程各自独立不会互相影响：

  • 对线程间共享状态的各种操作都被封装在消息中，线程间发送消息时对状态进行复制，确保每个线程都有自己的状态副本，从而避免不同数据竞争和冲突。
  • 在消息传递的边界上交出这个状态的所有权。发送方交出对该状态的所有权给接收方，接收方可以自由地使用和修改该状态。

消息传递机制减少了并发控制的复杂性，更适合分布式系统和微服务架构，可以很容易地在网络上扩展，常见实现方法有消息队列、管道、网络信号等。

消息传递机制主要面对的问题是网络延迟和额外的数据复制开销。

并发控制

标准库 runtime 包提供了一些与并发相关的控制函数，以及底层系统操作相关的功能。

Gosched

runtime.Gosched 函数让出当前 goroutine 的执行权，给其他的 goroutine 运行，类似其他语言的函数迭代器（Yield）。这是一种主动调度方式，让出 CPU 时间，但并不会挂起或停止当前 goroutine，而是将当前 goroutine 放回到调度队列中，使得调度器可以选择其他的 goroutine 进行执行。

当一个 goroutine 需要执行长时间的或计算密集型任务时，使用 Gosched 可以避免它占用 CPU 时间过长，给其他 goroutine 执行的机会。

在大多数情况下，Go 调度器已足够智能，能够高效地管理多个 goroutine，不需要手动干预。过度使用 Gosched 可能会导致性能下降，因为频繁的调度会增加开销。

Goexit

runtime.Goexit 函数用于立即终止当前 goroutine 的执行，但任何已注册的 defer 语句都会被执行：

package main

import (
	"fmt"
	"runtime"
	"time"
)

func main() {
	// 启动一个 goroutine
	go func() {
		defer fmt.Println("清理资源: 第一个 defer 被调用")
		defer fmt.Println("清理资源: 第二个 defer 被调用")

		fmt.Println("goroutine 开始执行")

		// 假设在某个点决定退出这个 goroutine
		if true { // 这里的 true 代表某个运行时条件
			fmt.Println("决定退出 goroutine")
			runtime.Goexit() // 终止 goroutine
		}

		// 这行代码将不会被执行
		fmt.Println("goroutine 继续执行")
	}()

	// 等待足够的时间以确保 goroutine 可以执行
	time.Sleep(1 * time.Second)
	fmt.Println("主程序结束")
}

NumCPU

runtime.NumCPU 用于获取处理器可用最大线程数：

package main

import (
	"fmt"
	"runtime"
)

func main() {
	fmt.Println("CPU 线程数：", runtime.NumCPU())
}

在 Go 1.5 版本以前，默认是以单核心执行，因此需要配合 runtime.GOMAXPROCS 函数来利用所有核心。从 Go 1.5 版本之后，默认会执行 runtime.GOMAXPROCS(runtime.NumCPU()) 设置，一般不需要手动配置。

GOMAXPROCS

runtime.GOMAXPROCS 用于手动设置可同时执行的用户级线程的最大数目，返回之前的设置值。默认是 CPU 核心数，但可以被调整以提高并发性或减少上下文切换：

package main

import (
	"fmt"
	"runtime"
)

func main() {
	// 设置 GOMAXPROCS 为 4
	prev := runtime.GOMAXPROCS(4)
	fmt.Println("Previous GOMAXPROCS:", prev)

	// 查询当前的 GOMAXPROCS 值
	current := runtime.GOMAXPROCS(0)
	fmt.Println("Current GOMAXPROCS:", current)
}

NumGoroutine

runtime.NumGoroutine 函数返回正在执行和排队的任务总数：

package main

import (
	"fmt"
	"runtime"
	"time"
)

func main() {
	// 启动 5 个示例 goroutine
	for i := 0; i < 5; i++ {
		go worker(i)
	}

	// 给 goroutines 一些时间启动
	time.Sleep(2 * time.Second)

	// 打印当前活跃的 goroutine 数量，加上主协程一共 6 个
	fmt.Printf("当前活跃的 goroutine 数量: %d\n", runtime.NumGoroutine())

	// 给 goroutines 一些时间运行
	time.Sleep(3 * time.Second)

	// 再次打印当前活跃的 goroutine 数量，只剩下主协程 1 个
	fmt.Printf("再次检查后的 goroutine 数量: %d\n", runtime.NumGoroutine())
}

// worker 模拟一个工作 goroutine
func worker(id int) {
	fmt.Printf("goroutine %d 开始工作...\n", id)
	time.Sleep(3 * time.Second) // 模拟耗时的工作
	fmt.Printf("goroutine %d 完成工作.\n", id)
}

GC

runtime.GC 用于手动强制进行一次垃圾回收，以清理未使用的内存。调试的时候可能会用到：

package main

import (
	"runtime"
)

func main() {
	runtime.GC() // 强制进行垃圾回收
}

ReadMemStats

runtime.ReadMemStats 可以读取当前内存的使用情况和垃圾回收的统计数据：

package main

import (
	"fmt"
	"runtime"
)

func main() {
	var m runtime.MemStats
	runtime.ReadMemStats(&m)
	fmt.Println("Allocated memory:", m.Alloc)
	fmt.Println("Total memory (from system):", m.Sys)
	fmt.Println("Number of GC:", m.NumGC)
}

goroutine

goroutine 是由 Go 运行时管理的轻量级线程。每个 goroutine 在启动时会分配一小块内存作为其初始的栈空间，随着程序的运行，栈空间可能会根据需要动态扩展，但在操作系统看来，一百个 goroutine 可能仅运行在很少的几个线程上。

启动一个 goroutine 只需在函数调用前加上关键字 go，直到函数返回时自动结束，返回值会被忽略：

package main

import (
	"fmt"
	"time"
)

func say(s string) {
	time.Sleep(10000 * time.Millisecond)
	fmt.Println(s) // 期望等待 10 秒后一次性输出结果
}

func main() {
	for i := 0; i < 5; i++ {
		go say("world")
	}
	fmt.Println("主程序运行完成")
}

上面例子没有任何输出，原因是 mian 函数启动了 5 个 goroutine 后直接返回，打印完信息就退出了，并不会等待 goroutine 中的 say 函数执行结束打印信息。在操作系统控制的多进程或多线程编程中，父进程会等待线程执行结束后再退出。而 Go 语言不守此规则，它使用其他技术来同步 goroutine 的执行。

channel

channel 通道是 Go 中用于多个 goroutine 之间的通信的管道，可以通过通道发送和接收类型相同的数据。通道从设计上确保同一时刻只有一个 goroutine 能从中接受数据，避免了使用互斥锁的问题。

声明通道

在 Go 语言中，通道是引用类型，只能传递一种类型的值，但传递的类型没有限制。声明通道变量的格式：var 通道名 chan 传递类型 或者使用 make 函数直接声明并初始化：

package main

import "fmt"

func main() {
	messages := make(chan string) // 创建一个通道类型变量，传递字符串

	var chanInt chan []int //  创建一个没有初始化的 nil 值通道变量，传递整数切片

	fmt.Printf("%T, %v", messages, chanInt) // 输出类型和值：chan string, <nil>
}

一般要使用 make 函数来初始化创建通道，对 nil 值的通道读写操作会导致永久阻塞。

收发数据

通道使用专用运算符 <- 来接受或发送数据。数据发送到通道的语法为：通道 <- 值，从通道读取数据的语法为：变量名 := <-通道。下面是一个简单示例：

package main

import "fmt"

func main() {
	messages := make(chan string) // 创建一个通道类型变量，传递字符串

	go func() { messages <- "hello" }() // 启动一个 goroutine，发送消息到通道

	msg := <-messages // 从通道取出消息到变量
	fmt.Println(msg)
}

还有一种常见的读取通道语句：<-通道，从通道获取数据，但不赋予给变量，直接丢弃数据。

通道阻塞

如果通道中没有数据，那么从通道中读取数据会导致阻塞，直到通道中被写入数据。同样，通道有写入但没有读取也会导致阻塞，就像管道的概念，管道两头必须通畅：

package main

import (
	"fmt"
	"time"
)

func main() {
	channel := make(chan string)

	go func() {
		// 发送协程先启动运行，然后开始等待 5 秒钟。主协程不会等待发送协程跑完再运行，而是同时运行下去
		time.Sleep(5 * time.Second)
		channel <- "接力棒"    // 5 秒等待完毕，发送数据到无缓冲通道
		fmt.Println("发送完成") // 发送完数据后立马打印信息，结束协程
	}()

	// 发送协程那边在等待时间，主协程这边同步运行
	fmt.Println("等待接收接力棒") // 最先打印的信息
	received := <-channel // 运行到这阻塞了，等待通道写入数据。一旦通道数据准备好，立即执行读取操作
	fmt.Println("接收到:", received) // 读取完毕后打印出数据

	go func() {
		// 再次写入通道，没有延迟等待。但由于接收者没准备好，所以发送协程在这里阻塞了，直到接收者准备好时数据才会发送
		channel <- "另一个接力棒"
		fmt.Println("第二次发送完成") // 通道发送完毕后，才执行打印
	}()

	time.Sleep(3 * time.Second) // 模拟耗时操作，等待 3 秒
	fmt.Println("第二次接收开始")      // 第二次最先打印的信息
	// 接收者准备好后，流程跳到发送协程写入通道并打印信息，之后回到这里从通道中读取数据并赋值
	received = <-channel
	fmt.Println("第二次接收到:", received) // 接受到数据后立即打印
	time.Sleep(1 * time.Second)      // 加入等待时间，主协程后于发送协程结束，才能确保每次发送协程打印动作的完成
}

第一次发送和接收流程：

  1. 发送协程启动后，首先执行休眠以模拟延时；
  2. 在发送协程休眠期间，主协程打印「等待接收接力棒」信息，并阻塞等待数据；
  3. 发送协程延时结束后，向无缓冲通道发送字符串「接力棒」，执行打印「发送完成」信息；
  4. 主协程从通道接收到「接力棒」后，继续执行并打印「接收到: 接力棒」，显示数据已被接收。

第二次发送和接收流程：

  1. 发送协程尝试立即向同一个无缓冲通道发送字符串「另一个接力棒」。由于通道无缓冲，这个发送操作会被阻塞，直到主协程准备接收数据；
  2. 主协程执行休眠以模拟延时；
  3. 主协程延时结束后，打印「第二次接收开始」信息，并准备好接收数据。此操作会解除发送协程的阻塞，开始传输数据；
  4. 发送协程数据发送成功，打印「第二次发送完成」信息；
  5. 主协程接收到数据后，打印「第二次接收到: 另一个接力棒」信息；

关闭通道

永久关闭通道使用内置 close 函数来操作，之后通道禁止写入，可以继续读取通道，直到通道数据耗尽，返回零值。可以通过读取通道返回值的特殊的语法来检查通道是否关闭，只要接收端使用循环接收数据，就必须显式关闭通道：

package main

import (
	"fmt"
)

func main() {
	// 创建一个字符串通道
	channel := make(chan string)

	// 启动生产者协程
	go producer(channel)

	// 消费者从通道接收数据
	consumer(channel)
}

// 生产者函数，负责发送数据到通道
func producer(channel chan string) {
	// 发送三个字符串到通道
	channel <- "Hello"
	channel <- "World"
	channel <- "Golang"

	// 发送完成后关闭通道
	close(channel)
}

// 消费者函数，从通道读取数据
func consumer(channel chan string) {
	// 循环读取通道数据直到通道关闭
	for {
		// 非阻塞方式尝试从通道读取数据
		msg, ok := <-channel
		if !ok {
			// 如果通道已关闭，ok 会为 false
			fmt.Println("通道已关闭!")
			break
		}
		// 打印接收到的消息
		fmt.Println("接收到数据:", msg)
	}
}

也能使用 for range 循环来处理通道中的数据，但发送端必须显式关闭通道，否则接收端会陷入死循环：

package main

import (
	"fmt"
)

func main() {
	// 创建一个字符串通道
	channel := make(chan string)

	// 启动一个匿名协程作为生产者
	go func() {
		// 发送三个字符串到通道
		for _, i := range []string{"Hello", "World", "Golang"} {
			channel <- i
		}

		// 发送完成后关闭通道。如果没有这句，将会引发错误：fatal error: all goroutines are asleep - deadlock!
		close(channel)
	}()

	// 使用 for range 循环从通道读取数据，通道关闭后字段结束遍历
	for msg := range channel {
		// 打印每个从通道接收到的消息
		fmt.Println("接收到数据:", msg)
	}

	// 当通道关闭且消息都已经读取完毕，for range 循环会自动结束
	fmt.Println("通道已关闭!")
}

只有发送端可以关闭通道，由接收端获得关闭状态。

单向通道

既能发送也能接收数据的叫做双向通道（Duplex-channel），还可以将通道指定为单向通道（Simplex-channel），只能用于接收或发送数据。

只写通道定义格式：var 通道名 chan<- 类型，只读通道定义格式：var 通道名 <-chan 类型。单独定义一个单向通道没有实际应用意义，所以单向通道常由双向通道转换而来，反过来不能转换：

package main

import (
	"fmt"
)

func main() {
	// 创建一个双向通道
	channel := make(chan string)

	// 由双向通道创建单向只写通道
	sendOnly := chan<- string(channel)

	// 创建单向只读通道，另一种声明方式
	var receiveOnly <-chan string = channel

	// 启动一个匿名协程向 sendOnly 发送数据
	go func() {
		sendOnly <- "Hello"
		sendOnly <- "World"
		close(sendOnly) // 关闭通道由发送方进行，且关闭的是原始的双向通道
	}()

	// 从 receiveOnly 读取数据
	receiveData(receiveOnly)
}

// receiveData 函数接收一个只读通道参数
func receiveData(ch <-chan string) {
	for msg := range ch {
		fmt.Println("接受到:", msg)
	}
	fmt.Println("通道关闭")
}

单向通道主要用于在函数或方法参数中明确通道的用途，确保函数或方法仅能通过该通道发送或接收数据：

package main

import (
	"fmt"
)

func main() {
	channel := make(chan int) // 创建双向通道

	go producer(channel) // 传递只写视图
	consumer(channel)    // 传递只读视图
}

// 生产者只能向通道发送数据
func producer(sendOnly chan<- int) {
	for i := 0; i < 5; i++ {
		sendOnly <- i
	}
	close(sendOnly)
}

// 消费者只能从通道接收数据
func consumer(receiveOnly <-chan int) {
	for num := range receiveOnly {
		fmt.Println("接受到:", num)
	}
}

// 接收双向通道参数写法
func controller(message chan int) {
	for i := 0; i < 5; i++ {
		message <- i
	}
	close(message)
}

缓冲通道

上面使用的通道是同步无缓冲（Buffer）的，在需要持续传输大量数据的时候，可以使用有缓冲通道，允许接收者准备接收前在缓冲区存储一定数量的数据，达到消息队列的效果。

定义缓冲通道需要在创建通道时指定缓冲区大小： make(chan 类型, 缓冲区大小) ，缓冲区大小指定能储存消息的数量，而不是内存大小，可以通过 cap 函数来查询。这样在缓冲未写满的情况下，不会阻塞发送操作；在缓冲未读完之前，也不会阻塞接收操作：

package main

import (
	"fmt"
	"time"
)

func main() {
	// 创建一个缓冲通道，缓冲大小为 3
	channel := make(chan int, 3)
	fmt.Printf("通道容量 %v\n", cap(channel))

	// 生产者协程
	go func() {
		// 连续发送 5 个整数到通道，前 3 个由于通道有足够的缓冲不会阻塞。
		for i := 1; i <= 5; i++ {
			fmt.Printf("正在发送 %d\n", i)
			// 当通道的缓冲区被填满后发生阻塞，直到消费者开始从通道接收数据并释放缓冲区空间
			channel <- i
			fmt.Printf("已发送 %d\n", i)
		}
		close(channel)
	}()

	// 消费者：主协程接收数据
	time.Sleep(2 * time.Second) // 延迟接收，在缓冲没满前发送方都不会阻塞
	// 数据一被消费者接收后，生产者立马能发送新数据到通道
	for num := range channel {
		fmt.Printf("已接收 %d，通道中元素数量 %v\n", num, len(channel))
		time.Sleep(1 * time.Second) // 模拟数据处理时间
		fmt.Printf("已处理 %d\n", num)
	}
}

随机选择

在 Go 语言中，select 语句用于监控多个通道操作并从中进行选择，配合超时机制将阻塞式通信转为非阻塞式，以提高通信效率。

select 语句的用法和 switch 语句非常相似，不过是把关键字 switch 变为 select，并且 case 语句必须是一个通道操作。如果没有 default 分支，select 会阻塞直到至少一个通道操作可以进行。如果有 default 分支，当所有通道操作都不能进行时，将执行 default 分支，实现非阻塞操作：

select {
case <-chan1:
    // 如果 chan1 准备好了接收，则执行这里的代码
case chan2 <- value:
    // 如果 chan2 准备好了发送（可写，即有足够缓冲空间或有接收方等待接收数据），则立即发送数据到通道，并执行这里的代码
default:
    // 如果以上都没有准备好，执行default分支
}

此外，select 语句强调随机选择分支去判断，而不是按从上到下顺序执行。如果多个条件同时满足，也是随机选择一个执行，以保证公平性：

package main

import (
	"fmt"
)

func main() {
	chan1 := make(chan int)
	chan2 := make(chan int, 3)
	for {
		select {
		case msg := <-chan1: // 无法满足，因为没有发送者
			fmt.Println("Received", msg)
		case chan2 <- 1: // 缓冲数量为 3，因此可以发送 3 次
			fmt.Println("Sent 1 to chan2")
		default: // 第 4 次循环必定被选择，退出程序
			fmt.Println("Neither send nor receive were ready")
			//break // 打断只能退出 select 代码块，无法退出 for 死循环
			return
		}
	}
	fmt.Println("Done") // 无法访问的代码
}

超时机制

在 Go 语言并发编程中，超时机制用于解决通信死锁：如果通信的双方在超时时间内没有处理完任务，则终止任务，返回对应的超时信息。

具体做法是使用 select 语句结合 time.After 函数创建超时处理。time.After 函数用于在指定时间结束后，自动发送时间信息。如果在超时时间内有任何一个 case 可以执行，select 就会执行那个 case，而不会执行超时的 case，然后重新计算超时时间：

package main

import (
	"log"
	"time"
)

func main() {
	chan1 := make(chan string)
	chan2 := make(chan string)

	// 程序启动 5 秒后发送数据到通道
	go func() {
		time.Sleep(5 * time.Second)
		chan1 <- "数据从 chan1"
	}()

	// 程序启动 1 秒后发送数据到通道
	go func() {
		time.Sleep(1 * time.Second)
		chan2 <- "数据从 chan2"
	}()

	// 同时有 3 个 goroutine 运行，一同开始计时等待
	for i := 0; i < 4; i++ {
		select {
		case msg1 := <-chan1: // 通道 1 接收到数据前会先超时 1 次
			log.Println("接收到:", msg1)
		case msg2 := <-chan2: // 通道 2 最先接收到数据
			log.Println("接收到:", msg2)
		// 第 4 次循环时，通道已经永远没有数据再写入
		case <-time.After(3 * time.Second):
			log.Println("超时，没有数据接收")
		}
	}
}

退出通道

在使用无限制阻塞式 select 语句的场景，想要手动控制退出通道时机，可以添加一个退出通道。通过向退出通道发送消息来激活结束语句相关代码，从而停止阻塞。

退出通道通常命名为 stop 或 quit：

package main

import (
	"fmt"
	"time"
)

func main() {
	stop := make(chan bool)

	go func() {
		// 启动后先等待 10 秒后，向退出通道发送数据
		time.Sleep(10 * time.Second)
		stop <- true
		close(stop)
	}()

	for {
		select {
		case <-time.After(3 * time.Second):
			// 10 秒钟内一直打印输出
			fmt.Println("超时，没有数据被接收")
		case <-stop:
			// 通道接收到数据后激活下面返回代码
			msg := <-stop
			fmt.Println("退出通道：", msg)
			// 用返回直接退出函数
			return
		}
	}
}
