Go 语言类型：接口

发表于 2024-03-18 更新于 2024-07-23 分类于 Go ， 2.类型系统 阅读次数： 本文字数： 2.2k 阅读时长 ≈ 7 分钟

接口定义

Go 语言中的接口（Interface）是用来定义行为的一种抽象结构类型。接口包含一组方法（方法集）声明，但不包含实现代码和数据字段，属于引用类型。定义接口类型语法结构如下：

type 接口类型 interface {
    方法1(参数列表1) 返回类型1
    方法2(参数列表2) 返回类型2
    ...
}

Go 语言中的接口都很简短，通常不超过 3 个方法。接口类型的名称常以 er 作为后缀结尾，接口名和里面的方法名首字母决定了是否可以被外部导入。

接口在 Go 语言中的作用体现在以下方面：

  • 多态性：接口为不同的类型提供统一的方法集调用方式，不同类型只要实现了接口中定义的方法即可。可以使用接口变量来调用实现了该接口的任何类型的实例，从而实现多态。
  • 低耦合：接口使得代码的依赖关系简化，具体类型的实现和使用可以分离，可以随时定义或移除接口。
  • 契约设计：接口定义了一个标准或协议，所有实现接口的类型都必须遵守这个标准。

由于接口是一种抽象类型，因此不能被直接实例化。但是，可以创建实现了该接口的具体类型的实例，并将该实例赋给接口类型的变量，以此达成接口类型的实例化。

接口一般定义在调用的包中。

实现接口

类型不需要显式声明它实现了某个接口，接口的实现是隐式的，一个类型只要实现了接口中的所有方法，就自动实现了该接口。这也被称为非侵入式设计，通过定义接口来分离抽象和具体实现，代码依赖于接口而不是具体实现，有助于降低代码间的耦合度：

package main

import (
	"fmt"
	"strings"
)

// Animal 接口定义了一个 Speak 方法，该方法返回字符串
type Animal interface {
	Speak() string
}

type Human interface {
	Talk() string
}

// Dog 结构体代表狗类型
type Dog struct{}

// Dog 通过实现 Speak 方法满足 Animal 接口的要求
func (d Dog) Speak() string {
	return "Woof"
}

// Dog 类型的 Run 方法是特有的，与 Animal 接口无关
func (d Dog) Run() string {
	return "Dog Run"
}

// Cat 猫的类型为整数
type Cat int

// Speak 方法实现了 Animal 接口。在 Cat 类型中，Speak 方法返回字符串 "Meow Meow Meow "。
func (c Cat) Speak() string {
	return strings.Repeat("Meow ", 3)
}

func main() {
	// 声明了一个 Animal 类型的变量 animal
	var animal Animal
	// 将结构体实例赋值给接口变量
	animal = Dog{}
	// 调用且只能调用接口中规定的方法，不能调用 Run 方法
	fmt.Println(animal.Speak())
	// 等同于实例化类型再调用类型方法
	cat := Cat(1)
	fmt.Println(cat.Speak())
	// 实例数据类型不需关心，只要实现了接口就可以赋值给接口
	animal = cat
	fmt.Println(animal.Speak())

	//var human Human = Dog{} // Dog 没实现 Human 接口的 Talk 方法，编译器不通过
}

在上面的例子中，Dog 和 Cat 类型方法 Speak 的签名与接口 Animal 要求的一致，所以它们隐式地实现了 Animal 接口，无论类型几何或实现了多少其他的方法。这种设计允许类型除被用在任何期望该接口的场景中外，还可以有其它额外能力。

虽然 Go 语言能自动处理方法调用时的引用和解引用，但对于接口的实现，关键在于类型的方法集是否包含了该接口所要求的所有方法，值类型和指针类型需要区分：

  • 值类型：只包括所有使用值接收者声明的方法。
  • 指针类型：包括所有使用指针接收者和值接收者声明的方法。

简单来说，如果类型的非指针版本能满足接口，那么它的指针版本也能够满足：

package main

import (
	"fmt"
)

type Shape interface {
	Area() float64
	Perimeter() float64
}

type Square struct {
	Side float64
}

// Area 方法使用值接收者
func (s Square) Area() float64 {
	return s.Side * s.Side
}

// Perimeter 方法使用指针接收者
func (s *Square) Perimeter() float64 {
	return 4 * s.Side
}

func main() {
	s := Square{Side: 5}
	p := &s

	// 通过值调用方法
	areaVal := s.Area()
	perimeterVal := s.Perimeter() // 会自动取址，需要值能寻址

	// 通过指针调用方法
	areaPtr := p.Area() // 自动解引用值
	perimeterPtr := p.Perimeter()

	var i Shape = &s     // i 可以是一个 *Square 指针实例
	areaIntf := i.Area() // 正确，通过接口调用 Area
	//var j Shape = s // 报错，因为 s 值实例没有 Perimeter 方法，不能赋值给接口

	fmt.Println(areaVal, areaPtr, areaIntf, perimeterVal, perimeterPtr)
}

值类型不能包含指针接收者的方法，是为了保持方法调用的语义一致性：当方法指定指针接收者时，表示能够直接修改原始数据的内存，可能会修改其接收者的值。如果将这样的方法包含在值类型的方法集中，那么在语义上会有冲突，因为值类型传递到方法的是该值的副本，要保证不改动原始值。

使用接口

接口被实现后，可以通过接口类型调用其方法，也可以用于函数的参数或者返回类型。

使用接口步骤一般分为：定义接口变量、实例化类型、将满足接口的类型实例赋值给接口变量、通过接口变量的方法来调用。下面示例合并了接口赋值的步骤：

package main

import "fmt"

// Speaker 接口定义 Speak 方法
type Speaker interface {
	Speak() string
}

// Person 结构体代表人
type Person struct {
	Name string
}

// Speak 方法实现了 Speaker 接口，使 Person 实例能通过 Speaker 接口调用
func (p Person) Speak() string {
	return fmt.Sprintf("Hello, my name is %s", p.Name)
}

func main() {
	// 标准写法
	//var speaker Speaker
	//p := Person{Name: "Bob"}
	//speaker = p

	// 实例化结构体并赋值给 Speaker 接口变量
	var speaker Speaker = Person{Name: "Bob"}

	// 通过 Speaker 接口调用 Speak 方法
	fmt.Println(speaker.Speak())
}

接口类型作为函数参数使用也很常见：

package main

import (
	"fmt"
	"math"
)

type Shaper interface {
	Area() float64
}

type Circle struct {
	Radius float64
}

func (c Circle) Area() float64 {
	return math.Pi * c.Radius * c.Radius
}

type Square struct {
	Side float64
}

func (s Square) Area() float64 {
	return s.Side * s.Side
}

func printArea(sh Shaper) { // 接口类型作为函数参数
	fmt.Println("Area:", sh.Area()) // 直接调用接口方法
}

func main() {
	c := Circle{Radius: 5}
	s := Square{Side: 4}
	printArea(c) // 结构体实例满足接口才能作为参数 
	printArea(s)
}

这种根据同一种类型在不同的示例上表现出不同的行为，叫做多态。多态性可以在功能新增或修改时，不必改动依赖接口的代码。

空接口

空接口 interface{} 不包含任何方法声明，因此任何类型都可以说是实现了空接口。空接口可以存储任何类型的值，也可以从空接口取出原值。例如用于创建通用的数据容器：

package main

import "fmt"

type Dog struct{}

func printAnything(v interface{}) {
	fmt.Println(v)
}

func main() {
	printAnything("Hello, World!")
	printAnything(123)
	printAnything(Dog{})
}

一个函数接收空接口，表示函数可以接受任意类型的值作参数。同理，返回空接口表示可以返回任意类型的值。

组合接口

Go 语言允许通过组合（embedding）现有的接口来创建新的接口，可以将小的简单接口组合成更复杂的接口：

package main

type Reader interface {
	Read(p []byte) (n int, err error)
}

type Closer interface {
	Close() error
}

type ReadCloser interface {
	Reader
	Closer
}

func main() {
}

也能混合嵌套接口和方法：

package main

type Closer interface {
	Close() error
}

type ReadCloser interface {
	Read(p []byte) (n int, err error)
	Closer
}

func main() {
}

类型断言

类型断言是在运行时检查和转换接口变量的类型的操作，用于从接口类型的变量中提取或确认其值的类型。类型断言的基本形式是：value, ok := 接口变量.(断言类型)。其中返回的 value 如果符合断言类型，则会转为底层值，否则返回零值。而 ok 变量标记类型断言是否成功：

package main

import (
	"errors"
	"fmt"
)

// checkType 检查输入的参数是否是字符串，是的话返回值和 nil 错误，否则返回错误消息
func checkType(t interface{}) (string, error) {
	if value, ok := t.(string); ok {
		return "The type is a string：" + value, nil
	}
	return "", errors.New("the provided value is not a string")
}

func main() {
	value, err := checkType("dog")
	if err != nil {
		fmt.Println(err)
	} else {
		fmt.Println(value) // 输出：The type is a string：dog
	}

	value, err = checkType(42)
	if err != nil {
		fmt.Println(err) // 输出：the provided value is not a string
	} else {
		fmt.Println(value)
	}

	// 等效于 switch 类型断言的写法
	var element interface{} = "Hello, world!"

	if value, ok := element.(string); ok { // 尝试将 element 断言为 string 类型
		fmt.Printf("字符串类型: %s\n", value)
	} else if value, ok := element.(int); ok { // 尝试将 element 断言为 int 类型
		fmt.Printf("整数类型: %d\n", value)
	} else {
		fmt.Println("未知类型")
	}
}
