Go 语言中日志管理

发表于 2024-06-24 更新于 2024-07-23 分类于 Go ， 4.基本应用 阅读次数： 本文字数： 930 阅读时长 ≈ 3 分钟

Log 库

日志指记录程序执行期间发生的信息。可以使用自带 log 包来进行日志管理。log 包提供的功能包括日志记录、自定义日志格式和日志输出定位：

  • 创建日志器：log 包默认使用标准错误流（stderr）输出日志，可以通过 SetOutput 方法来更改日志输出的目标。
  • 设置日志格式：通过 SetFlags 方法设置日志的输出格式，包括日期、时间和文件信息等。
  • 记录日志：可以使用 Print, Printf, Println 来记录普通的日志消息，或使用 Fatal, Fatalf, Fatalln 在记录严重错误日志后结束程序，或使用 Panic, Panicf, Panicln 在记录日志后抛出异常。

Go 的 log 包不像其他语言的日志库那样提供日志级别，所有消息都以同一标准输出。如果需要不同级别的日志处理，得手动定制日志格式或使用第三方日志库（如 zap、zerolog 或 logrus）来实现。

在 log.SetFlags 中可以使用的格式标志：

  • log.Ldate: 在日志消息中包括日期。
  • log.Ltime: 在日志消息中包括时间。
  • log.Lmicroseconds: 在时间中包括微秒。
  • log.Llongfile: 在日志条目中包括完整文件路径和行号。
  • log.Lshortfile: 在日志条目中包括文件名和行号，不包括路径。
  • log.LUTC: 使用 UTC 格式化时间。
  • log.Lmsgprefix: 将前缀放在日志消息的开头。
  • log.LstdFlags: 标准日志器的初始值，等同于 log.Ldate | log.Ltime。

多个标志之间用 | 来分割。

下面是一个使用示例：

package main

import (
	"log"
	"os"
)

func main() {
	// 创建日志文件
	file, err := os.OpenFile("app.log", os.O_APPEND|os.O_CREATE|os.O_WRONLY, 0666)
	if err != nil {
		log.Fatal("Error opening log file:", err)
	}
	defer file.Close()

	// 日志默认输出到终端，默认格式：2024/06/24 14:51:32 An stderr message
	log.Println("An stderr message")

	// 设置日志输出到文件
	log.SetOutput(file)

	// 设置日志前缀和输出格式
	log.SetPrefix("INFO: ")
	log.SetFlags(log.Ldate | log.Ltime | log.Lshortfile)

	// 记录日志信息
	log.Println("An informational message")
	log.Printf("A formatted %s message", "info")

	// 发出致命错误
	log.SetPrefix("ERROR: ")
	log.Fatal("A fatal message") // 程序退出
	log.Println("不会执行的代码")
}

上面代码会生产日志文件 app.log，并写入内容：

INFO: 2024/06/24 14:54:01 main.go:27: An informational message
INFO: 2024/06/24 14:54:01 main.go:28: A formatted info message
ERROR: 2024/06/24 14:54:01 main.go:32: A fatal message

Logrus 库

Logrus 是一个流行第三方日志库，完全兼容标准库 log 包，并做了一些增强，如级别控制、日志条目的结构化、钩子系统、以及可扩展的日志输出格式等。

使用 go get 命令安装 Logrus：

go get github.com/sirupsen/logrus

以下是一个简单的使用例子：

package main

import (
	"github.com/sirupsen/logrus"
)

func main() {
    // 创建日志实例
	log := logrus.New() 

	// 设置日志记录级别为 Info 以上
	log.SetLevel(logrus.InfoLevel)

	// 设置日志格式为 JSON
	log.SetFormatter(&logrus.JSONFormatter{})

	// 记录不同级别的日志
	log.WithFields(logrus.Fields{
		"username": "example_user",
		"scope":    "test_run",
	}).Info("用户登录成功")

	log.WithFields(logrus.Fields{
		"username": "example_user",
	}).Warn("用户尝试访问未授权的资源")
}

Zerolog 库

Zerolog 据称是最快的日志库，核心优势在于对零内存分配的支持。Zerolog 通过减少内存分配、使用高效的 I/O 处理和强制使用 JSON 格式输出，极大地提高了性能。

通过 go get 命令来安装 Zerolog：

go get github.com/rs/zerolog

下面是基础示例：

package main

import (
	"errors"
	"os"
	"time"

	"github.com/rs/zerolog"
	"github.com/rs/zerolog/log"
)

func main() {
	// 输出到标准输出，而不是默认的 stderr，设置日志级别为 info
	log.Logger = zerolog.New(os.Stdout).With().Timestamp().Logger().Level(zerolog.InfoLevel)

	// 模拟一些应用日志
	log.Info().Str("role", "my-service").Str("environment", "production").Msg("服务启动")
	log.Error().Err(errors.New("错误！")).Msg("发生错误")

	// 记录请求日志
	log.Info().
		Str("method", "GET").
		Str("url", "http://example.com").
		Dur("duration", time.Second).
		Msg("HTTP 请求")
}
